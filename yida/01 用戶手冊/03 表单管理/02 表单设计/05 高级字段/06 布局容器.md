## 1. 布局容器组件介绍

### 1.1 什么是布局容器

-   布局：是对已有的可视化组件的位置摆放控制，表示了一种动作；
-   布局容器：是一种有布局能力的容器，可以放入多个组件到布局容器中进行展示；
-   对于复杂场景的布局，我们优先推荐宜搭弹性布局，即[**flex-layout**](https://css-tricks.com/snippets/css/a-guide-to-flexbox/)，也是目前业界前端的主流 CSS 布局方式；
-   对于更详细的 flex 布局介绍，可参见 [**教程链接**](http://www.ruanyifeng.com/blog/2015/07/flex-grammar.html)。

### 1.2 使用场景

-   当页面位置需要分块划分时，比如希望页面按两列进行展示时
-   当页面需要适配不同屏幕尺寸时
-   当页面的 layout 布局有嵌套，位置相对性，位置决定性时
-   当页面的样式实现，需要借助 CSS 等高级布局技术能力时

### 3.2 状态

以传统 PC 端的组件状态进行分组设置：

-   默认：正常状态
-   hover: 鼠标浮动在组件热区，且未产生点击行为
-   focus：页面原色处于光标焦点位置
-   active: 页面元素处于选中激活状态

![[yida_support-wtwabe-zk6dhy-wql6d2_1639361538058-25254118-eb4e-4dba-80e1-7d1a7779459a.png]]

**注：**

-   不同的状态设置，默认状态会覆盖其他 3 种状态
-   而 hover、focus、active 3 种状态的配置则相互隔离，互不影响
-   体现在 CSS 中，是独立的选择器设置

![[yida_support-wtwabe-zk6dhy-wql6d2_1639361693932-6f0b8baa-c659-4308-a0a5-b58bb9067ce2.png]]

### 3.3 显示

对应 CSS 的 display 模式：

-   block：盒子模型
-   inline-block：行内盒子模型
-   inline：行内模型
-   flex：弹性布局模型

![[yida_support-wtwabe-zk6dhy-wql6d2_1639361729080-6ff938db-f112-4dbb-bbe4-b71cde08683d.png]]
