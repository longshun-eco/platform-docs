更新于 2025-09-23本文介绍了工作台组件的稳定性规范。钉钉开放平台提供了统一的、不同指标的稳定性运维标准，开发者需根据稳定性指标判断是否存在稳定性隐患，并及时做出整改计划。

## 名词解释

|  |  |
| --- | --- |
| **名词** | **解释** |
| 标准生态组件 | 应用服务商开发，每一个组件和一个应用绑定，生效于所有安装了该应用的标准工作台。 |
| 定制组件 | 定制服务商开发，生效于定制工作台。详情可参考[开发自建组件](/document/dashboard/dashboard-component-develop-overview#topic-2038185)。 |
| 插件 | 组件的代码集，一个插件内可以包含多个组件，每一个组件可单独承载不同的业务形态。每次功能更新、发布时，只能针对插件维度操作。 |
| OPS | 应用、定制服务商组织内的稳定性跟进人。 |

## 稳定性运维

### OPS跟进

**说明**

OPS需要在告警群中跟进告警，不满足要求会触发相应处罚措施，详情可参考工作台组件稳定性指标及处理措施。

|  |  |  |  |
| --- | --- | --- | --- |
| 组织类型 | 服务商 | 客户 | 钉钉官方 |
| 生态组件服务商 | 至少2名OPS | — | 值班账号 |
| 定制服务商 | 2名OPS | 建议1名OPS | 值班账号 |

### 稳定性得分指标

基于一些关键指标，所有插件通排，钉钉通过特定算法得出。

|  |  |
| --- | --- |
| 指标 | 解释 |
| 页面成功率 | jserror、jserror/pv。 |
| 接口成功率 | 网关接口请求成功率、响应耗时。 |
| 客诉 | 用户投诉。 |
| 故障响应率 | 告警群中的故障预警响应率，平均响应时间。 |

### 稳定性指标及处理措施

|  |  |  |  |
| --- | --- | --- | --- |
| **指标** | | **详情** | 处理措施 |
| 稳定性得分 | 生态组件 | 健康 | * 插件自动审批免抽查 |
| 一般 | * 插件审批定期抽查（被抽查到走手动审批） |
| 不稳定 | * 插件手动审批（效率低） * 插件管理入口提示不稳定 * 需要提交整改计划（基于具体情况给出严重等级） |
| 定制组件 | 健康 | 插件自动审批免抽查 |
| 一般 | 插件审批定期抽查（被抽查到走手动审批） |
| 不稳定 | * 插件手动审批（效率低） * 插件管理入口提示不稳定 * 需要提交整改计划（基于具体情况给出严重等级） * 通知客户服务商系统有隐患问题 |
| OPS人员 | | OPS 跟进人员不满足要求。 | * 产品更新-禁止更新、发布 * 线上问题修改-正常-走人工审核机制 |
| 整改计划 | | **严重**程度周期外超过1倍时间；或**其它**程度周期外超过1个工作日未整改。 | 预警 |
| **严重**程度周期外超过3倍时间；或**其它**程度周期外超过3个工作日未整改。 | 服务商积分扣4分 |
| **严重**程度周期外超过5倍时间；或**其它**程度周期外超过5个工作日未整改 | 服务商积分扣6分 |
| 拒不整改。 | 下架整改并且服务商积分扣12分 |

### 整改计划

未构成系统故障，但根据稳定性指标判断存在稳定性隐患的也需要进行整改，整改周期按严重程度区分，如下表所示：

**说明**

如果未能及时整改，会触发相应的处罚措施，详情可参考工作台组件稳定性指标及处理措施。

|  |  |
| --- | --- |
| 严重程度 | 周期 |
| 轻度 | 要求2~3个发布内进行修复，不宜超过20工作日。 |
| 一般 | 要求1~2个发布内进行修复，不宜超过10工作日。 |
| 严重 | 要求立刻恢复，恢复时间应在小时级（按具体场景定时间）。 |
