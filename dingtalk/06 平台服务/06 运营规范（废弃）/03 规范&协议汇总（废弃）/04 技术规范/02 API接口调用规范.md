## 1. 为什么上架应用必须要满足接口调用规范

上架应用作为面向钉钉所有客户开放购买的商品，需要应用满足基于钉钉的用户体验一致性、客户数据安全性的要求，使得应用获得与官方应用无差别的服务能力，实现应用更广泛、更快速的客户推广和转化。

## 2. 接口规范的更新机制

接口调用规范会不断更新，更新后对已经上架的应用不会产生影响，但新上架应用、应用更新发布需要遵循最新的接口规范。

**接口规范更新时间：2020年02月07日**

## 3. 接口规范要求

以下接口规范中列举的接口，类型为“**全场景必接API**”的如未对接，无法提交上架审核。

|  |  |  |  |
| --- | --- | --- | --- |
| **接口分类** | **接口名称** | **使用场景** | **必接类型** |
| 身份验证 | 第三方企业应用免登 | 用户使用钉钉登录态直接打开应用，无需单独输入账号名称。不允许通过其它账号体系登录，也不允许在应用内引导用户输入登录凭据 | 全场景必接API |
| 通讯录管理 | 用户管理 | 获取应用安装组织的用户基本信息（不含手机号），用于用户添加、授权等管理 | 全场景必接API |
| 部门管理 | 获取应用安装组织的部门信息，用户部门管理  如果是教育类的应用对应接入家校通讯录的学校信息，实现学校、年级、班级管理 | 全场景必接API  (部门管理或学校信息至少接入一类) |
| 角色管理 | 获取应用安装组织的角色信息，进行角色管理、授权等  如果是教育类的应用对应接入家校通讯录的角色信息，实现学生和家长的角色管理 | 全场景必接API  (角色管理或角色信息至少接入一类) |
| 消息通知 | 工作通知消息 | 用户向用户发送钉钉工作通知 | 全场景必接API |

以下接口规范中列举的接口，类型为“**场景关联必接API**”的，如应用中未涉及对应的场景功能可不对接，但必须要备注接口未对接的原因，以便上架审核人员确认。例如OA审批接口在应用中有审批的功能场景时必接，如未对接但上架审核时发现有审批场景，上架审核将被拒绝。

|  |  |  |  |
| --- | --- | --- | --- |
| **接口分类** | **接口名称** | **使用场景** | **必接类型** |
| 身份验证 | 应用管理后台免登 | PC端应用或登录钉钉OA后台使用钉钉登录态直接打开，无需单独输入账号名称。不允许通过其它账号体系登录，也不允许在应用内引导用户输入登录凭据  **应用有PC端必接** | 场景关联必接API |
| 通讯录管理 | 外部联系人管理 | 获取应用安装组织的外部联系人信息，实现客户管理等外部人员对接管理  **应用有录入非组织成员功能场景时必接** | 场景关联必接API |
| 待办任务 | 发起待办 | 发起一个钉钉待办事项  **应用有审批场景必接** | 场景关联必接API |
| 更新待办 | 更新钉钉待办事项状态**应用有审批场景必接** | 场景关联必接API |
| 获取用户待办事项 | 获取用户在应用安装组织的待办任务列表**应用有审批场景必接** | 场景关联必接API |
| 考勤 | 假勤审批 | 用户通过应用提交请假、出差审批，可以和企业钉钉考勤数据关联  **应用有假勤审批场景必接** | 场景关联必接API |
| 文件存储 | 管理钉盘文件 | 应用内管理钉盘文件  **应用有文件存储、处理场景必接** | 场景关联必接API |
| 应用内购 | 应用内购接入开发 | 在应用内实现内购  **应用内有支付场景必接** | 场景关联必接API |

## 4. 接口需求反馈方式

如果在开发过程中，发现某些接口未开放，可以将需求反馈给钉钉团队。我们收到您的需求后，会及时评估并给予回复。提交方法请参考[需求建议](/document/contactus/eupgs5#topic-2625347)。

## Q&A

**Q:应用上架API检测是否可以跳过?**

A：API检测是必须产品流程，不可跳过，全场景必接API必须全部对接后才可对接。

**Q:对规范和政策有疑问可以咨询谁?**

A：对应类目的运营负责人，请查看[运营支持](/document/contactus/mr22h2#topic-1949249)。

**Q:上述规范中没有包含的接口如果想对接是否可以对接?**

A：钉钉开放平台基于钉钉的核心业务能力封装了大量的API，鼓励应用多对接，提升应用的专业化能力和用户体验。

**Q:为什么接口对接了但是API检测显示未接入?**

A：API检测当前日期往前30天的API调用记录，如果当天对接API需要次日才能检测生效。
