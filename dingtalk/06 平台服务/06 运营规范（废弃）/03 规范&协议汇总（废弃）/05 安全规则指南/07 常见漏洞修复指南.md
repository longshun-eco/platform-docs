## SQL注入修复

### 使用预编译方式处理执行的SQL

1. Java语言

   * JDBC使用PreparedStatement预编码和占位符

     预编译使用占用符“?”以后调用setObject()、setInt()、setString()、setSQLXML()等函数实现占用符值的插入，例如：

     ```java
     sql = "select * from users where id = ?";
     JdbcConnection conn = new JdbcConnection();
     PreparedStatement stmt = conn.prepareStatement(sql);
     stmt.setString(1, id);
     ```
   * 使用Mybatis框架

     + 使用“#”替换“$”，例如：

       ```sql
       select * from users where id = #{id}
       ```
     + like语句使用concant()函数，并手动过滤%，例如：

       ```sql
       select * from users where tile like concat(‘%’,#{title}, ‘%’)
       ```
     + mybatis的in语句使用<foreach></foreach>方式，例如：

       ```sql
       select * from users where id in #{item}
       ```
     + mybatis的order by语句使用过滤或者白名单映射的方式。
2. PHP语言

   * 使用PDO扩展进行预处理

     ```php
     $sql = "SELECT id FROM users WHERE id=:id";
     $stmt = $pdo->prepare($sql);
     $id = filter_input(INPUT_GET, 'id');
     $stmt->bindValue(':id', $id);
     ```
3. asp.net

   * 使用Parameters.Append参数处理

     ```sql
     <%
      Dim oComm, oRs
      Set oConn = Server.CreateObject("ADODB.Connection")
      oConn.Open "Provider=MSDAORA;Password=sth;Persist Security Info=True;User ID=whats;Data Source=mescp"
      Set oComm = CreateObject("ADODB.Command")
      oComm.ActiveConnection = oConn
      oComm.CommandType = 1
      oComm.CommandText = "select * from all_objects where rownum = ? "
      oComm.Parameters.Append
      oComm.CreateParameter("v1",3,1,4,100)
      Set oRs = oComm.Execute
      %>
     ```
   * 过滤关键字

     ```sql
     关键字：and|exec|execute|insert|select|delete|update|chr|mid|master|or|truncate|char|declare|join|cmd|drop|xp_cmdshell|xp_delete_file|xp_regread|xp_regwrite|xp_dirtree
     ```
   * 过滤特殊字符

     `' " \ < > & *`

## CORS修复

### 配置跨域白名单

* 方式一：

  通过`@CrossOrigin(origins = {"https://a.examples.com", "https://b.examples.com"})`配置白名单的方式。
* 方式二：

  WebMvcConfigurer全局添加cors白名单，例如：

  ```java
  @Configuration
  public class CorsConfig
  {
      @Bean
      public WebMvcConfigurer corsConfigurer()
      {
          return new WebMvcConfigurerAdapter() {
              @Override
              public void addCorsMappings(CorsRegistry registry) {
                  // 设置cors origin白名单。不允许设置为*，会有漏洞。区分http和https，并且默认不会拦截同域请求。
                  String[] allowOrigins = {"https://a.b.examples.com", "https://c.b.examples.com"};

                  registry.addMapping("/路径")
                          .allowedOrigins(allowOrigins)
                          .allowedMethods("GET", "POST")
                          .allowCredentials(true);
              }
          }
      }
  }
  ```

### 修复验证

1. 在控制台输入`curl -vv -H "Origin: http://hack${你的域名}" http://${你的域名}`进行修复验证。

   例如：`curl -vv -H "Origin: http://hackexamples.com" http://xxxx.examples.com`
2. 观察ACCESS-CONTROL-ALLOW-ORIGIN设置内容。

   * 如果Access-Control-Allow-Origin返回了\*或http://hackexamples.com，则存在漏洞。
   * 如果未返回Access-Control-Allow-Origin头，则不存在漏洞。

## 任意文件读取

1. 确保路径不能使用绝对路径。
2. 确保路径中没有**'..'**这种回退路径。

```java
public static boolean GetPathInfo(String path){
        if (path.charAt(0) == '/'){
            return false;
        }

        if(path.indexOf("..") != -1){
            return false;
        }

        return true;
    }
```

## CSRF修复

* 验证请求是信任页面发起。

  + 在表单中填充一次性随机的 csrf token，同时将此串 token 置入http header和cookie中，在后端对http header和cookie里的token进行一致性校验。
  + referer 验证。
* 验证请求是合法用户发起。

  + 输入验证码。

## SSRF修复

* 禁止跳转。
* 过滤返回信息，验证远程服务器对请求的响应是比较容易的方法。如果web应用是去获取某一种类型的文件，那么在把返回结果展示给用户之前先验证返回的信息是否符合标准。
* 禁用不需要的协议，仅允许http和https请求。可以防止类似于file://, gopher://, ftp:// 等引起的问题。
* 设置URL白名单或者限制内网IP（使用gethostbyname()判断是否为内网IP）。
* 限制请求的端口为http常用的端口，例如 80、443、8080、8090。
* 统一错误信息，避免用户可以根据错误信息来判断远端服务器的端口状态。

## 任意文件上传修复

* 使用OSS存储上传文件

  + 禁止使用主账号的AK/SK。
  + OSS bucket建议设置私有权限，读写权限分离。
  + 建议使用[服务端签名直传并设置回调](https://help.aliyun.com/document_detail/31927.html)。
  + 文件名随机命名避免同名覆盖。
  + 校验上传文件的类型。
  + 限制文件大小。
* 上传文件白名单控制

  + 对上传接口做严格的后缀检测。
  + 采用白名单模式，只允许上传图片文件的格式。
  + 上传文件禁止存储在web目录。

## .svn/.git文件泄露修复

* 方式一：在web执行目录下删除对应.git/.svn等目录。
* 方式二：配置nginx拒绝访问.svn/.git，如下：

  1. 查找nginx配置文件地址。

     ```
     nginx -t
     ```
  2. 添加如下访问配置。

     ```
     location ~ /\.git {
       deny all;
     }
     或者
     # or, all . directories/files in general (including .htaccess, etc)
     location ~ /\. {
       deny all;
     }
     ```
  3. 重启nginx。

     ```
     nginx -s reload
     ```

## web xml 配置文件泄露

* 方式一：反向代理为Apache。

  1. 在 httpd.conf中设置目录访问控制。

     **重要**

     /home/xxx/WEB-INF 是 WEB-INF 所在位置，需根据实际情况配置

     ```
         Order allow,deny
         Deny from all
     ```
  2. 重启Apache服务。
* 方式二：反向代理为nginx。

  1. 在nginx.cong中设置目录访问控制。

     **重要**

     /en/WEB-INF/是 WEB-INF 所在位置，需根据实际情况配置。

     ```
     location  /en/WEB-INF/ {
         deny all;
     }
     ```
  2. 重启Nginx服务。
