## **第一章 目的和范围**

### **1.1 目的**

为营造规范、有序、安全的钉钉开放平台环境，提升钉钉开放平台合作伙伴(以下简称“开发者”或“您”)提供的应用安全性，从而保障开发者及用户的合法权益，特制定此规范指南。

### **1.2. 适用范围**

本规范指南规定了对开发者所提供的应用在开发、部署、运营、维护和管理等方面的安全准则要求，为开发者提供安全指引，适用于钉钉生态内合作伙伴的所有相关人员。

## **第二章 术语和定义**

| 术语 | 定义 |
| --- | --- |
| 钉钉开放平台 | 指由钉钉所拥有并且经营，提供给开发者使用的网站及相关页面，简称“开放平台”。 |
| 开发者 | 指按照开放平台流程经注册、登录，通过技术文档进行软件开发的自然人、法人或者组织。开发者通过开放平台提供的技术文档进行软件开发，服务自身或者其他客户。 |
| 应用 | 指开发者基于开放平台所开发的软件或服务，包括企业内部应用和第三方企业应用、第三方个人应用三种类型。 |
| App key & App secret | 指开发者在申请开发新应用时获得的由钉钉授予的应用程序接入账户和密钥。App key是应用的唯一标识，钉钉通过App key来识别应用开发者的身份。App secret是钉钉给应用分配的密钥，该密钥在一定技术条件下可保证应用来源的可靠性。 |
| 计算巢/聚石塔/钉钉云 | 计算巢/聚石塔/钉钉云给开发者的应用提供了一个非常稳定、安全的环境，其中包括了云主机的安全性、RDS的安全性以及数据集成中的安全性。 |
| 计算巢/聚石塔/钉钉云内调用 | 指由计算巢/聚石塔/钉钉云环境内的系统和应用发起的对钉钉开放平台API的使用行为，计算巢/聚石塔/钉钉云环境包括云主机、应用运行引擎和托管主机等，以开发者发起调用的IP地址是否属于环境认定的IP范围为准。 |
| 敏感数据 | 依据《阿里巴巴集团数据安全规范（总纲）》的规定，敏感数据的范围包括L3级别（保密数据）、L4级别（机密数据）和B2、S2级别（公司/业务内部数据），针对本标准中的敏感数据特指用户的姓名、昵称、电话号码及电子邮箱等敏感数据。 |
| ECS主机 | ECS（Elastic Compute Service，云计算服务）主机是一种简单高效、处理能力可弹性伸缩的计算服务器，可以帮助用户快速构建更稳定、安全的应用。提升运维效率，降低IT成本，使用户更专注于核心业务创新 |
| RDS | RDS（Relational Database Service，关系型数据库服务）是采取主从双机热备架构，具有多重安全防护措施和完善的性能监控体系，并提供专业的数据库备份、恢复及优化方案，使用户能专注于应用开发和业务发展。 |

## **第四章 概述**

开发者所开发的应用为钉钉上的用户提供支撑和服务，必须具备保证其参与钉钉业务的数据安全的能力。本规范指南对开发者所开发的应用所提出的安全要求包括但不限于云设施的安全技术配置、应用的安全功能开发、用户使用的安全风险告知和开发者运营的安全保障管理等方面的要求。

## **第五章 生态数据安全管理规范指南**

钉钉对生态内的数据安全实施严格的管理标准，不仅对数据全生命周期进行安全管理，还对应用运行的基础设施、服务部署实施人员的操作进行明确的规范。钉钉会定期对生态内的数据安全状况进行审计，对发现的任何数据安全风险都会进行及时治理，对于任何违规问题都会给予必要的处罚。

### **5.1 基础设施安全**

任何在钉钉平台提供服务的业务都需要确保支撑业务的基础设施本身安全、可靠、安全、合规，能够持续不间断的提供安全、高可用、高可信的服务，并且需要有专门的运维团队随时保障基础设施的稳定、高效运行。

### **5.2 主机系统安全**

业务运行的主机系统需要具备必要的安全能力，包括但不限于病毒扫描、漏洞扫描、入侵检测、系统恶意行为、非法注入等防护能力。如果业务运行过程中需要对数据进行持久化存储，需要确保数据管理系统也要具备同样的安全防护能力，数据库相关的安全要求参见数据库安全部分。

参照：第6章节

### **5.3 网络安全及高可用**

需要确保提供业务的网络环境具备必要的安全能力，包括但不限于网络防火墙、访问控制管理、路由管理、网络异常行为检测、防DDOS &CC攻击等。

参照：第6章节

### **5.4 身份及授权管理**

业务服务和管理人员需要提供有效的身份认证和授权管理机制，做到数据最小化访问和使用，要根据人员角色变化及时调整权限，避免出现敏感数据由于业务操作人员的不当访问和使用导致的泄露风险。

参照：第6章

### **5.5 数据库安全**

对存有业务相关数据和客户敏感数据的数据库系统，需要对数据库进行必须的帐号管理、授权管理、密钥管理、数据访问控制等，确保对客户数据进行分离度隔离，避免造成数据污染。

参照：第7章节

### **5.6 业务应用软件安全**

业务应用软件安全不仅包括提供服务的运行环境的安全，如使用容器的安全，还包括业务软件自身的漏洞、业务相关使用风险、流程设计缺陷、接口的安全能力等。

参照：第6、9章节

### **5.7 数据采集合规**

业务采集的所有数据都需要在钉钉平台经过用户授权，完成授权以后才可以进行采集，严禁未经许可擅自收集或者引导客户录入不合规数据。

参照：第7章节

### **5.8 数据安全存储**

业务运行期间产生的数据包括日志需要进行安全存储，如有需要，对数据进行加密保护，密钥需要安全保存，严禁将密钥在本地明文保存。

参照：第7章节

### **5.9 数据隔离**

当业务同时向多个客户提供服务时，需要对数据进行相同粒度的隔离和使用，避免用户数据的非正常使用和泄露，确保不同客户彼此数据的完全隔离。

参照：第7章节

### **5.10 数据使用审批及审计**

对业务人员的数据操作和使用需要有完善的审批和审计机制，要做到事前审批，事后审计，及时发现数据的不正当使用情况，并根据风险情况做必要的处置。

参照：第7章节

### **5.11 数据安全共享**

任何与第三方组织之间的数据共享都需要提前经钉钉平台授权许可，不得在未经许可的情况下将数据共享给第三方。获得许可后，需要采取有效的措施对数据的访问进行控制确保只有已授权用户才可以对共享数据进行操作。需要与第三方明确相关的数据安全风险，必要时需要和钉钉团队一起对数据进行预处理。

参照：第7章节

### **5.12 运维风险安全管理**

业务运营人员需要密切关注业务运营的各项风险指标，对异常信息要做到早发现、及时通报、即时处理。

对运维人员的数据访问权限做必要的访问控制，对敏感数据或大量数据的操作需要做审批和备案管理。

参照：第9章节

### **5.13 服务人员安全培训**

对部署服务人员进行安全培训，明确服务标准，提高服务人员数据安全意识，对于服务过程中接触到的客户数据需要严格保密。

参照：第9章节

### **5.14 服务人员安全管理**

对部署服务人员的日常工作和行为进行管理，不得私自获取客户数据，对违规行为要采取必要处罚措施，有效管理服务人员的流动，避免造成客户数据泄露。

参照：第9章节

## **第六章 云设施的安全技术配置**

本部分是对开发者应用所依赖云设施的技术配置所提出的安全要求，包括：云环境的安全部署、云环境安全功能的启用、应用所依赖的主机资源（如：ECS主机和RDS等）的安全性配置和应用相关访问时的安全性配置。

### **6.1 云环境的安全部署**

| 安全项 | 具体要求 |
| --- | --- |
| 阿里云 | a）阿里云部署的第三方应用，只允许向可开放给阿里云的OpenAPI发起数据请求；  b）阿里云部署的第三方应用，应与其他应用分开独立部署，不得混用服务器和数据库，网络和数据应与其他应用做好隔离；  c）阿里云部署的第三方应用，应通过采购云上的安全产品服务或自建安全能力进行安全防护，定期进行安全自查（如安全测试、安全扫描等）或者由第三方安全服务进行安全风险评估或渗透测试，并保存安全防护措施方案和安全检测材料，以备钉钉进行查验核实；  d）阿里云部署的第三方应用，应遵循钉钉的运营要求，若需要获取钉钉开放的敏感数据或者在其他已明确要求情况下（例如需要上架应用），必须将应用迁移到钉钉提供的安全可控（计算巢/聚石塔/钉钉云）环境内；  e）阿里云部署的第三方应用，应在用户协议中注明告知用户数据的存储环境和服务商自主承担用户数据的保护职责；  f）企业内部（自建/定制化）应用，在签署数据授权协议和安全承诺协议情况下，不受以上要求约束。 |
| 计算巢/聚石塔/钉钉云 | a）计算巢/聚石塔/钉钉云部署的第三方应用，原则上允许向钉钉已开放给第三方的所有OpenAPI发起数据请求，且根据业务需求可以申请部分敏感数据的OpenAPI。  b）计算巢/聚石塔/钉钉云部署的第三方应用，应按照钉钉的安全要求进行应用的部署、配置和数据存储，不允许将数据传输存储在非聚石塔/钉钉云环境内；  c）计算巢/聚石塔/钉钉云部署的第三方应用，应接入钉钉的安全域名，接入钉钉安全提供的基础防护和安全巡检审计，配合钉钉处理各项安全风险；  d）计算巢/聚石塔/钉钉云部署的第三方应用，应结合自身业务安全的要求，补充自身的安全能力，例如企业级的入侵检测防护、内容安全检测等；  e）企业内部（自建/定制化）应用，在签署数据授权协议和安全承诺协议情况下，不受以上要求约束。 |

### **6.2 云安全功能的启用**

**6.2.1 边界保护**

| 安全项 | 具体要求 |
| --- | --- |
| 应用的ECS主机要求 | a）所有涉及钉钉定义的敏感数据业务的应用，应使用聚石塔/钉钉云提供的ECS主机或将ECS主机接入到计算巢，但不允许在计算巢/聚石塔/钉钉云管控环境内部署其他非钉钉业务的应用，包括不限于：企业类站点、学校类站点、论坛类站点、政府类站点以及娱乐类站点等；  b）每台ECS主机只能被指定到唯一的安全组且该ECS主机所属的安全组不能更改；  c）对于在同一个安全组内的ECS主机其网络是可互通的，但对该安全组外的ECS主机其网络是不可互通的； |
| 应用的安全隔离 | 如果同一个开发者有多个应用，开发者应为不同的应用使用不同的APP Key，不同的应用需要独立部署在不同的ECS主机中，确保应用之间是被安全隔离的，禁止出现应用混部的问题。 |

**6.2.2 攻击检测及防御**

开发者需先在钉钉开放平台后台--计算巢/聚石塔/钉钉云--云产品开通处开通对应的云产品。若开发者使用的是私有环境，例如阿里云私有VPC，也需考虑购买或自建攻击检测和防御能力。

| 安全项 | 具体要求 |
| --- | --- |
| Web应用防火墙能力 | a）部署在计算巢/聚石塔/钉钉云的应用接入钉钉安全域名，默认提供此能力  b）部署在非计算巢/聚石塔/钉钉云环境，应购买或自建此能力，则Web应用防火墙应具备以下常见功能:  1）常见Web应用攻击性防;  2）CC恶意攻击防护；  3）精准访问控制；  推荐使用云盾Web应用防火墙，进入云盾Web应用防火墙控制台并开启Web应用防火墙。  详见  <https://help.aliyun.com/document_detail/28518.html> |
| 反入侵安全能力 | a）部署在计算巢/聚石塔环境的应用默认提供基础版能力；  b）部署在非聚石塔环境的应用，应购买或自建此能力，则反入侵安全能力应具备以下常见功能:  1) 反入侵行为发现  2) 文件完整性监控  3) 恶意文件/软件检测  推荐使用云安全中心（安骑士），进入云盾安骑士控制台并开启安骑士。  详见 <https://help.aliyun.com/document_detail/31770.html> |

### **6.3 主机的安全配置**

| 安全项 | 具体要求 |
| --- | --- |
| ECS主机管理员账号的重命名及默认口令的修改 | 系统管理员应在安装完成后，对ECS主机系统管理员的默认账号进行重命名，并修改账号的默认口令，修改后的口令应达到一定的口令强度。  说明  a)  对于Windows，应重命名Administrator；  b)  对于Linux，应为需要相关权限的账号配置sudo权限，并且限制root登录。 |
| RDS管理员  账号的重命名及默认口令的修改、 | 数据库管理员在完成RDS数据库的初始化后，对RDS管理员的默认账号进行重命名，并修改账号的默认口令，修改后的口令应达到一定的口令强度。  说明:  a)  对于mysql类型的RDS数据库，应重命名管理员账号 root；  b)  对于sql server类型的RDS数据库，应重命名管理员账号 sa； |
| 删除ECS主机无用账号 | 系统管理员应在安装完成后，删除ECS主机的临时账号和测试账号。 |
| 删除RDS无用账号 | 数据库管理员应在安装完成后，删除RDS的临时账号和测试账号。 |
| 匿名登录 | 对于部署在ECS主机系统上的FTP应用程序，应不得开启匿名登录的功能，其FTP目录不得为操作系统的根目录，并同时不能在Web的目录下。 |
| 安全组白名单配置 | a) ECS主机对公网的默认开放端口仅包括：80、443、21、22，3389，其中21、22、3389远程端口建议设置访问源的IP白名单；  b）ECS主机安全组入网方向，除Web端口，其他端口原则上不应全网开放，限制访问源IP段情况下允许配置全端口开放，137、139、445、1433、1521、2181、3306、3690、5432、6379、9200、9300、11211、27017、27018等高风险端口禁止配置全网开放；  c）ECS主机安全组出网方向，禁止配置全网全端口开放，根据业务需求允许申请固定IP网段全端口开放；  d）RDS数据库白名单只允许申请内网网段的白名单，禁止配置公网白名单；  e) 对于在同一个安全组内的ECS主机之间，其端口都是直接开放且可相互访问的，但在不同的安全组之间的ECS主机之间，其端口默认是禁止的且网络是隔离的。 |

### **6.4 应用的安全配置**

**6.4.1 访问控制**

| 安全项 | 具体要求 |
| --- | --- |
| 基于白名单访问控制 | a）对于APPKEY IP白名单，应用管理员应在钉钉开放平台中设置应用调用API的IP访问来源范围，应绑定主机外网IP且不允许绑定IP网段；  b）对于RDS IP白名单，应用管理员应在部署环境的管理后台，为每一个RDS支持可以访问的ECS主机 IP范围，只允许其配置为特定的1个或多个IP且不允许绑定IP网段，默认情况下RDS是没有外网的访问权限的。 |
| 基于黑名单访问控制 | 应用应提供黑名单的保护机制，通过黑名单来拦截非法的访问，黑名单的纬度包括IP、用户身份。 |

**6.4.2 权限控制**

| **安全项** | **具体要求** |
| --- | --- |

### **7.3 数据保护**

**7.3.1 数据获取**

| **安全项** | **具体要求** |

| --- | --- |

**7.3.4 数据使用**

| 安全项 | 具体要求 |
| --- | --- |
| 数据处理 | 应用在对其敏感数据（比如企业、用户个人隐私数据等）进行后台的处理或计算时，其相关功能的组件和模块应部署云环境内部的系统里，若敏感数据为从钉钉平台上获取，必须将其相关功能组件和模块部署在计算巢/钉钉云/聚石塔内。 |
| 数据展示 | 应用应对涉及敏感数据（比如电话号码、邮箱、钉钉昵称等）的展示，进行脱敏处理（模糊化、匿名处理等）。  建议的脱敏方案：  a) 【手机号】显示\*+后 4 位。如：\*\*\*\*\*\*1050；  b) 【固定号码】显示区号和后三位，如0571-\*\*\*\*\*123；  c) 【邮箱】@前面的字符如果大于或等于3个字符，@前面的字符只显示其前3位且再增加2个\*,并且@后面显示其第1个字符、‘.’分隔符及其后面的字符，但其中间其他字符都统一替换成2个\*，例如：con\*\*@1\*\*.com；如果少于三位，@前面的字符全部显示且再增加2个\*，并且@后面显示其第1个字符、‘.’分隔符及其后面的字符，但其中间其他字符都统一替换成2个\*，例如tt@163.com则显示为tt\*\*@1\*\*.com；  d) 钉钉昵称】显示首/尾各1位，中间加\*例如：a\*1；  e) 【地址】隐藏区/县级以下部分的地址。 |

**7.3.5 数据共享**

| 安全项 | 具体要求 |
| --- | --- |
| 数据共享 | a) 涉及用户隐私数据未经客户的充分授权，不得收集、分析或向任何第三方（包括同一个开发者的不同应用和不同开发者的应用）输出或共享。  b) 涉及数据输出或共享的，应由钉钉侧法务及安全进行审核通过后方可输出及共享(因法律法规要求需向公众公平公开输出数据的情况除外)。 |
