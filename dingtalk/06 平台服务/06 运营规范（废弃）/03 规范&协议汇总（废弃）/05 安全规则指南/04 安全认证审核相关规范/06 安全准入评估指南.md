更新于 2025-09-23本文介绍了安全准入指南，包括应用基础信息、安全技术、安全管理和其他钉钉自检评估项的安全准入评估要求。

## 安全准入评估说明

|  |  |
| --- | --- |
| 指标 | 说明 |
| 基础项 | 适用于S1/S2/S3安全保护等级的应用的基础要求，上架到应用市场及定期安全评估必须满足。 |
| S2/S3基础项 | 适用于S2/S3安全保护等级的应用的基础要求，上架到应用市场及定期安全评估必须满足，在S1安全保护等级的应用作为计分项。 |
| S3基础项 | 适用于S3安全保护等级的应用的基础要求，上架到应用市场及定期安全评估必须满足，在S1/S2安全保护等级的应用作为计分项。 |
| 计分项 | 适用于S1/S2/S3安全保护等级的应用，根据计分项的满足情况计算应用的安全得分，非必须满足。 |
| 加分项 | 适用于S1/S2/S3安全保护等级的应用，作为计算应用的安全得分体系之外单独加分要求项，非必须满足。 |

## 应用安全保护层级说明

### 层级说明

|  |  |  |  |
| --- | --- | --- | --- |
| 层级 | 说明 | | |
| S1 | 满足非敏感行业类目、未调用敏感API、用户月活且开通组织数＜1w的应用。 | | |
| S2 | 满足属于敏感行业类目、调用敏感API、用户月活或开通组织数≥1w三者其一的应用。 | | |
| S3 | 满足属于敏感行业类目、调用敏感API、用户月活或开通组织数≥1w三者其二以上的应用。 | | |

### 分级维度说明

|  |  |
| --- | --- |
| 分级 | 说明 |
| 敏感类目 | 包括数字教育、人事管理、财税管理、商旅出行、数字政务、数字医疗、安全。 |
| 敏感API | 包括可获取个人敏感信息（如手机号、身份证号、身份证图片、通讯信息、地址信息、金融账号信息等）、企业敏感数据（如企业的人事主数据、完整的组织架构数据等）的OpenAPI。 |

## 应用基础信息

**说明**

通过收集基础信息，方便针对开发语言及框架级别的漏洞做预警推送。

|  |  |  |
| --- | --- | --- |
| 基础信息 | 填写说明 | 示例 |
| 1.开发语言 | 填写应用的开发语言。 | Java、PHP、C#、Go、Python、JavaScript等。 |
| 2.开发框架及版本 | 填写应用使用的开发框架及版本。 | Spring Boot 2.4.4、Spring Framework 5.3.x、Spring Cloud、ThinkPHP 5.6、Laravel 5.6、Golang 1.7.3、Django 4.0.4、Flask、.NET Framework、Node.js、Vue.js、Angular.js等。 |
| 3.数据库类型及版本 | 填写应用使用的数据库类型及版本。 | MySQL 5.7、MSSQL 2019、Redis 4.0、MongoDB 4.4等。 |
| 4.连接数据库框架 | 填写应用连接数据库使用的框架。 | Mybatis 3.5.9、JDBC、Hibernate 4.1、JPA 2.1、SQLAlchemy 1.4等。 |

## 安全技术

|  |  |  |  |
| --- | --- | --- | --- |
| 类别 | 操作 | 指标 | 填写说明 |
| 1.网络安全 | 1.1 是否安全配置数据库网络（选“是”弹框提供证据） | 基础项 | 提供数据库实例未配置公网地址的截图则满足要求。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430125.png]] |
| 1.2 是否实现高风险端口白名单管控（选“是”弹框提供证据） | S2/S3基础项，S1计分项 | 提供ECS和数据库配置的安全组规则截图，安全组白名单中无高危端口配置了0.0.0.0/0规则（高风险端口包括22、137、139、445、1433、1521、2181、3306、3389、3690、5432、6379、9200、9300、11211、27017、27018等）。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430145.png]]  ![[services-security-access-evaluation-guide_p430146.png]] |
| 1.3 是否实现网络隔离（选“是”弹框提供证据） | 计分项 | 提供网络分区的拓扑图或云平台VPC网络隔离的分组截图，实现了上钉应用单独VPC或与其他业务系统网段隔离安全组白名单管控。 |
| 2.部署安全 | 2.1 是否开启云平台主账号的双因素认证（选“是”弹框提供证据） | S2/S3基础项，S1计分项 | 在云平台的“访问控制”功能模块，设置主账号开启MFA。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430151.png]] |
| 2.2 是否通过堡垒机或公钥登录服务器进行远程管理（选“是”弹框提供证据） | 计分项 | 配置堡垒机统一管控，禁止密码登录。  提供以下截图，例如： ![[services-security-access-evaluation-guide_abdd50ab-c201-4338-b85c-83c167e4d906.png]] |
| 3.主机安全 | 3.1 是否部署主机反入侵检测系统（选“是-非计算巢部署”弹框提供证据） | 基础项 | * 接入计算巢且完成资源授权的默认满足。 * 若未接入计算巢，需提供已经购买或自建部署的主机反入侵系统（例如阿里云上购买的企业版安骑士或云安全中心）的截图。 |
| 3.2 是否满足无高危主机漏洞和异常事件告警（选“是”弹框提供证据） | S3基础项，S1/S2计分项 | * 部署在云平台上的，需提供云安全中心的“漏洞管理”和“安全告警处理”截图。 * 未部署在云平台上的，需提供计算巢应用管理的“安全风险”页面截图。   截图内显示高危漏洞和异常事件告警数量为0则满足要求。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430153.png]]  ![[services-security-access-evaluation-guide_p430171.png]] |
| 4.应用安全 | 4.1 是否已经接入waf（选“是-已自建/购买waf产品”弹框提供证据） | 基础项 | * 若已经接入配置[钉钉安全域名](/document/isvapp/config-domain-name)，则默认满足。 * 若未接入钉钉安全域名，请提供自有域名的waf防护配置截图，waf配置截图中的域名必须与应用基础信息的开发管理中配置首页地址的域名一致。  **说明**  自有域名的waf防护配置必须是采购的商业waf产品，自建waf不能满足准入的基本要求。 |
| 4.2 是否定期进行漏洞扫描或代码白盒审计（选“是-定期扫描”弹框提供证据） | 基础项 | * 若已经接入配置[钉](https://open.dingtalk.com/document/isv/config-domain-name)[钉钉安全域名](/document/isvapp/config-domain-name)，则默认满足。 * 若未接入，请提供通过第三方工具完成对自有域名的安全扫描或者应用代码白盒扫描的报告。 |
| 5.数据安全 | 5.1 是否已经开启了数据库备份策略（选“是”弹框提供证据） | S3基础项，S1/S2计分项 | 提供自建数据库的备份策略截图或在RDS的“备份恢复”设置备份策略。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430173.png]] |
| 5.2 是否满足敏感数据加密存储（选“是”弹框提供证据） | 计分项 | 提供敏感数据加密存储的截图，例如账号密码、身份证号、手机号等字段的加密截图。  **重要**  若只对账号密码加密，对存储的其他敏感字段未加密则不满足要求。 |
| 6.安全审计 | 6.1 是否已经提供应用日志审计能力（选“是”弹框提供证据） | 基础项 | * 若应用已有功能模块实现用户操作行为的日志审计功能，可提供自实现的功能截图。 * 若应用无，可以通过配置计算巢[日志审计功能](https://help.aliyun.com/document_detail/417461.html)，并提供已配置的截图满足此项要求。   提供以下截图，例如： ![[services-security-access-evaluation-guide_p430175.png]] |
| 6.2 是否实现对日志的统一管理（选“是”弹框提供证据） | 计分项 | 通过sls实现对应用的各种类型日志进行统一管理，且配置存储时长超过6个月。  提供以下截图，例如： ![[services-security-access-evaluation-guide_p430177.png]] |
| 7.内容安全 | 7.1 是否已建设内容安全检测机制（选“是”弹框提供证据） | 计分项 | 购买内容安全检测服务（如阿里云的内容安全服务）或自建内容安全检测机制，提供相应的截图证明。 |

## 安全管理

|  |  |  |  |
| --- | --- | --- | --- |
| 类别 | 操作 | 指标 | 填写要求 |
| 1.安全组织和制度 | 1.1 设置安全联系人 | 基础项 | 填写安全联系人姓名，用于安全风险的定向通知。 |
| 1.2 设置安全联系人的联系方式 | 基础项 | 填写安全联系人手机号，用于安全风险的定向通知。 |
| 1.3 安全组织架构 | 基础项 | 上传公司目前的安全组织架构的截图或者文档。 |
| 1.4 安全管理制度 | 基础项 | 上传公司已有的安全管理制度内容截图或文档。管理制度内容需包括网络、应用、数据等维度的安全管理规范。 |
| 1.5 应急预案和流程 | 基础项 | 上传应急预案及流程的内容截图或文档。 |
| 2.安全评估自检 | 2.1 提交安全自查报告 | 基础项 | 上传第三方安全工具的安全扫描报告，或参考[安全测试报告指南](/document/isvapp/security-certification)的要求完成安全自测的自测报告。 |
| 3.安全认证 | 3.1 已取得的安全认证证书（选“是”弹框提供证据） | 加分项 | 提供取得的安全认证证书截图或报告，例如等级保护、ISO27001、SOC2隐私性原则审计等认证证书。 |

## 其他钉钉自检的评估项

|  |  |  |
| --- | --- | --- |
| 操作 | 指标 | 自测通过要求 |
| 1.是否完成应用自采集的用户信息使用说明填写 | 基础项 | 在**基础信息** > **权限管理**页面，单击用户信息使用说明，填写采集的用户信息及使用场景说明。钉钉会自行检测，已填写过即可。示例如下： ![[services-security-access-evaluation-guide_p430412.png]] |
| 2.是否接入《隐私协议》和《授权详情》 | 基础项 | 现阶段上架到钉钉广场默认接入满足要求。示例如下： ![[services-security-access-evaluation-guide_p430182.png]] |
| 3.是否已经签署钉钉的《安全责任共担及安全承诺》 | 基础项 | 已签署安全协议，使用管理员角色权限的账号登录钉钉合作伙伴平台，在**产品方案** > **合同管理**页面，找到《安全责任共担及安全承诺》”点击查看签署。签署链接和示例如下：https://partner.dingtalk.com/indexReource#/isv\_service/contract\_manage ![[services-security-access-evaluation-guide_p430185.png]] |
| 4.是否按照网络架构最佳实践部署应用 | S2/S3基础项，S1计分项 | 参考[架构最佳实践](/document/isvapp/architecture-best-practices)部署应用，保证应用的稳定性和扩展性。 |
| 5.是否已经接入waf | 基础项 | 接入钉钉安全域名则默认满足。 |
| 6.是否接入数据安全中心，开启默认防护 | 基础项 | 使用计算巢+数据安全中心的部署方式，参考[数据安全中心操作流程](/document/isvapp/security-center-operation-process-and-faq)完成数据安全中心的接入配置，在应用上架的功能自检中通过此项校验。 |
