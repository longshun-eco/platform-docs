更新于 2025-09-23本文介绍了安全年度评估的检查项。

## 年度评估检查项说明

年度评估检查项适用于**已上架**的第三方企业应用。

## 安全年度评估检查项

| 安全类 | 安全域 | 序号 | 安全要求 | 安全条件 | 要求类型 |
| --- | --- | --- | --- | --- | --- |
| 1.技术安全能力 | 1.1 网络安全 | 1.1.1 | 出口ip白名单管控 | 应用开发管理模块已配置ip白名单地址 | 基础要求项 |
| 1.1.2 | 数据库网络配置 | 数据库未配置公网地址 | 基础要求项 |
| 1.1.3 | 安全组白名单管控，高危端口禁止全网开放 | 安全组无高危端口配置0.0.0.0/0 | S2/S3基础要求项，S1计分项 |
| 1.1.4 | 架构最佳实践 | 部署架构为SLB+ECS集群+RDS+NAT网关 | S2/S3基础要求项，S1计分项 |
| 1.1.5 | 网络分区隔离 | 单独VPC或与其他业务系统网段隔离白名单管控 | 计分要求项 |
| 1.2 部署安全 | 1.2.1 | 云控制台维护管理云上资源 | 接入计算巢 | 基础要求项 |
| 1.2.2 | 通过堡垒机或公钥登录服务器进行远程管理 | 配置堡垒机统一管控 | 计分要求项 |
| 1.2.3 | 云平台账号开启MFA | 主账号开启双因素认证 | S2/S3基础要求项，S1计分项 |
| 1.3 主机安全 | 1.3.1 | 部署主机反入侵检测系统 | 接入计算巢或购买hids | 基础要求项 |
| 1.3.2 | 无高风险漏洞和异常事件告警 | S3基础要求项，S1/S2计分项 |
| 1.4 应用安全 | 1.4.1 | 全站https部署 | 接入钉钉的应用使用域名且已配置https | 基础要求项 |
| 1.4.2 | 配置WAF防护 | 已接入钉钉安全域名或购买/自建waf | 基础要求项 |
| 1.4.3 | 漏洞扫描或代码白盒审计 | 已接入钉钉安全域名 | 基础要求项 |
| 1.5 数据安全 | 1.5.1 | 数据防泄漏 | 接入数据安全中心，开启默认规则 | 基础要求项 |
| 1.5.2 | 数据安全中心无异常高风险告警 | S3基础要求项，S1/S2计分项 |
| 1.5.3 | 开启数据库备份配置 | 配置RDS备份策略 | S2/S3基础要求项，S1计分项 |
| 1.5.4 | 敏感数据加密存储 | 各类用户敏感数据（密码、手机号、身份证等）加密存储 | 计分要求项 |
| 1.6 安全审计 | 1.6.1 | 异常流量分析审计 | 接入计算巢 | 基础要求项 |
| 1.6.2 | 网络、主机、数据、应用的审计，使用SLS统一存储，且审计记录保留6个月以上 | 应用开启并配置计算巢的日志审计功能或自实现日志审计功能 | 基础要求项 |
| 1.6.3 | 日志统一收集到SLS | 计分要求项 |
| 1.7 隐私合规 | 1.7.1 | 应用采集的用户信息上报 | 在应用基础信息的权限管理填写采集的用户信息及使用场景说明 | 基础要求项 |
| 1.7.2 | 隐私政策和数据权限清单，明确采集的数据类型、使用目的和场景 | 上架应用默认接入钉钉提供的隐私政策模板和授权清单 | 基础要求项 |
| 1.8 内容安全 | 1.8.1 | 具备内容安全检测能力机制 | 购买内容安全检测服务或自建基础的内容安全检测机制 | 计分要求项 |
| 2.安全管理能力 | 2.1 安全组织和制度 | 2.1.1 | 在开发者后台设置安全负责人名单和联系方式 | 设置安全负责人名单和联系方式，录入到开发者后台 | 基础要求项 |
| 2.1.2 | 提供安全组织架构和职责、安全管理制度（含网络、应用、数据安全等） | 提供安全组织架构和职责、安全管理制度（含网络、应用、数据安全等）截图 | 基础要求项 |
| 2.1.3 | 制定应急预案和流程，并设置对接人（需包括数据泄露、数据破坏等重大风险事件） | 提供应急预案、对接人和相关记录截图 | 基础要求项 |
| 2.2 基础运营 | 2.2.1 | 签署安全协议 | 已签署钉钉的《安全责任共担及安全承诺》 | 基础要求项 |
| 2.2.2 | 应用基础信息上报 | 已填写应用基础信息，例如应用的开发语言及框架、数据库类型及版本等 | 基础要求项 |
| 2.3 安全认证 | 2.3.1 | 开展等级保护测评认证，并提供定级备案材料。（或其他安全资质认证，如ISO27001、SOC审计等） | 提供已取得的安全认证证书的截图证明 | 加分要求项 |
| 2.4 安全运营 | 2.4.1 | 年度安全评估自查，提交年度安全自查报告 | 提供年度安全自查报告 | 基础要求项 |
| 2.4.2 | 安全违规行为响应及处理 | 违规将虚假资源录入到计算巢，脱离钉钉的安全风险检测管控 | 计分要求项 |
| 2.4.3 | 其他安全违规行为24小时处理并3天内反馈 | 计分要求项 |
| 2.4.4 | 重大安全事件响应及处理 | 重大数据泄露事件人工通知后超过24小时未响应及处理 | 计分要求项 |
| 2.4.5 | 严重高危漏洞事件人工通知后超过3天未完成自查及反馈 | 计分要求项 |
| 2.4.6 | 安全巡检风险响应及处理 | 高危主机异常告警超过3天未处理 | 计分要求项 |
| 2.4.7 | 数据泄露告警事件超过3天未处理 | 计分要求项 |
| 2.4.8 | 高危漏洞超过10天未处理 | 计分要求项 |
| 2.4.9 | 高风险配置项超过10天未处理 | 计分要求项 |
| 2.4.10 | 数据库配置公网地址超过10天未处理 | 计分要求项 |
| 2.4.11 | 未接入安全域名超过10天未处理 | 计分要求项 |
| 2.4.12 | Web漏洞工单超时未处理 | 计分要求项 |
