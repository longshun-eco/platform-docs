## 一、概述

为营造安全、规范、有序的开发环境，保障开发者（亦称“服务商”）及其用户（即开发者通过应用进行服务的钉钉平台其他用户）的合法权益，依据《钉钉产品方案商入驻服务协议》、《应用上下架流程》、《技术规范》、《应用市场运营规则》、钉钉平台相关规则等内容制定本规范，适用于钉钉开放平台（以下简称“开放平台”）所有开发者及所有应用。

本规范系对《钉钉产品方案商入驻服务协议》的有效补充。本规范与《钉钉产品方案商入驻服务协议》有冲突的，以本规范为准。

## 二、违规处罚措施

违规处罚措施开发者应当遵守《钉钉产品方案商入驻服务协议》（[请点击查看](/document/isvapp/dingtalk-service-agreement-with-customized-service-providers)）、《应用上下架流程》、《技术规范》、《应用市场运营规则》及开放平台将来可能发布的各类规则。如有违规，开放平台有权对开发者行为及应适用的规则进行单方认定，针对应用或开发者视情节轻重采取一项或多项处罚措施：

1）扣分：开放平台根据违规行为的定级扣除相应的运营基本分（默认为36分）；

2）处罚金额：开放平台根据违规行为的定级处罚不同金额的罚款；

3）警告：由开放平台发出警告通知，并限时整改；

4）约谈：开放平台以邮件等方式正式发出约谈信函通知；

5）广场不展示：开发者应用在钉钉的应用市场上不展示；

6）广场下架：开发者应用在钉钉的应用市场上直接下架，用户无法搜索到应用；

7）通报：开放平台根据违规级别将安全违规行为和处罚结果通报给服务商的接口人、对接负责人、CEO等；

8）公示：开放平台将安全违规行为的处罚结果发布在开放平台官方网站上进行公示；

9）开放平台根据市场管理需要不时制定的其他措施。

## 四、处罚标准

**1、安全违规的处罚标准**

|  |  |  |  |
| --- | --- | --- | --- |
| 扣分 | 处罚方式 | 通报级别 | 公示 |
| 0~5分 | 警告 | 接口人 | 不公示 |
| 6~11分 | 约谈 | 对接负责人 | 官网公示 |
| 12~15分 | 钉钉内（包括不限于广场、工作台等）不展示7天 | 服务商CEO | 官网公示并服务商群内通知 |
| 16~23分 | 钉钉内（包括不限于广场、工作台等）下架7天 | 服务商CEO | 官网公示并服务商群内通知 |
| 24~35分 | 钉钉内（包括不限于广场、工作台等）下架30天 | 服务商CEO | 官网公示并服务商群内通知 |
| 36分 | 永久下架 | 服务商CEO | 官网公示并服务商群内通知 |

注：如果同一开发者在同一天之内存在多次安全违规行为时，对其安全处罚按最高扣分标准执行。

### **2、安全违规的处罚原则**

1） 如果同一开发者在同一天之内存在多次安全违规行为时，对其安全处罚将需执行如下原则：a) 该开发者在同一天内多次安全违规行为，将根据违规行为对其累计扣分；b）该开发者在同一天之内，对其处罚的累计扣分大于等于36分时，开放平台将进行应用永久下架处罚。

2） 开发者及其发布的服务或应用一旦违规，钉钉将依据其违规程度给予不同类型的处罚，同时暂停该开发者所有线上运营推广资源，并对开发者给予扣分、罚款及处罚结果公示等处罚；

3） 因违规被处罚的开发者及应用，当其全部违规行为已被纠正后，方可恢复正常状态；

4） 所有违规在要求的整改时限内未及时整改，则视为同一违规内容再次违规，对该应用按照相关规定再次处罚，且对应级别违规次数增加一次；

5） 与服务市场的安全违规处罚为联动处罚机制，无论触犯开放平台或应用市场中任一规则，都将依据相关处罚标准对开发者同时执行启动处罚机制；

6） 违规行为的认定与处罚，应基于违规举证或钉钉认定的事实依规执行。但钉钉有权依据实际违规严重情况加重或减轻处罚力度，处罚措施包括但不限于：扣分、罚款、警告、约谈、应用不展示、应用下架。

### **3、安全违规处罚金额用途**

1）若开发者的安全违规行为是被外部白帽子发现并上报，则处罚金额的一部分用于提供给白帽子作为奖励；

2）为安全违规被处罚的开发者引入第三方安全服务商进行安全评估或渗透测试服务，将处罚金额作为服务费用；

3）纳入钉钉为开放平台开发者提升安全能力的资金池，用于给开发者的应用提供专项安全能力服务。
