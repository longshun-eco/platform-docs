## **一、API接口介绍**

### **1.概述**

钉钉应用市场致力于为企业用户提供：满足丰富办公场景需求、使用体验优秀、与钉钉实现有意义结合的优质产品服务。第三方应用需使用钉钉开放平台提供的API、SDK，与钉钉实现全面结合，提升用户使用体验，并保证企业在钉钉内的信息同步，与钉钉共同打造体验优秀的钉钉产品生态，创建企业客户愿意为之付费的应用。

本规范将详细介绍第三方应用在开发过程中，使用API的规范。

## **二、API接口规范**

### **1. 为什么上架应用必须要满足接口调用规范**

上架应用作为面向钉钉所有客户开放购买的商品，需要应用满足基于钉钉的用户体验一致性、客户数据安全性的要求，使得应用获得与官方应用无差别的服务能力，实现应用更广泛、更快速的客户推广和转化。

### **2. 接口规范的更新机制**

接口调用规范会不断更新，更新后对已经上架的应用不会产生影响，但新上架应用、应用更新发布需要遵循最新的接口规范。

### **4. 接口权限及申请**

调用钉钉服务端接口前，需要先添加所需接口的调用权限，权限点用于访问应用运行所需的企业或用户数据。钉钉开放平台鼓励应用以场景合理、能力丰富为目标，通过多种接口与钉钉尽可能深入地融合，实现跟钉钉核心能力的全面互通。但基于对用户数据安全的保护义务，您需要通过申请的形式为**您创建的第三方企业应用**添加相关的权限点。

**接口权限是以应用为维度授予的**，每个应用的接口权限都是独立存在的，如果多个应用都需要调用同一个接口，那么每个应用都要添加相对应的接口调用权限。

例如：企业内同时拥有A、B两个应用，且应用A已经拥有通讯录权限，此时如果应用B要获取企业通讯录，那么也需要添加通讯录权限，添加方式与应用A一样。

#### **权限分类分级标准**

钉钉对第三方企业应用开发将使用到的权限进行了分类分级，下表可以帮助您更好的理解不同类型权限点的定义。

|  |  |
| --- | --- |
| **权限等级** | **权限点说明** |
| 特殊权限点 | 权限点涉及到的功能或相关数据属于敏感范畴，权限点等级高，需进行申请开通，非必需情况下不允许开通。 |
| 高级权限点 | 特定类目下大多数应用需使用的、与类目属性/核心功能相关的权限点（范围由钉钉定义）。若您应用不属于对应类目时需要进行申请开通，若属于则会直接通过。 |
| 普通权限点 | 权限点涉及的功能以及相关数据属于非敏感范畴且保障应用基本运行的权限点，点击申请即可直接通过。 |

#### **权限申请原则**

权限申请需要遵循合理及必要原则。

钉钉开放平台鼓励应用以场景合理、能力丰富为目标，通过申请并使用多种权限点，实现应用与钉钉的能力全面互通，但无节制地申请权限点既会对使用应用的用户及企业的数据安全造成威胁，同时也为应用本身的安全心智带来了负面影响、增加用户在开通应用前的顾虑。

为和您共同打造安全可靠的开放平台生态，钉钉将通过权限分级以及审核流程协助您更合理、有效地申请并使用权限。

#### **权限申请流程**

对于本规范发布后创建或上架的应用需使用的权限点，或存量应用拟新增的权限点，开发者应提交权限申请流程。详细的权限申请流程暂可参考[添加接口调用权限](/document/development/add-api-permission)。我们将会在未来对权限点的描述以及申请流程进行全面升级，届时会通过操作指引文档的形式对升级后的权限申请流程通知到您。

钉钉开放平台的审核流程通过后，在您组织开通应用时，同样需要经过企业管理员的授权同意后，应用申请的权限点才可以在该企业内使用。

### **5. 权限点使用管理机制**

出于对平台企业及用户数据安全保护的考量，钉钉会统计您应用申请权限点的实际使用情况并根据统计结果，对您已申请但长期未被调用的权限点进行提示并逐步回收，以更好地协助您管理并使用所申请的权限点。

#### **权限点使用率**

权限点使用率 = 近180天申请并使用的权限点数 / 当前申请的总权限点个数

例如：应用A共申请了10个权限点，但在180天内仅有其中的8个权限点存在使用记录，则应用A的权限点使用率为80%。

#### **权限点动态管理机制**

**要求一：**您需要保障您应用的**权限点使用率超过（大于等于）80%，即已申请权限点的未使用率低于20%**。

若您应用的权限点使用率低于80%，平台会通知并提示您需要对申请的权限点进行相关处理操作，具体可参考如下流程，待能力正式上线后平台会给到您详细的操作指引：

1. 平台会通过运营渠道等方式向应用负责人发送相关通知，列举长期未使用的权限点明细；
2. 您需要进入开发者后台确认未被使用的权限点是否不满足合理必要的要求；
3. 撤回&保留申请：

   1. 对您确认可以撤回的权限点，我们将提供撤回能力，您可以通过权限点撤回功能进行操作；
   2. 对您认为存在业务必要性、需要进行保留的权限点，届时可以通过提交申请工单的方式，描述未被使用的权限点的具体业务用途，由钉钉小二审核确认后可以进行权限点保留；
   3. 在您应用的**权限点使用率长期低于80%、平台进行多次通知且仍未治理**的前提下，平台有权利对您应用近180天内未使用过的权限点发起主动回收的发布行为进行限制；
4. 在您对您应用申请权限点进行操作并存在申请权限点变更的情况时，我们会协助您向企业管理员再次发起授权链路，对权限点变更项及当前所需的权限点进行说明与请求授权操作。

**要求二：**您需要保障您应用的权限点**符合“合理必要”的原则**，对**存量**不符合权限申请原则或违规收集用户与企业信息的权限点及时进行撤回操作。

平台将不定期对您申请的权限点进行审核，若发现您应用所使用/调用的权限点存在疑似不符合“合理必要”的情况时，同样会按照**要求一**中的流程对存在疑问的权限点进行询问与通知。

您仍可以在确认后针对有明确业务需要的权限点进行申诉，同时有义务对不符合申请原则的权限点进行撤回操作。

### **6. 接口需求反馈方式**

如果在开发过程中，发现某些接口未开放，可以将需求反馈给钉钉团队。我们收到您的需求后，会及时评估并给予回复。提交方法请参考[需求建议](/document/services/eupgs5)。

## **Q&A**

**Q:应用上架API检测是否可以跳过?**

A：API检测是必须产品流程，不可跳过，全场景必接API必须全部对接后才可对接。

**Q:对规范和政策有疑问可以咨询谁?**

A：对应类目的运营负责人，请查看[运营支持](/document/services/mr22h2)。

**Q:上述规范中没有包含的接口如果想对接是否可以对接?**

A：钉钉开放平台基于钉钉的核心业务能力封装了大量的API，鼓励应用多对接，提升应用的专业化能力和用户体验。

**Q:为什么接口对接了但是API检测显示未接入?**

A：API检测当前日期往前30天的API调用记录，如果当天对接API需要次日才能检测生效。
