更新于 2025-09-10调用本接口，创建日志。

**说明**

调用本接口创建日志，对应日志模板中的每个组件只允许是文本类型，其他类型组件暂不支持接口调用。

调用本接口，可创建日志。接口调用效果与下图操作实现效果一致。

![[development-create-a-log_p521894.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 管理员工日志数据权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.report.create) |
| 第三方企业应用 | 是 | 管理员工日志数据权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.report.create) |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/report/create`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | bE3xxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| create\_report\_param | OapiCreateReportParam | 是 |  | 创建日志的参数对象。 |
| contents | OapiReportContentVo[] | 是 |  | 日志内容数组，根据该日志模板中每一项信息传参。 |
| sort | Number | 是 | 0 | 写日志对应的模板某个字段的唯一序列ID，可调用[获取模板详情](/document/development/query-template-details)接口获取sort参数值。 |
| type | Number | 是 | 1 | 写日志对应的模板某个字段的类型，可调用[获取模板详情](/document/development/query-template-details)接口获取type参数值。  **说明**  只支持文本类型日志组件，即type参数固定值为1，其他类型不支持。 |
| content\_type | String | 是 | markdown | 日志内容的类型。  **说明**   * 目前支持markdown类型。 * 支持[获取模板详情](/document/development/query-template-details)接口中，日志组件是**文本**的类型。 |
| content | String | 是 | ### 序号1 | 日志内容。  **说明**   * 只支持 Markdown 语法。 * 内容不能超过 1000 字符，超出的内容会被截断。 |
| key | String | 是 | 今日完成工作 | 写日志对应的模板某个字段的标题，可调用[获取模板详情](/document/development/query-template-details)接口获取field\_name参数值。 |
| template\_id | String | 是 | 12345abcde | 模板ID，可调用[获取模板详情](/document/development/query-template-details)接口获取id参数值。 |
| to\_userids | String[] | 否 | ["123","456"] | 日志发送到的员工userId。 |
| to\_chat | Boolean | 是 | true | 发送日志到员工时是否发送单聊消息。   * **true**：发送日志消息给指定用户 * **false**：不单独发送消息 |
| to\_cids | String[] | 否 | 123 | 日志要发送到的群ID。配置接收群后，可调用[获取模板详情](/document/development/query-template-details)接口获取**conversation\_id**参数值，即为发送到的群ID。  **重要**   * 该群是在日志模板中预先已经配置好的接收群。如果没有配置，则无法发送到指定的群。 * 进入**钉钉工作台**，然后选择**日志 > 模板管理 > 权限设置 > 默认发送范围**，设置该日志模板的默认接收群。 ![[development-create-a-log_p521908.png]] |
| dd\_from | String | 是 | report | 日志来源，每个组织可以自己起一个唯一的来源标识，自定义的值。 |
| userid | String | 是 | 12345 | 创建日志的员工userId。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/report/create?access_token=ACCESS_TOKEN
```

请求正文

```json
{
    "create_report_param": {
        "contents": [
            {
                "content_type": "markdown",
                "sort": "0",
                "type": "1",
                "content": "“Mg+ZnS04=MgS04+Zn”你的镁偷走了我的锌",
                "key": "今日完成工作"
            },
            {
                "content_type": "markdown",
                "sort": "1",
                "type": "1",
                "content": "“哪有什么满眼星辰，分明就是满眼的你。",
                "key": "未完成工作"
            }
        ],
        "dd_from": "ygx",
        "template_id": "1734xxxxxxe08500e",
        "userid": "user123",
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/report/create");
OapiReportCreateRequest req = new OapiReportCreateRequest();
OapiCreateReportParam createReportParam = new OapiCreateReportParam();
List list3 = new ArrayList();
OapiReportContentVo obj4 = new OapiReportContentVo();
list3.add(obj4);
obj4.setSort(0L);
obj4.setType(1L);
obj4.setContentType("markdown");
obj4.setContent("### 序号1");
obj4.setKey("字段1");
createReportParam.setContents(list3);
ArrayList toUserids = new ArrayList<>();
toUserids.add("user123");
toUserids.add("user456");
createReportParam.setToUserids(toUserids);
createReportParam.setTemplateId("12345abcde");
createReportParam.setToChat(true);
createReportParam.setDdFrom("report");
createReportParam.setUserid("user123");
ArrayList toCids = new ArrayList<>();
```

**返回示例**

```json
{
    "errcode": 0,
    "result": "1734xxxxxxe08500e",
    "request_id": "5kaikoe9uc8i"
}
```
