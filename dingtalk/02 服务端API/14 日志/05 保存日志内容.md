更新于 2025-09-10调用本接口，保存自定义的日志内容，后续进入钉钉日志可在写日志页面再拉取此内容。以便将第三方系统的日志集成至钉钉日志中。

## **接口调用说明**

调用本接口，请配合[获取日志接入流程](/document/development/log-api-use-cases)案例使用，相当于把需要发送的日志内容打包，在需要发送日志的时候拉取该内容显示到日志模板中。

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 管理员工日志数据权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.report.savecontent) |
| 第三方企业应用 | 是 | 管理员工日志数据权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.report.savecontent) |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/report/savecontent`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | bE3xxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| create\_report\_param | OapiCreateReportParam | 是 |  | 保存日志的参数对象。 |
| contents | OapiReportContentVo[] | 是 |  | 日志内容数组。 |
| sort | Number | 是 | 0 | 写日志对应的模板某个字段的唯一序列ID，可调用[获取模板详情](/document/development/query-template-details)接口获取sort参数值。 |
| type | Number | 是 | 1 | 写日志对应的模板某个字段的类型，可调用[获取模板详情](/document/development/query-template-details)接口获取type参数值。  **说明**  只支持文本类型日志组件，即type参数固定值为1，其他类型不支持。 |
| content\_type | String | 是 | markdown | 日志内容的类型。  **说明**   * 目前支持markdown类型。 * 支持[获取模板详情](/document/development/query-template-details)接口中，日志组件是**文本**的类型。 |
| content | String | 是 | ### 序号1 | 日志内容。  **说明**  只支持Markdown语法。 |
| key | String | 是 | 字段1 | 写日志对应的模板某个字段的标题，可调用[获取模板详情](/document/development/query-template-details)接口获取field\_name参数值。 |
| template\_id | String | 是 | 12345abcde | 模板ID，可调用[获取模板详情](/document/development/query-template-details)接口获取id参数值。 |
| dd\_from | String | 是 | report | 日志来源，每个组织可以自己起一个唯一的来源标识。 |
| userid | String | 是 | 12345 | 创建日志的员工的userId。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| result | String | 12345 | 调用结果。 |
| errcode | Number | 0 | 返回码。 |
| errmsg | String | ok | 返回码描述。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/report/savecontent?access_token=ACCESS_TOKEN
```

请求正文

```
{
    "create_report_param": {
        "contents": [
            {
                "content_type": "markdown",
                "sort": "0",
                "type": "1",
                "content": "“这里是今天已经完成的工作。",
                "key": "今日完成工作"
            },
            {
                "content_type": "markdown",
                "sort": "1",
                "type": "1",
                "content": "“这是待处理的任务。",
                "key": "未完成工作"
            }
        ],
        "dd_from": "report",
        "template_id": "1734bff776fcc65ce6944f749e08500e",
        "userid": "user456"
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/report/savecontent");
OapiReportSavecontentRequest req = new OapiReportSavecontentRequest();
OapiCreateReportParam obj1 = new OapiCreateReportParam();
List list3 = new ArrayList();
OapiReportContentVo obj4 = new OapiReportContentVo();
list3.add(obj4);
obj4.setSort(0L);
obj4.setType(1L);
obj4.setContentType("markdown");
obj4.setContent("### 序号1");
obj4.setKey("字段1");
obj1.setContents(list3);
obj1.setTemplateId("12345abcde");
obj1.setDdFrom("report");
obj1.setUserid("12345");
req.setCreateReportParam(obj1);
OapiReportSavecontentResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
     "errcode": 0,
     "result": "175da298c2d6eb61373bb314725ae15d",
     "request_id": "14g3ttfdbjfpv"
}
```
