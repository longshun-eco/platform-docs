更新于 2025-09-23本文档介绍了如何调用服务群接口在服务群发送消息等流程。首先创建一个企业内部应用，再使用服务群提供的API，实现创建场景服务群、发送服务群消息和群发任务流程。

## 流程简介

步骤一，登录[开发者后台](https://open-dev.dingtalk.com/#/)，创建[企业内部应用](/document/dingstart/create-application)。

步骤二，获取AppKey和AppSecret。

步骤三，[申请服务群接口权限](/document/development/add-api-permission)。搜索“服务群”，申请相应的权限。

步骤四，获取应用访问凭证[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)。

步骤五，调用服务端服务群相关API。

1. 调用服务端API-[创建场景服务群](/document/development/create-a-scenario-service-group)接口，进行创建场景服务群，获取服务群openConversationId。
2. 根据服务群openConversationId，调用服务端API-[添加服务群成员](/document/development/add-service-group-members)接口，新增服务群成员。
3. 根据服务群openConversationId发送消息。

   * 单个群发送消息，根据服务群openConversationId，调用服务端API-[发送服务群消息](/document/development/service-group-message-sending-interface)接口，实现单个群消息的发送
   * 群发消息，根据不同服务群openConversationId，调用服务端API-[群发任务](/document/development/service-group-sending-task-interface)接口，实现不同服务群进行群发消息。
4. 根据服务群openConversationId，调用服务端API-[查询服务群活跃用户](/document/development/queries-active-service-users)接口，查看服务群的活跃用户。

## 步骤一：创建企业内部应用

**说明**

如果已有企业内部应用，可直接使用已有应用，可忽略此步骤。

1. 登录[开发者后台](https://open-dev.dingtalk.com/#/)，创建[企业内部应用](/document/dingstart/create-application)。

   * 应用类型：选择H5微应用。
   * 开发方式：选择企业自主开发。

## 步骤四：获取应用访问凭证accessToken

**重要**

服务端API差异详情参见[旧版服务端API、新版服务端API的区别](/document/development/how-to-call-apis#section-8lr-id4-rbz)。以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK下载](/document/development/download-the-server-side-sdk)。
