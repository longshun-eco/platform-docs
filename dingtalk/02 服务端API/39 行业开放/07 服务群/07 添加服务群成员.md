更新于 2025-09-23调用本接口，将企业下成员添加到智能服务群中。

### 接口功能介绍

调用本接口实现的效果与下图操作实现效果一致，可以向已存在的智能服务群中添加企业下成员。

![[development-add-service-group-members_O1CN018x7eU51f96ORNLm1J_!!6000000003963-2-tps-2484-1272.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 场景服务群读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=serviceGroup_1.0%23AddMemberToServiceGroup) |
| 第三方企业应用 | 暂不支持 | 场景服务群读写权限 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 场景服务群读写权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/serviceGroup/groups/members HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "openTeamId" : "String",
  "openConversationId" : "String",
  "userIds" : [ "String" ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| openTeamId | String | 是 | 开放团队ID。如下图所示，查看**ID信息**内的**团队ID**值。 ![[development-add-service-group-members_O1CN017qV2Yi1NQoZjuGx7D_!!6000000001565-2-tps-956-254.png]] |
| openConversationId | String | 是 | 服务群openConversionId，可调用[创建场景服务群](/document/development/create-a-scenario-service-group)接口获取openConversationId参数值。 |
| userIds | Array of String | 是 | 待添加员工在钉钉组织内的的userId列表，最大值100。  **说明**  请确保userId值的正确性，如果userId值不正确，该接口不会报错，添加时会自动忽略该成员。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| success | Boolean | 添加是否成功，true表示成功。 |
