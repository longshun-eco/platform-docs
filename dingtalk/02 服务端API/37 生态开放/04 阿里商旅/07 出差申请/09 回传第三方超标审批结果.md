更新于 2026-01-29通过此接口回传第三方商旅系统中的超标审批结果至阿里商旅与钉钉平台，实现审批状态的同步。

**说明**

第三方超标审批单推送到企业后，企业审批结束，将审批结果回传给阿里商旅。

## 使用场景

适用于接入阿里商旅生态的第三方商旅管理系统，在以下业务流程中使用：

* **典型场景**：企业员工在第三方商旅平台提交超标差旅申请 → 审批流程在钉钉内完成 → 第三方系统接收审批结束事件 → 调用本接口回传最终审批结果。
* **适用企业类型**：已开通阿里商旅服务的企业客户，尤其是使用定制化或独立商旅系统的中大型组织。
* **调用时机**：在钉钉侧审批实例结束后，由第三方系统主动回调本接口，建议在收到审批完成事件通知后立即调用。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 阿里商旅专用权限点 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=alitrip_1.0%23SyncExceedApply) |
| 第三方企业应用 | 支持 | 阿里商旅专用权限点 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=alitrip_1.0%23SyncExceedApply) |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/alitrip/exceedapply/sync?remark=String&applyId=String&corpId=String&thirdpartyFlowId=String&userId=String&status=Integer HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| remark | String | 是 | 审批意见。 |
| applyId | String | 是 | 商旅超标审批单号。 |
| corpId | String | 是 | 企业的corpId。 |
| thirdpartyFlowId | String | 是 | 第三方流程实例ID。 |
| userId | String | 是 | 员工的userid。 |
| status | Integer | 是 | 审批单状态，取值：   * **1**：同意 * **2**：拒绝 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| module | Boolean | 调用是否成功。 |
