更新于 2025-09-23调用本接口查询企业是否已在e签宝完成实名认证。已实名则会返回实名企业名称，未实名则可在应用内提醒用户进行实名。

**说明**

企业完成实名认证后才会生成印章，可在应用内展示给用户关联的e签宝企业名称。

![[development-query-enterprise-information_O1CN01MxF8dQ1PMLzv4GAWd_!!6000000001826-2-tps-2586-832.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | E签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=esign_2.0%23GetCorpInfo) |
| 第三方企业应用 | 支持 | E签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=esign_2.0%23GetCorpInfo) |
| 第三方个人应用 | 暂不支持 | E签宝数据管理权限 | 暂不支持 |

## 请求方法

```
GET /v2.0/esign/corps/infos HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| serviceGroup | String | 否 | 预留参数，此版本无需此参数。 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| isRealName | String | 企业是否实名。 |
| orgRealName | String | 如果已经实名，会返回实名企业名称。 |
