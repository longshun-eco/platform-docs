更新于 2025-09-23企业开启电子签章时，第一步首先需要确保企业已安装e签宝微应用，需要通过该接口查询是否安装，已安装则直接获取授权地址进授权页，未安装则进行展示渠道码进行安装引导。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | e签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=esign_2.0%23GetIsvStatus) |
| 第三方企业应用 | 支持 | e签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=esign_2.0%23GetIsvStatus) |
| 第三方个人应用 | 暂不支持 | e签宝数据管理权限 | 暂不支持 |

## 请求方法

```
GET /v2.0/esign/corps/appStatus HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| serviceGroup | String | 否 | 预留参数，此版本无需此参数。 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| installStatus | String | 企业微应用安装状态，取值：   * **OPEN**：微应用已经安装 * **CLOSE**：微应用停止或卸载 |
| authStatus | String | 企业微应用授权状态，取值：   * **AUTH**：微应用授权 * **UNAUTH**：微应用未授权 |
