更新于 2025-09-23调用本接口获取企业授权的页面地址，企业授权使用钉签开放平台提供的功能。

企业开启电子签章前，需要通过该地址进行授权（授权前需确保已安装e签宝微应用），授权后将正式开启电子签章服务。
同时ISV只可查询已授权的企业在e签宝的相关信息。

**说明**

地址有效期为2小时。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | e签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=esign_2.0%23GetAuthUrl) |
| 第三方企业应用 | 支持 | e签宝数据管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=esign_2.0%23GetAuthUrl) |
| 第三方个人应用 | 暂不支持 | e签宝数据管理权限 | 暂不支持 |

## 请求方法

```
POST /v2.0/esign/auths/urls HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "redirectUrl" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| serviceGroup | String | 否 | 预留参数，此版本无需此参数。 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| redirectUrl | String | 否 | 完成后的重定向地址。  **说明**  授权成功后，在URL中会携带授权结果，其中：   * status：授权状态，取值    + SUCCESS：成功   + FAIL：失败 * taskid：任务ID   例如：http://xxx.com/status=SUCCESS&taskid=AUTH-AB312C2CA |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| taskId | String | 任务ID。 |
| pcUrl | String | PC端授权地址。 |
| mobileUrl | String | 手机授权地址。 |
