更新于 2025-11-14本文介绍了如何调用场景群的群管理操作流程、群成员相关操作流程、群模板机器人信息查询等功能介绍。

## **接入流程简介**

本文档展示了，创建一个企业内部应用，使用场景群相关的API，介绍了场景群的群管理操作流程、群成员相关操作流程、群模板机器人信息查询等功能介绍。：

步骤一：登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击应用开发-企业内部开发，创建[企业内部应用](/document/orgapp/create-orgapp)。

步骤二：单击基础信息 > 应用信息，获取应用AppKey和AppSecret。

步骤三：[添加接口调用权限](/document/orgapp/add-api-permission)。单击“场景群”，申请chat相关接口的管理权限。

步骤四：获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)。调用接口时，通过accessToken鉴权调用者身份。

步骤五：调用群相关API：

1. 根据群模板ID`template_id`，调用服务端API-[创建群](/document/orgapp/create-a-scene-group-v2)接口，创建场景群，获取唯一群ID `open_conversation_id`。

   **说明**

   获取`template_id`参数，详情参见[基础概念](/document/orgapp/session-management-overview)。
2. 群管理操作流程：

   * 服务端API：

     + 根据唯一群ID `open_conversation_id`，调用服务端API-[更新群](/document/orgapp/scene-group-update)接口，更新群信息。
     + 根据唯一群ID `open_conversation_id`，调用服务端API-[查询群信息](/document/orgapp/queries-the-basic-information-of-a-scenario-group)接口，获取群信息。
   * 新版服务端API：

     + 根据唯一群ID `open_conversation_id`，调用新版服务端API-[查询群简要信息](/document/orgapp/query-group-information)接口，获取群相关信息。
     + 根据唯一群ID `open_conversation_id`，调用新版服务端API-[查询群内群模板机器人](/document/orgapp/search-group-scene-template-robot)接口，获取场景群模版机器人信息。
