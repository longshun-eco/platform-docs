更新于 2025-11-14本文介绍了如何调用群模板相关接口实现普通企业内部群升级为群聊会话操作。

## **接入流程简介**

本文档展示了，创建一个企业内部应用，使用群聊会话相关的API，实现群聊会话的创建，更新，查询和群成员的新增，删除，查询等相关功能的流程：

步骤一：登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击应用开发-企业内部开发，创建[企业内部应用](/document/orgapp/create-orgapp)。

步骤二：单击基础信息 > 应用信息，获取应用AppKey和AppSecret。

步骤三：[添加接口调用权限](/document/orgapp/add-api-permission)。单击“场景群”，申请chat相关接口的管理权限。

步骤四：获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)。调用接口时，通过accessToken鉴权调用者身份。

步骤五：调用群相关API：

1. 调用服务端API[创建群会话](/document/orgapp/create-group-session)接口，创建企业内部群接口，获取群会话ID`open_conversation_id`。
2. 创建企业群聊会话群模板，获取群模板ID`template_id`，详情参见[基础概念](/document/orgapp/session-management-overview)。
3. 根据群模板ID `template_id`和`open_conversation_id`，调用服务端API-[启用群模板](/document/orgapp/enable-a-group-template)接口，升级群聊会话。
4. 根据群模板ID `template_id`和`open_conversation_id`，调用服务端API-[停用群模板](/document/orgapp/disable-a-group-template)接口，停用群模板。

## **步骤一：创建企业内部应用**

**说明**

如果已有企业内部应用，可直接使用已有应用，可忽略此步骤。

登录[开发者后台](https://open-dev.dingtalk.com/#/)，创建[企业内部应用](/document/orgapp/create-orgapp)。

应用类型：选择H5微应用。

开发方式：选择企业自主开发。

## **步骤四：获取应用访问凭证accessToken**

**重要**

服务端API差异详情参见[新旧版规范服务端API区别](/document/orgapp/differences-between-server-apis-and-new-server-apis)。

* 服务端API接口SDK下载，详情参见[服务端SDK下载](/document/orgapp/download-the-server-side-sdk)。
* 新版服务端API接口SDK下载，详情参见[新版服务端SDK下载](/document/orgapp/sdk-download)。

以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK下载](/document/orgapp/download-the-server-side-sdk)。

根据步骤二中的AppKey和AppSecret，获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)。

```java
public void getAccessToken() throws ApiException {
        DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/gettoken");
        OapiGettokenRequest req = new OapiGettokenRequest();
        req.setAppkey("dingxxxxxxxxxhgn");
        req.setAppsecret("9G_xxxxxxxxxxxxxxx1JDf0Qq3nexxxxxxxxGIO");
        req.setHttpMethod("GET");
        OapiGettokenResponse rsp = client.execute(req);
        System.out.println(rsp.getBody());
}
```

## **步骤五：调用服务端相关API**

1. 调用服务端API[创建群会话](/document/orgapp/create-group-session)接口，创建企业内部群接口，获取群会话ID`open_conversation_id`。

   ```java
   public void createGroup() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/chat/create");
           OapiChatCreateRequest req = new OapiChatCreateRequest();
           req.setName("企业内部群");
           req.setOwner("mana****675");
           req.setUseridlist(Arrays.asList("08521******272","014728********041"));
           OapiChatCreateResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
2. 创建企业群聊会话群模板，获取群模板ID`template_id`，详情参见[基础概念](/document/orgapp/session-management-overview)。
3. 根据群模板ID `template_id`和`open_conversation_id`，调用服务端API-[启用群模板](/document/orgapp/enable-a-group-template)接口，升级群聊会话。

   **重要**

   调用本接口前，需要在[钉钉开发者后台](https://open-dev.dingtalk.com/#/)群模板设置灰度企业。否则无法启用群模板。

   ![[development-upgrade-the-enterprise-internal-group-to-the-scene-group_p368591.png]]

   ```java
   public void templateOpen() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/im/chat/scenegroup/template/apply");
           OapiImChatScenegroupTemplateApplyRequest req = new OapiImChatScenegroupTemplateApplyRequest();
           req.setOwnerUserId("man*****75");
           req.setTemplateId("77a70847-****-****-****-fdf607082bcf");
           req.setOpenConversationId("cidzfi*******wA==");
           OapiImChatScenegroupTemplateApplyResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
4. 根据群模板ID `template_id`和`open_conversation_id`，调用服务端API-[停用群模板](/document/orgapp/disable-a-group-template)接口，停用群模板。
