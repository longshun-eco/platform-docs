更新于 2025-09-08本文档介绍了机器人发送互动卡片（普通版）。

**重要**

为提升接口使用体验，针对互动卡片接口进行升级，本文档预计于 2024 年 12 月 31 日迁移至历史文档（不推荐）目录，接口不再支持新应用接入, 已接入的应用可继续调用，且本接口仅保持现有功能，不再新增支持其他能力。

* 如果未使用本接口，推荐使用[创建并投放卡片](/document/orgapp/create-and-deliver-cards)接口。
* 如果已使用本接口，建议您根据自身实际情况评估是否切换至推荐接口。

### 接口功能介绍

通过应用机器人发送普通版互动卡片消息，相比于高级版，普通版通过内置卡片模板，减少卡片模板构建过程，以提升开发效率，更多详情参见[互动卡片普通版](https://open.dingtalk.com/document/resourcedownload/ding-card-interactive-card-product-overview)。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=im_1.0%23SendRobotInteractiveCard) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=im_1.0%23SendRobotInteractiveCard) |

## 请求方法

```
POST /v1.0/im/v1.0/robot/interactiveCards/send HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "cardTemplateId" : "String",
  "openConversationId" : "String",
  "singleChatReceiver" : "String",
  "cardBizId" : "String",
  "robotCode" : "String",
  "callbackUrl" : "String",
  "cardData" : "String",
  "userIdPrivateDataMap" : "String",
  "unionIdPrivateDataMap" : "String",
  "sendOptions" : {
    "atUserListJson" : "String",
    "atAll" : Boolean,
    "receiverListJson" : "String",
    "cardPropertyJson" : "String"
  },
```
