## 接入流程简介

步骤一：登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击应用开发-企业内部开发，创建[机器人](https://open.dingtalk.com/document/orgapp/enterprise-created-chatbot)。

步骤二：点击机器人应用-基础信息，获取AppKey和AppSecret。

步骤三：设置开发管理，添加消息接收地址。

步骤四：[添加接口调用权限](https://open.dingtalk.com/document/orgapp/call-robot-api-operations)，点击“机器人”，申请**企业内机器人发送消息权限**。申请权限无需审批，默认开通。

步骤五：上线机器人。进入版本管理与发布页面，点击上线，机器人的状态变更为已发布。

步骤六：调用[批量发送单聊消息](https://open.dingtalk.com/document/orgapp/chatbots-send-one-on-one-chat-messages-in-batches)接口，实现单聊交互查询。

步骤七：搜索对应钉钉机器人，主动发送问题，实现交互响应。

步骤八：下线机器人，进入版本管理与发布页面，点击下线，机器人的状态变更为未发布。

## 步骤一：创建企业内部机器人

登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击应用开发-企业内部开发，创建[机器人](https://open.dingtalk.com/document/orgapp/enterprise-created-chatbot)。

![[development-dingtalk-chatbot-for-one-on-one-query_p385216.png]]

## 步骤二：获取AppKey和AppSecret

获取机器人AppKey和AppSecret。

![[development-dingtalk-chatbot-for-one-on-one-query_p385218.png]]

## 步骤三：设置开发管理

设置开发管理，添加消息接收地址。

**说明**

消息接收地址是指@群机器人时，将消息发送到指定服务端地址。详情请参考文档[接收消息的消息协议](https://open.dingtalk.com/document/orgapp/receive-message-2)。

![[development-dingtalk-chatbot-for-one-on-one-query_p385219.png]]

## 步骤四：添加接口权限

[添加接口调用权限](https://open.dingtalk.com/document/orgapp/call-robot-api-operations)，点击“机器人”，申请**企业内机器人发送消息权限**。申请权限后无需审批，默认开通权限

![[development-dingtalk-chatbot-for-one-on-one-query_p374374.png]]

## 步骤五：发布机器人

上线机器人。进入版本管理与发布页面，点击上线，机器人的状态变更为已发布。

**说明**

点击“调试”所进入的测试群“xxxxxxxxx-TEST”@机器人无法获取当前人员的senderStaffId，即当前人员的userId，请使用以下创建企业内部群的方式测试机器人。

![[development-dingtalk-chatbot-for-one-on-one-query_p385221.png]]

## 步骤六：发送单聊消息

调用服务端API-[批量发送单聊消息](https://open.dingtalk.com/document/orgapp/chatbots-send-one-on-one-chat-messages-in-batches)接口，实现单聊交互查询。

**说明**

* 参考以下示例代码，要求必须列举子项问题所有情况，否则无法实现交互，子项问题例如“发票要普票还是专票”。

  ![[development-dingtalk-chatbot-for-one-on-one-query_p374904.png]]
* actionURL1等地址链接中的content内容要进行UrlEncode且content内容必须与子项问题保持一致。
* robotCode为机器人应用的AppKey。
* dtmd协议只能在markdown、actioncard、feedcard 消息类型中使用。

```java
@RequestMapping(value = "/robots", method = RequestMethod.POST)
    public String helloRobots(@RequestBody(required = false) JSONObject json) throws Exception {
        System.out.println(JSON.toJSONString(json));
        String content = json.getJSONObject("text").getString("content");
        String userId = json.get("senderStaffId").toString();
        System.out.println(content);
        if (content.equals("发票")) {
            sendMessage1(userId);
        } else if (content.equals("发票要普票还是专票")) {
            sendMessage2(userId);
        } else if (content.equals("发票内容开什么")) {
            sendMessage3(userId);
        } else if (content.equals("差旅住宿普通发票可以报销吗")) {
            sendMessage4(userId);
        }
        return null;
    }
```
