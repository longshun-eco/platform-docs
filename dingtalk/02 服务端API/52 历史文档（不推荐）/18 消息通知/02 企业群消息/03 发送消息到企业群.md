更新于 2025-09-08调用本接口发送消息到企业群。

**重要**

为提升接口的使用体验，发送消息到企业群接口计划升级，后续完善更多功能，重新开放时间请关注文档更新日志。

* 发送消息到企业群接口相关文档，已于2022年09月23日迁移至历史文档（不推荐）目录。
* 不再支持新应用接入，已接入的应用可以正常调用。

发送消息到企业群会话是指可以调用接口创建企业群会话，会以**企业名义**向企业群会话里推送消息。目前钉钉群共有5类，发送消息到企业群支持的钉钉群类型如下：

|  |  |
| --- | --- |
| 钉钉群类型 | 是否支持发送系统群消息 |
| 内部群 | 是 |
| 全员群 | 是 |
| 部门群 | 是 |
| 普通群 | 否 |
| 合作群 | 否 |

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 开发者后台申请 | — |
| 第三方企业应用 | 否 | — | — |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/chat/send`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | 6d1bxxxx | 调用服务端API的应用凭证。   * 企业内部应用可通过[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| chatid | String | 是 | chate39f5xxxxxx335 | 群会话的ID。   * 企业内部应用    + 调用服务端API获取，调用[创建群会话](/document/orgapp/create-group-session)接口的返回chatid字段。   + 调用前端API获取，小程序调用[选择会话](/document/orgapp/select-session)，微应用调用[根据corpid选择会话](/document/orgapp/select-session-based-on-corpid) |
| msg | JSON Object | 是 | {"msgtype":"text","text":{"content":"请提交日报。"}} | 消息内容，最长不超过2048个字节。   * 企业内部应用消息类型和样例参考[消息类型与数据格式](/document/orgapp/message-types-and-data-format)。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| errcode | Number | 0 | 返回码。 |
| errmsg | String | ok | 返回码描述。 |
| messageId | String | msgpjzCxxxxxB | 加密的消息ID。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/chat/send?access_token=ACCESS_TOKEN
```

请求正文

```json
{
    "chatid": "chate39f5xxxxxx335",
    "msg": {
        "msgtype": "text",
        "text": {
            "content": "请于本月底提交月度工作报告。"
        }
    }
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/chat/send");
OapiChatSendRequest req = new OapiChatSendRequest();
req.setChatid("chate39f5xxxxxx335");
Msg msg = new Msg();
Text text = new Text();
text.setContent("请于本月底提交月度工作报告。");
msg.setText(text);
msg.setMsgtype("text");
req.setMsg(msg);
OapiChatSendResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
    "errcode": 0,
    "errmsg": "ok",
    "messageId": "msgpjzCxxxxxB"
}
```

## 错误码

|  |  |  |
| --- | --- | --- |
| 错误码（errorcode） | 错误码描述（errmsg） | 解决方案 |
| 43007 | 权限不足 | 请校验access\_token是否正确 |
| 34001 | 无效的chatid | 请校验chatid是否正确 |
| 34002 | 无效的消息发送者 | 请校验sender是否正确 |
| 34004 | 无效的消息类型 | 请校验消息类型是否正确 |
| 34016 | 消息内容太长 | 请缩短消息体长度 |
| 34014 | 无效的消息 | 请校验消息体是否符合json格式，消息体中内容是否按上述入参要求完整填写 |
| 40048 | 含有不安全的链接 | 请检验消息体中URL消息是否合法 |
| 40056 | 无效的agentid | 请确认是否以正确的企业进行token授权 |
| 400001 | 系统错误 | 请稍后再试 |
| -1 | 系统繁忙 | 请稍后再试 |
