更新于 2026-03-06本文是对发消息接口中消息内容的补充描述，开发者可以通过本文了解到在不同情况下如何填写消息内容消息内容，即不同的`contentType`如何填写`content`。

**重要**

本文档已于 2026年 03 月 05 日迁移至历史文档（不推荐）目录，且本文档及相关接口仅保持现有功能，不再新增支持其他能力。

## **AI卡片**

* **contentType** = "ai\_card"

|  |  |  |  |
| --- | --- | --- | --- |
| **content 参数** | | | |
| 名称 | 类型 | 是否必填 | 描述 |
| templateId | String | 是 | 卡片模板 ID。 |
| cardData | String | 否 | 卡片内容，必须是 JSON 格式。 |

### **示例**

#### **示例二：发送包含流式组件的 AI 卡片**

如果你需要发送包含流式组件的 AI 卡片：

1. 你需要填写指定的`cardData`。
2. 你需要填写`options`参数。

|  |  |  |  |
| --- | --- | --- | --- |
| **content 参数** | | | |
| 名称 | 类型 | 是否必填 | 描述 |
| templateId | String | 是 | 卡片模板 ID。 |
| cardData | String | 否 | 卡片内容，必须是 JSON 格式，见下方 |
| options | String | 否 | 卡片附加选项，必须是 JSON 格式，见下方。 |

|  |  |  |  |
| --- | --- | --- | --- |
| **cardData 参数** | | | |
| 名称 | 类型 | 是否必填 | 描述 |
| key | String | 是 | 流式组件的键。 |
| value | String | 是 | 流式组件的内容。 |
| isFinalize | Boolean | 是 | 是否是最后一条流式数据：   * **true**：是 * **false**：否 |
| isFull | Boolean | 否 | 是否覆盖原有内容，默认覆盖：   * **true**：是，直接覆盖 * **false**：否，在原有内容后追加 |

|  |  |  |  |
| --- | --- | --- | --- |
| **options 参数** | | | |
| 名称 | 类型 | 是否必填 | 描述 |
| componentTag | String | 否 | 卡片组件类型：   * **streamingComponent**：流式组件 * **staticComponent**：静态组件 |
| supportForward | Boolean | 否 | 卡片是否支持转发，不填默认不支持。   * **true**：支持 * **false**：不支持 |

[AI 助理更新消息（主动发送模式）](/document/development/the-ai-assistant-updates-active-message-sending-mode)和 [AI 助理发消息（回复消息模式）](/document/development/ai-assistant-messages-reply-mode) 接口均支持包含流式组件的 AI 卡片，下方以 /update 为例。

## **普通卡片（Schema模式）**

**contentType** = "basic\_card\_schema"

|  |  |  |  |
| --- | --- | --- | --- |
| **content 参数** | | | |
| 名称 | 类型 | 是否必填 | 描述 |
| 无 | String | 是 | JSON 格式，可以通过[钉钉卡片搭建平台](https://card.dingtalk.com/card-builder?cardType=normal)搭建。 |
