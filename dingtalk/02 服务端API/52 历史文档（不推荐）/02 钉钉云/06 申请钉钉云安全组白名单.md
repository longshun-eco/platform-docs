更新于 2025-12-18本文档指导开发者如何在钉钉云中申请安全组白名单，确保网络访问符合安全规范。请严格按照流程和填写要求操作，避免因信息不全或不符合标准导致审批失败。

## 步骤一，打开安全组白名单申请页面

登录[开发者后台](http://open-dev.dingtalk.com/)，点击 **钉钉云 > 安全组白名单申请** ，进入安全组白名单申请页面。在当前页面，查看已申请、已处理和已取消的所有安全组白名单申请列表。

**说明**

首次申请前，请提前准备好相关材料，如IP信息、域名备案证明、接口文档等，以提升审批通过率。

![[development-apply-for-dingtalk-security-group-whitelist_p163358.png]]

## 步骤二，填写安全组规则内容

点击右上角“+新增安全组规则” ，弹出新增安全组规则页面，填写相关信息申请安全组白名单。按照字段要求填写相关信息：

**重要**

填写不规范将可能导致系统自动拒绝，请务必准确、完整地提交信息。

* 鼠标悬停于字段右侧的问号图标（?），可查看该字段的详细说明。
* 页面顶部提供《填写说明》链接，或查看本文<安全组规则填写说明>部分内容，建议仔细阅读后再提交。

![[development-apply-for-dingtalk-security-group-whitelist_p163359.png]]

## 步骤三，查看申请状态

点击申请后，可查看当前申请的状态，具体含义如下：

|  |  |
| --- | --- |
| **状态** | **说明** |
| 新申请 | 新申请提交成功后，会显示该状态，可取消或查看详情。 |
| 已取消 | 用户主动取消，不可再操作，可以点击“重新提交”，带入原信息创建新申请（附件除外）。 |
| 已通过 | 审核通过，规则即时生效，可查看详情及审批备注。 |
| 已拒绝 | 审核未通过，作废状态。可通过“查看详情”了解拒绝原因，并修改后重新提交。  **说明**  被拒后请根据反馈认真修正内容，避免重复提交错误信息。 |

## 安全组规则填写说明

### 重要前提

白名单只支持针对IP或IP范围开放域名只做备案使用，域名和IP要对应否则按违规处理。

* **公司名**：您注册钉钉云账户时填写的公司名称。
* **网卡类型**： 钉钉云默认内网，不可修改。

### **1.规则方向**

|  |  |
| --- | --- |
| **规则方向** | **说明** |
| 入方向 | 钉钉云外的服务器访问钉钉云中部署服务。 |
| 出方向 | 从钉钉云ECS访问钉钉云外的服务，例如三方软件源，信息服务，阿里高德地图服务等。 |

### **2.授权策略**

|  |  |
| --- | --- |
| **策略类型** | **说明** |
| 允许访问 | 开放指定协议和端口的通信权限。 |
| 拒绝访问 | 明确禁止某些流量通过（优先级高于“允许”）。 |

### **3.协议类型**

出入方向访问的协议类型，目前数据库相关访问是禁止的，如必须访问请直接联系管理员。

### **4.服务方**

#### 公共三方服务

公共三方服务指由可靠公共第三方服务商提供的服务，如三方短信服务、国家征信查询服务等。如果阿里云可替代服务，建议选用阿里云服务，如无法迁移到阿里云服务需提供**详细充分理由**，否则系统会自动审批拒绝。

#### ISV自建服务

建议将钉钉相关业务和系统迁移到钉钉云内部署，不予开放ISV从钉钉云到公网自建服务的访问。

如需特殊申请，必须要访问ISV自建服务且无法找到替代方案时，可按钉钉安全原则规定，申请开放安全组白名单。ISV服务申请白名单，必须详细描述充分理由，并列出业务场景、接口列表和出入参数说明，以文档附件形式上传，接口访问会由网关统一过滤。**文档填写说明如下，审批通过后，可以开通ISV自建服务安全组白名单，填写不规范会自动拒绝。**

（1）保证敏感数据不会泄露的情况下（包括但不限于企业通讯录，组织架构，微应用使用产生的用户数据）

（2）阿里云，钉钉云无替代方案实现所需开通安全组功能

（3）ISV承诺所提供接口文档属实准确**（钉钉云接口监控如果检测到和备案接口请求不一致，将追究相应责任，后果严重）**

通常只允许运维系统等，设置在钉钉云外，**钉钉业务核心系统，必须放在钉钉云内，**没有充分理由不予批准。

#### 接口文档示例

**格式**：`.doc`、`.txt` 或 `.pdf`

**内容**：除列出详细的接口，参数字段外，还要详述接口字段作用。

**样例**：

```
费用申请接口，用于申请费用
http://www.demodomain.com/service/fee_request
      requestParam:{
userId:xxx //用户ID，由ISV自己生成
time:xxxx,xxx//费用使用时间
data:xxxx // 所需业务数据
} //不可以包含企业数据敏感信息

    response:{xxx,xxx,xxx,xxx} //业务数据回调。
```

### **5.域名**

访问服务对应域名，用作校对，域名前不要加http/https，也做为二方自动审批的依据。

示例：`api.example.com`

❌ 错误写法：`https://api.example.com\`（请勿带协议头）

### **6.端口范围**

填写需要开放访问的具体端口或端口区间，多个项用英文逗号 `,` 分隔。

例如：

* "80" 代表80端口
* "80/80" 代表80端口，即单个端口
* "8080/8085" 代表端口范围 8080～8085
* "80,443,8080" 代表80和443和8080等多个独立端口 (不需要加引号)

### **7.优先级**

取值范围：**20 ~ 80**

* 数值越小，优先级越高（20 最高，80 最低）
* **拒绝规则的优先级高于允许规则**

### **8.授权对象**

填写需要访问的出方向或入方向IP 地址或 IP 段，一次最多10个，使用英文逗号 `,` 分隔。

单个IP：139.43.23.4

IP段：139.43.23.41/30 (表示139.43.23.41开始，IP二进制前30位不变的IP范围)

多个IP或段：139.43.23.4,139.43.2.4/28,23.45.44.3

#### 实际应用场景示例

假设 ISV 需从钉钉云 ECS 访问 `www.xxxx.com`，其解析 IP 为 `58.216.xx.xx` 和 `114.236.xx.xxx`，服务分为 HTTP/HTTPS服务。

则应填写：

* **域名**：`www.xxxx.com`
* **端口范围**：`80,443`
* **授权对象**：`58.216.xx.xx,114.236.xx.xxx`

> 审批通过后，ISV即可从钉钉云服务器访问该网站 （入方向申请同理）。

**注意：**针对所有IP的端口开放用授权对象0.0.0.0/0，端口1/65535，**只有在部署阶段**可以**临时开放**，申请时需注明关闭时间，需要开放的详细充分理由：

|  |  |
| --- | --- |
| **授权对象** | **说明** |
| 出方向授权对象 | 钉钉云内要访问的域名对应的IP列表；若无域名，填写 `no.domain`。 |
| 入方向授权对象 | 希望钉钉云特定端口的IP列表；若无域名，填写 `no.domain`。 |

## 温馨提示

* 所有申请均需经过安全合规审查，请耐心等待。
* 提交前请反复检查：

  + 是否遗漏必填项
  + IP 与域名是否匹配
  + 接口文档是否完整、无敏感信息
* 如遇问题，建议联系技术支持或查阅官方帮助中心获取最新指南。
