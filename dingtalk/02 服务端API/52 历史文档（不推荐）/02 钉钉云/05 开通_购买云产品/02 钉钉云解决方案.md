更新于 2025-12-18钉钉云解决方案是基于钉钉云/阿里云基础设施，为应用提供服务端的一站式云服务解决方案，是钉钉云架构最佳实践的具体实现方案。帮助开发者实现资源自动购买、应用快速创建和基础监控自动化，显著降低接入成本。

## **核心优势**

* **提高效率**

  提供一键式构建方案，大幅降低钉钉应用接入门槛，缩短部署周期。
* **高可用性**

  基于阿里云高可用、可扩展的技术架构设计，支持自有业务快速增长，保障系统稳定性。
* **合规上架推荐**

  **对于需上架到钉钉广场的应用，强烈推荐使用钉钉云解决方案。**钉钉云解决方案已将架构最佳实践整合，符合上架规范，开发者只需一键式构建方案，减少上架过程中的问题。

  **说明**

  若开发者自行购买资源，资源配置等需与解决方案相同，可能会影响上架时效。

### **注意事项**

* **NAT 共享机制**

  NAT为钉钉云账号共用，多个解决方案会共用一套NAT。购买基础版解决方案(带NAT)后，再次购买解决方案不会加入NAT费用，NAT费用请以实际创建解决方案时为准。如果NAT为多解决方案共用，开发者需要考虑NAT绑定EIP带宽问题，必要时自行升级。
* **计费模式说明**

  用户根据规格，选配合适的解决方案，**默认为按量收费**，可在解决方案详情页“转包年包月”。
* **扩容与变配规则**

  支持对现有解决方案进行**扩容**（增加 ECS 实例数量）或**升配**（提升 ECS/RDS 规格），但**不支持降配或删除资源**。例如购买解决方案之后，通过钉钉云解决方案扩容，或在阿里云控制台购买OSS，Redis等。
* **版本升级限制**

  解决方案不支持直接升级（例如基础版升级为高级版，需要手动迁移），可根据自己需求购买合适的解决方案后进行升级配置和扩容来提升服务性能。**其中体验版不支持扩容，支持升配。**

## 购买解决方案

钉钉云账户可以购买多个解决方案，建议每个应用购买一个解决方案。购买前请确认账户余额 **大于 100 元**（最低消费门槛，低于此金额无法下单）。

更多财务操作（充值、退款等）详见：[充值/退款等财务问题](/document/isvapp/financial-problems-such-as-recharge-refund)。

### **步骤一，登录解决方案控制台**

**未注册用户**：请先申请钉钉云账号，操作指引见 [申请钉钉云账号](/document/isvapp/apply-for-a-dingtalk-cloud-account-1)。

**已有账号**：

1. 进入[开发者后台](https://open-dev.dingtalk.com/#/cloudHome) > **钉钉云** > **基础信息**，点击**登录控制台**。

   ![[development-dingtalk-cloud-solution_p163335.png]]
2. 输入用户名密码，登录控制台。

   ![[development-dingtalk-cloud-solution_p163336.png]]
3. 登录后，进入钉钉云解决方案首页，在解决方案首页可以创建、管理钉钉云解决方案，查看解决方案的基础监控。

   ![[development-dingtalk-cloud-solution_p163337.png]]

### **步骤二，创建解决方案**

1. 点击解决方案页面右上角“创建新方案”按钮，创建解决方案。

   * 输入解决方案名称：建议和小程序名称一致，方便区分。

     ![[development-dingtalk-cloud-solution_p163338.png]]
   * 选择运行环境：如 Java、PHP、Node.js、Python 或纯净 CentOS 7.x。

     ![[development-dingtalk-cloud-solution_p163339.png]]
   * 选择“资源规格”。

     **说明**

     购买默认为按量付费（按使用天数及流量计费），页面显示为包年价格。

     ![[development-dingtalk-cloud-solution_p163340.png]]
2. 选择规格后，点击**立即购买**，并完成付款流程。系统开始自动创建钉钉云解决方案，如下图所示：

   ![[development-dingtalk-cloud-solution_p163341.png]]
3. 约等待 **10 分钟**，刷新列表，当状态变为 **“运行中”** 表示创建成功。

   ![[development-dingtalk-cloud-solution_p163342.png]]
4. 点击“详情”，“参数信息”一栏，Public Address 处，点击公网Ip进行验证，页面显示Demo正常即为创建成功。

   ![[development-dingtalk-cloud-solution_p163343.png]]

**重要**

在钉钉云解决方案中，仍然保持钉钉云所有安全相关的限制（安全组限制，资源释放限制避免意外释放导致事故等）。如购买错误，请通过[技术答疑](https://ding-doc.dingtalk.com/document/contactus?pnamespace=app)联系管理员释放资源。

### **步骤三，解决方案运行环境**

当前支持运行环境包括（可根据服务自行选择）：

* 纯净 CentOS 7.x
* Java
* PHP
* NodeJS
* Python

**Windows用户注意**：若需部署 IIS 或其他 Windows 服务，可选择“纯净 CentOS”版本，在 ECS 创建完成后停止实例，更换操作系统为 Windows Server，并自行安装所需软件完成部署。

## 解决方案管理

购买解决方案后，进入解决方案首页，跳转到解决方案管理页面，可以进行解决方案的管理。开发者常使用解决方案管理主要有：扩容升配、转包年包月、资源监控等。

### **1.扩容升配**

如果在使用过程中随流量增加，需要更高的配置或更多的资源，可在解决方案详情页面进行扩容，或直接在[云产品控制台](https://home.console.aliyun.com/new#/)对应资源项目中变更配置（详情页面资源列表可以找到解决方案对应资源，进行升配）。操作详情查看钉钉云解决方案。

### **2.转包年包月**

将按量付费转换为包年包月模式，可获得更低单价，适合长期稳定运行的应用。

操作步骤：进入解决方案详情页 → 右上角点击“转包年包月” → 选择购买时长（≥6个月）→ 完成支付。

更多详情请参阅：[转包年包月](#MkIYa)。

## 常见问题

### Q1：一个解决方案能否支持多个小程序？

不推荐。建议 **一个小程序对应一个解决方案**，以实现资源隔离、权限管控和独立监控。多个应用混用可能导致故障扩散或管理混乱。

### Q2：解决方案与小程序/微应用的关系

* **钉钉小程序**

  [钉钉小程序](/document/isvapp/introduction-to-dingtalk-mini-programs)服务端需要部署在钉钉云解决方案中，建议一个小程序部署在一个解决方案中。解决方案中所创建的应用为Service应用。Service应用对应服务化概念中的Service，可以有单个或多个Service应用构成你的小程序产品后端。对于小程序来说，至少要有一个外部Service应用以提供对外访问的入口。
* **微应用**

  微应用的前端及服务端均需部署在同一解决方案中，建议一个微应用部署在一个解决方案中，Service应用（解决方案中所创建的应用）可以是微应用的前端、服务端。
