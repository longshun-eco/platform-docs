更新于 2025-09-08本教程介绍了如何通过官方OA审批API实现发起带附件的审批流，及下载附件。

**说明**

* 本文档以企业内部应用为例，第三方企业应用实现流程类似。
* 仅调用服务端API无法实现审批附件的操作流程，必须与客户端JSAPI结合使用。

## 接入流程简介

本文档展示了，创建一个企业内部应用，实现使用**官方**OA审批发起带附件的审批、下载审批附件等流程：

步骤一：登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击**应用开发-企业内部开发**，创建[企业内部应用](/document/app/create-orgapp#topic-2024342)。

步骤二：单击**基础信息 > 应用信息**，获取应用AppKey和AppSecret。

步骤三：[添加接口调用权限](/document/orgapp/add-api-permission)。单击“OA审批”，申请OA审批对应的权限。

步骤四：获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)。调用接口时，通过accessToken鉴权调用者身份。

步骤五：调用OA审批相关API和客户端JSAPI：

1. 调用服务端API-[获取审批钉盘空间信息](/document/orgapp/query-the-space-of-an-approval-nail)接口，获取审批钉盘空间`space_id`。
2. 调用客户端JSAPI-[上传附件到钉盘/从钉盘选择文件](/document/orgapp/upload-attachment-to-nail-plate-select-file-from-nail-plate-h5)接口，获取文件基本信息。
3. 获取模板的唯一编码`process_code`。

   * 通过[钉钉管理后台](https://aflow.dingtalk.com/dingtalk/web/query/dashboard?dinghash=aflowSetting#/aflowSetting)-OA审批-打开对应审批模版获取。
   * 调用服务端API-[创建或更新审批模板](/document/orgapp/create-or-update-approval-templates)接口，创建审批模板获取`process_code`。
4. 调用服务端API-[发起审批实例](/document/orgapp/initiate-approval)接口发起审批，获取审批实例`process_instance_id`。
5. 根据审批实例`process_instance_id`，调用服务端API-[获取单个审批实例详情](/document/orgapp/obtains-the-details-of-a-single-approval-instance)接口获取审批实例详情中的`file_id`。
6. 调用服务端API-[下载审批附件](/document/orgapp/grants-the-permission-to-download-the-approval-file)接口，获取文件的链接`download_uri`实现下载。目前不支持第三方企业应用调用。

## 步骤四：获取应用访问凭证accessToken

**重要**

服务端API差异详情参见[新旧版规范服务端API区别](/document/orgapp/differences-between-server-apis-and-new-server-apis)。

* 服务端API接口SDK下载，详情参见[服务端SDK下载](/document/orgapp/download-the-server-side-sdk)。
* 新版服务端API接口SDK下载，详情参见[新版服务端SDK下载](/document/orgapp/sdk-download)。

以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK](/document/orgapp/download-the-server-side-sdk)。

根据步骤二中的AppKey和AppSecret，获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp/obtain-orgapp-token)。

```java
public void getAccessToken() throws ApiException {
        DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/gettoken");
        OapiGettokenRequest req = new OapiGettokenRequest();
        req.setAppkey("dingxxxxxxxxxhgn");
        req.setAppsecret("9G_xxxxxxxxxxxxxxx1JDf0Qq3nexxxxxxxxGIO");
        req.setHttpMethod("GET");
        OapiGettokenResponse rsp = client.execute(req);
        System.out.println(rsp.getBody());
    }
```

## 步骤五：调用OA审批相关API

调用OA审批相关API和客户端JSAPI：

1. 调用服务端API-[获取审批钉盘空间信息](/document/orgapp/query-the-space-of-an-approval-nail)接口，获取审批钉盘空间`space_id`。

   ```java
    public void ProcessSpaceId() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/cspace/info");
           OapiProcessinstanceCspaceInfoRequest req = new OapiProcessinstanceCspaceInfoRequest();
           req.setUserId("manager7675");
           OapiProcessinstanceCspaceInfoResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
2. 调用客户端JSAPI-[上传附件到钉盘/从钉盘选择文件](/document/orgapp/upload-attachment-to-nail-plate-select-file-from-nail-plate-h5)接口，获取文件基本信息，本流程示例使用[JSAPI Explorer](https://open-dev.dingtalk.com/apiExplorer#/jsapi?api=biz.util.uploadAttachment)实现。

   **重要**

   调用该方法前，需要调用[获取审批钉盘空间信息](/document/orgapp-server/query-the-space-of-an-approval-nail#topic-1936895)进行授权操作。

   ![[development-tutorial-initiate-an-approval-task-with-attachments_p478641.png]]
3. 获取模板的唯一编码`process_code`。有2种方式可以获取processCode，可以选择以下任一种：

   * 方式一，通过[钉钉管理后台](https://aflow.dingtalk.com/dingtalk/web/query/dashboard?dinghash=aflowSetting#/aflowSetting)-OA审批-打开对应审批模版获取。

     **说明**

     钉钉管理后台版本不同，获取processCode的方式不同。登录钉钉管理后台，在首页查看版本。如下图所示，页面展示**回到旧版**和**新版反馈**，说明当前是新版。

     ![[development-tutorial-initiate-an-approval-task-with-attachments_p381720.png]]

     **新版钉钉管理后台**：在审批模板编辑页-基础设置-**页面底部**查看。

     ![[development-tutorial-initiate-an-approval-task-with-attachments_p443734.png]]

     **旧版钉钉管理后台**：在审批模板编辑页的URL中查看。

     ![[development-tutorial-initiate-an-approval-task-with-attachments_p344894.png]]
   * 方式二：调用服务端API-[创建或更新审批模板](/document/orgapp/create-or-update-approval-templates)接口，创建审批模板获取`process_code`。

     ```java
     public void createTemplate() throws ApiException {
             DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/process/save");
             OapiProcessSaveRequest request = new OapiProcessSaveRequest();
             OapiProcessSaveRequest.SaveProcessRequest saveProcessRequest = new OapiProcessSaveRequest.SaveProcessRequest();
             saveProcessRequest.setName("创建附件的审批模板");
             //有参数processCode时为更新模板，无参数processCode时，为创建模板，该示例为创建模板
             //saveProcessRequest.setProcessCode("PROC-37801FB3-xxxx-xxxx-xxxx-6C6ADB87CD9A");
             saveProcessRequest.setAgentid(118******5L);
             saveProcessRequest.setDescription("创建附件的审批模板");

             // 注意，每种表单组件，对应的componentName是固定的，参照以下示例代码
             List formComponentList = new ArrayList<>();

             // 附件
             OapiProcessSaveRequest.FormComponentVo attachmentComponent = new OapiProcessSaveRequest.FormComponentVo();
             attachmentComponent.setComponentName("DDAttachment");
             OapiProcessSaveRequest.FormComponentPropVo attachmentComponentProp = new OapiProcessSaveRequest.FormComponentPropVo();
             attachmentComponentProp.setRequired(true);
             attachmentComponentProp.setLabel("附件");
             attachmentComponentProp.setId("DDAttachment-J78F0572");
             attachmentComponent.setProps(attachmentComponentProp);
     ```
4. 调用服务端API-[发起审批实例](/document/orgapp/initiate-approval)接口发起审批，获取审批实例`process_instance_id`。

   ```java
    public void createProcessInstance() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/create");
           OapiProcessinstanceCreateRequest req = new OapiProcessinstanceCreateRequest();
           //应用标识，可在开发者后台的应用详情页获取。
           req.setAgentId(1151185599675L);
           req.setProcessCode("PROC-84B33D09-****-****-****-F74456C1DB55");
           //审批实例发起人的userid。
           req.setOriginatorUserId("01472******877041");
           //发起人所在的部门，如果发起人属于根部门，传-1。
           req.setDeptId(-1L);
           //抄送人userid列表，最大列表长度20。
           req.setCcList("mana****675,085218********272");
           //在审批结束后发送抄送
           req.setCcPosition("FINISH");

           List formComponentValueVoList = new ArrayList();

           //上传审批附件
           OapiProcessinstanceCreateRequest.FormComponentValueVo formComponentValueVo = new OapiProcessinstanceCreateRequest.FormComponentValueVo();
           formComponentValueVoList.add(formComponentValueVo);
   ```
5. 根据审批实例`process_instance_id`，调用服务端API-[获取单个审批实例详情](/document/orgapp/obtains-the-details-of-a-single-approval-instance)接口获取审批实例详情中的`file_id`。

   ```java
    public void processInstanceInfo() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/get");
           OapiProcessinstanceGetRequest req = new OapiProcessinstanceGetRequest();
           //审批实例process_instance_id
           req.setProcessInstanceId("Va5qa5RaQ***************0791502");
           OapiProcessinstanceGetResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
6. 调用服务端API-[下载审批附件](/document/orgapp/grants-the-permission-to-download-the-approval-file)接口，获取文件的链接download\_uri实现下载。目前不支持第三方企业应用调用。

   ```java
   public void download() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/file/url/get");
           OapiProcessinstanceFileUrlGetRequest req = new OapiProcessinstanceFileUrlGetRequest();
           OapiProcessinstanceFileUrlGetRequest.GrantCspaceRequest obj1 = new OapiProcessinstanceFileUrlGetRequest.GrantCspaceRequest();
           obj1.setProcessInstanceId("Va5qa5RaQ***************0791502");
           obj1.setFileId("662*****12");
           req.setRequest(obj1);
           OapiProcessinstanceFileUrlGetResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
