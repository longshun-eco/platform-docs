## 接入流程简介

本文档展示了，创建一个企业内部应用，使用官方OA审批提供的API，实现发起审批、撤销审批等流程：

步骤一：登录[开发者后台](https://open-dev.dingtalk.com/#/)，点击应用开发-企业内部开发，创建[企业内部应用](/document/app/create-orgapp#topic-2024342)。

步骤二：单击基础信息 > 应用信息，获取应用AppKey和AppSecret。

步骤三：[添加接口调用权限](/document/orgapp-server/add-api-permission#topic-2021642)。单击“OA审批”，申请OA审批对应的权限。

步骤四：获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp-server/obtain-orgapp-token#topic-1936350)。调用接口时，通过accessToken鉴权调用者身份。

步骤五：调用OA审批相关API：

1. 调用服务端API-[创建或更新审批模板](/document/orgapp-server/create-or-update-approval-templates#topic-2022852)接口，获取模板的唯一编码`process_code`。
2. 根据模板编码`process_code`，调用服务端API-[发起审批实例](/document/orgapp-server/initiate-approval#topic-1936894)接口发起审批实例，获取审批实例`process_instance_id`。
3. 根据审批实例`process_instance_id`，调用服务端API-[获取单个审批实例详情](/document/orgapp-server/obtains-the-details-of-a-single-approval-instance#topic-1936897)接口，获取审批实例详情。
4. 查看审批后，可根据审批实例`process_instance_id`，调用服务端API-[撤销审批实例](/document/orgapp-server/terminate-a-workflow-by-using-an-instance-id#topic-1955705)，实现审批单的撤销操作。

## 步骤四：获取应用访问凭证accessToken

**重要**

服务端API差异详情参见[新旧版规范服务端API区别](/document/orgapp-server/differences-between-server-apis-and-new-server-apis#topic-2148151)。

* 服务端API接口SDK下载，详情参见[服务端SDK下载](/document/orgapp-server/download-the-server-side-sdk#topic-1950896)。
* 新版服务端API接口SDK下载，详情参见[新版服务端SDK下载](/document/orgapp-server/sdk-download#topic-2045226)。

以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK](/document/orgapp-server/download-the-server-side-sdk#topic-1950896)。

根据步骤二中的AppKey和AppSecret，获取应用访问凭证[获取企业内部应用的access\_token](/document/orgapp-server/obtain-orgapp-token#topic-1936350)。

```java
public void getAccessToken() throws ApiException {
        DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/gettoken");
        OapiGettokenRequest req = new OapiGettokenRequest();
        req.setAppkey("dingxxxxxxxxxhgn");
        req.setAppsecret("9G_xxxxxxxxxxxxxxx1JDf0Qq3nexxxxxxxxGIO");
        req.setHttpMethod("GET");
        OapiGettokenResponse rsp = client.execute(req);
        System.out.println(rsp.getBody());
    }
```

## 步骤五：调用服务端OA相关API

1. 调用服务端API-[创建或更新审批模板](/document/orgapp-server/create-or-update-approval-templates#topic-2022852)接口，获取模板的唯一编码`process_code`。

2. 根据模板编码`process_code`，调用服务端API-[发起审批实例](/document/orgapp-server/initiate-approval#topic-1936894)接口发起审批实例，获取审批实例`process_instance_id`。

   ```java
   public void createProcessInstance() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/create");
           OapiProcessinstanceCreateRequest req = new OapiProcessinstanceCreateRequest();
           //应用标识，可在开发者后台的应用详情页获取。
           req.setAgentId(115xxxxx675L);
           req.setProcessCode("PROC-6xxxxxx-Bxxx-4xxxx-Bxxxx-CD2xxxxxB28C");
           //审批实例发起人的userid。
           req.setOriginatorUserId("01472xxxxxxxx77041");
           //发起人所在的部门，如果发起人属于根部门，传-1。
           req.setDeptId(-1L);
           //抄送人userid列表，最大列表长度20。
           req.setCcList("manxxxxx75,08521xxxxxx84272");
           //在审批结束后发送抄送
           req.setCcPosition("FINISH");

           //单行输入框
           List formComponentValueVoList = new ArrayList();
           OapiProcessinstanceCreateRequest.FormComponentValueVo formComponentValueVo = new OapiProcessinstanceCreateRequest.FormComponentValueVo();
           formComponentValueVoList.add(formComponentValueVo);
           formComponentValueVo.setName("单行输入框");
           formComponentValueVo.setValue("测试单行输入框");
   ```
3. 根据审批实例`process_instance_id`，调用服务端API-[获取单个审批实例详情](/document/orgapp-server/obtains-the-details-of-a-single-approval-instance#topic-1936897)接口，获取审批实例详情。

   ```java
    public void processInstanceInfo() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/processinstance/get");
           OapiProcessinstanceGetRequest req = new OapiProcessinstanceGetRequest();
           //审批实例process_instance_id
           req.setProcessInstanceId("T-0rxjjHS-ixxxxxxI4_A08811660537111");
           OapiProcessinstanceGetResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
4. 查看审批后，可根据审批实例`process_instance_id`，调用服务端API-[撤销审批实例](/document/orgapp-server/terminate-a-workflow-by-using-an-instance-id#topic-1955705)，实现审批单的撤销操作。

   ```java
    public void terminateProcessInstance() throws ApiException {
           DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/process/instance/terminate");
           OapiProcessInstanceTerminateRequest req = new OapiProcessInstanceTerminateRequest();
           OapiProcessInstanceTerminateRequest.TerminateProcessInstanceRequestV2 processInstanceRequestV2 = new OapiProcessInstanceTerminateRequest.TerminateProcessInstanceRequestV2();
           processInstanceRequestV2.setProcessInstanceId("T-0rxjjHS-ixxxxxxI4_A08811660537111");
           //是否通过系统操作：true,由系统直接终止;false,由指定的操作者终止
           //本示例采用由指定的操作者终止
           processInstanceRequestV2.setIsSystem(false);
           processInstanceRequestV2.setRemark("审批单提交错误，进行撤回");
           //指定操作人的userid
           processInstanceRequestV2.setOperatingUserid("01472xxxxxxxx77041");
           req.setRequest(processInstanceRequestV2);
           OapiProcessInstanceTerminateResponse rsp = client.execute(req, "access_token");
           System.out.println(rsp.getBody());
       }
   ```
