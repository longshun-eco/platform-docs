更新于 2025-10-09用于给某个用户批量新增可信设备。

例如，登录**企业OA管理后台**，在**专属钉钉** > **高级专属** > **可信设备**页面，设备列表如下图所示：

![[development-create-multiple-trusted-devices_O1CN01PIsAMK1HSIB0oBN53_!!6000000000756-2-tps-2832-1356.png]]

调用本接口给员工小钉1添加两个Mac端的可信设备。调用效果如下图所示，设备列表新增了两条设备记录。

![[development-create-multiple-trusted-devices_O1CN01LivdtN1UOi5Or8cMG_!!6000000002508-2-tps-2844-1338.png]]

## 权限

要调用此API，需要以下权限之一。

| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| --- | --- | --- | --- |
| 企业内部应用 | 支持 | 专属钉钉可信设备信息读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=exclusive_1.0%23CreateTrustedDeviceBatch) |
| 第三方企业应用 | 支持 | 专属钉钉可信设备信息读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=exclusive_1.0%23CreateTrustedDeviceBatch) |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/exclusive/trustedDevices/batchAdd HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "staffId" : "String",
  "platform" : "String",
  "macAddressList" : [ "String" ]
}
```

## Header参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：  企业内部应用，调用[获取企业内部应用的accessToken](https://open.dingtalk.com/document/development/obtain-the-access-token-of-an-internal-app)接口获取。  第三方企业应用，调用[获取第三方应用授权企业的accessToken](https://open.dingtalk.com/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| userId | String | 是 | 员工userid，为0时表示这个设备为公共设备 |
| platform | String | 是 | 操作端。   * Mac端 * Win端 |
| macAddressList | Array of String | 否 | 设备的Mac地址。 |
| detailList | Array | 否 | 设备列表，如果需要设置mac地址以外的设备信息，优先使用detailList，该列表与macAddressList至少有一项不为空。 |
| title | String | 否 | 设备标题。 |
| macAddress | String | 否 | 设备的Mac地址。 |
| serialNumber | String | 否 | 设备序列号。 |

## 返回参数

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| result | Boolean | 处理结果。   * true：成功 * false：失败 |
