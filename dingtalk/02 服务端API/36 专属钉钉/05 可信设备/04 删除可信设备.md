更新于 2025-09-23仅开通专属钉钉功能-可信设备的企业可以调用本接口，通过mac地址删除可信设备。

例如，员工小钉的可信设备如下图所示。

调用本接口，删除其中一个可信设备，效果如下图所示。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 专属钉钉可信设备信息读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=exclusive_1.0%23DeleteTrustedDevice) |
| 第三方企业应用 | 支持 | 专属钉钉可信设备信息读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=exclusive_1.0%23DeleteTrustedDevice) |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/exclusive/trustedDevices/remove HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "userId" : "String",
  "macAddress" : "String",
  "did" : "String",
  "kickOff" : Boolean
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 设备所属的用户userId，请参考[基础概念--UserId](/document/development/basic-concepts-beta#section-aa9-zk1-os6)。 |
| macAddress | String | 否 | 可信设备的mac地址，可调用[查询可信设备详细信息](/document/development/query-trusted-device-details)接口获取macAddress。 |
| kickOff | Boolean | 是 | 是否踢下线：   * true：是 * false：否   **说明**  该参数如果传true，该设备上登录的专属账号会全部下线，包括其他设备上登录的该账号，也会下线。 |
| id | Long | 否 | 设备编号。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| success | Boolean | 删除是否成功，true表示成功。 |

## 错误码

|  |  |  |  |
| --- | --- | --- | --- |
| HttpCode | 错误码 | 错误信息 | 说明 |
| 400 | param.format.error | 参数格式错误，请检查Mac地址 | 一般为mac地址格式问题 |
| 400 | param.missing | 参数不合法，Mac地址和did必填一项 | Mac地址和did均缺失时报错 |
| 500 | service.error | 系统内部调用错误 | 一些内部意外错误，比如调用超时 |
