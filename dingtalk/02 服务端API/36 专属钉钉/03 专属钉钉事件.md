更新于 2025-10-10本文介绍了专属钉钉的事件类型、数据推送、以及数据解密后参数的详细说明。

## 事件类型

专属钉钉包含如下事件类型：

| 事件类型 | 说明 |
| --- | --- |
| org\_file\_send\_check | 企业员工发送文件的检测事件。 |

## 数据推送

**数据解密后示例：**

```json
{
  "CorpId": "dingxxx",
  "fileRecords": [
    {
      "openCid": "cidhoEzLAXnkhNZYlHg6PqcTzSlxxxdsfs",
      "fileName": "测试",
      "ossKey": "lA6h5dW5kaXNrMATOIQUokAXNB9IGAAxxx",
      "requestId": "dlpId4c5219a39xxx",
      "userId": "304235493xxxx"
    }
  ],
  "EventType": "org_file_send_check",
  "EventTime": 1648103877212
}
```

**参数说明**：

| 参数 | 说明 |
| --- | --- |
| CorpId | 事件所属的组织corpId。 |
| EventType | 事件类型：   * **org\_file\_send\_check**：企业员工发送文件的检测事件。 |
| EventTime | 事件发生时间。 |
| fileRecords | 发送文件记录。 |

**fileRecords数据**：

| 参数 | 说明 |
| --- | --- |
| openCid | 发送文件时的会话id。 |
| fileName | 文件名。 |
| ossKey | 文件在专属存储上的ossKey。 |
| requestId | 发送该文件的请求id。 |
| userId | 发送文件的企业员工userId。 |
