更新于 2026-01-22本文档旨在帮助开发者全面了解钉钉开放平台支持的四大核心应用能力：**小程序**、**网页应用**、**酷应用**和**机器人**。这些能力覆盖了从轻量级工具到复杂业务系统集成的多种场景，助力企业实现高效协同与数字化升级。

## **小程序**

钉钉小程序是一种运行在钉钉客户端内的轻量级应用，具备接近原生应用的性能体验，广泛应用于协同办公、人力资源管理、审批流程、客户关系管理等企业服务场景。

通过钉钉提供的开发框架，开发者可使用类似 React Native 的语法结构，编写一次即可在 iOS、Android、Windows 和 Mac 多端统一运行，显著降低跨平台开发成本。

### 适用对象

|  |  |
| --- | --- |
| **应用类型** | **是否支持** |
| 企业内部应用 | 是 |
| 第三方企业应用 | 是 |
| 第三方个人应用 | 是 |

### 权限说明

* 普通成员：可使用已发布的小程序功能。
* 管理员：可配置应用权限、发布版本、设置可见范围。
* 开发者：需拥有“应用开发”权限，可在“基础信息 > 凭证与基础信息”中获取`Client ID`与`Client Secret`。

### 核心优势

* **高性能渲染**

  采用双线程模型与原生组件桥接机制，页面切换流畅，响应迅速。
* **多端一致体验**

  统一的 UI 组件库保障各终端视觉与交互一致性。
* **离线可用性**

  支持资源本地缓存，弱网或断网环境下仍能访问部分内容。

### 开发与调试支持

* 提供专属的**钉钉开发者工具**，支持项目创建、模拟器预览、真机调试等功能。
* 支持 NPM 包引入与模块化代码组织，提升工程化水平。
* 可调用丰富的 JSAPI，如扫码、地理位置、音视频播放等。

## **网页应用**

网页应用（又称 H5 微应用）是基于标准 Web 技术开发的应用形式，可通过浏览器内核在钉钉中运行，适用于快速上线、轻量级功能集成的场景。

尽管运行环境为 Web 容器，但钉钉提供了强大的 JSAPI 接口支持，使网页应用能够调用设备能力（如摄像头、GPS）以及钉钉特有功能（如分享、钉盘上传、联系人选择器），从而获得接近原生的用户体验。

### 适用对象

|  |  |
| --- | --- |
| **应用类型** | **是否支持** |
| 企业内部应用 | 是 |
| 第三方企业应用 | 是 |
| 第三方个人应用 | 否 |

### 权限说明

* 普通成员：可访问已授权的网页应用页面。
* 管理员：可配置访问权限、设置安全域名（需在“安全管理 > 白名单设置”中添加）。
* 开发者：需拥有“应用开发”权限，可在“基础信息 > 凭证与基础信息”中获取`Client ID`与`Client Secret`。

### 主要特点

* **开发门槛低**：可复用现有 Web 工程技术栈（Vue、React 等），快速迁移已有系统。
* **灵活集成**：适用于单页面或多页面结构，易于与外部系统对接。
* **支持 NPM 与 UI 框架**：可自由引入第三方库和开源 UI 组件。
* **调试方式多样**：UI 层可在 PC 端调试，涉及 JSAPI 功能需通过真机测试验证。

### 小程序与网页应用对比

小程序应用和网页应用区别如下：

|  |  |  |
| --- | --- | --- |
| **对比项** | **小程序** | **H5微应用** |
| **支持的端** | 支持移动端和PC端   * Android * iOS * Windows * Mac | 支持移动端和PC端   * Android * iOS * Windows * Mac |
| **加载性能** | 首次使用略慢，后续加载快（资源缓存于本地） | 受网络环境影响大，未优化时易出现延迟 |
| **使用性能** | 流畅顺滑，接近 Native 体验 | 存在卡顿可能，尤其复杂动画或大量 DOM 操作 |
| **页面跳转/切换** | 与原生效果一致，支持过渡动画 | 切换体验依赖 WebView 性能，难以完全还原 Native 效果 |
| **开发环境搭建** | 提供钉钉开发者工具，快速创建项目 | 提供钉钉开发者工具，快速创建项目 |
| **调试** | 提供钉钉开发者工具，可以在电脑上调试大部分功能 | 仅支持 UI 调试；涉及 JSAPI 必须真机调试 |
| **使用开源 UI 组件** | 目前不支持 | 支持任意 Web UI 框架（如 Ant Design、Element Plus） |
| **使用NPM包** | 支持 | 支持 |
| **模块化组织代码** | 支持小程序特有的模块化 | 使用Vue、React等框架可以获得模块化支持 |
| **灰度发布** | 钉钉平台提供原生支持 | 需开发者自行设计与实现 |
| **CDN** | 小程序包默认在CDN | 需开发者自行购买和配置 CDN 服务 |
| **开发个人应用** | 支持 | 不支持 |
| **应用离线化** | 支持 | 支持 |

### 注意事项

* 页面加载性能受网络影响较大，建议对静态资源进行压缩与缓存优化。
* 不支持平台级灰度发布，需自行实现发布策略。
* 需开发者采购并配置 CDN 服务以保障访问速度。

## **酷应用**

酷应用是以“卡片+交互”为核心的新型应用形态，专为提升群聊、文档、音视频会议等协作场景中的信息透出效率而设计。

它将传统应用的功能解构为动态卡片，在用户沟通过程中实时呈现关键数据与操作入口。例如，在群聊中直接展示审批进度卡片，并允许成员一键同意或驳回，无需跳转至独立应用界面。

### 适用对象

|  |  |
| --- | --- |
| **应用类型** | **是否支持** |
| 企业内部应用 | 是 |
| 第三方企业应用 | 是 |
| 第三方个人应用 | 否 |

### 权限说明

* 普通成员：可查看和操作收到的卡片内容。
* 管理员：可配置卡片推送规则、管理事件订阅。
* 开发者：在“应用能力”中添加“酷应用”能力。

### 典型应用场景

* **群内任务协作**：展示待办任务列表，支持点击完成。
* **审批流程推进**：发送审批请求卡片，收件人可在聊天中直接处理。
* **订单状态同步**：订单更新后自动推送状态卡片，相关人员即时可见。
* **会议议程互动**：在音视频会议中插入议题卡片，参会者可投票或填写反馈。

### 核心特性

* **浅透出设计**：功能以卡片形式自然融入沟通流，减少上下文切换。
* **强交互能力**：支持按钮点击、表单提交、状态更新等实时交互。
* **多场景融合**：不仅限于群聊，还可嵌入文档、日历、待办等空间。
* **事件驱动更新**：通过订阅机制监听业务变化，自动刷新卡片内容。

## **机器人**

机器人是一项独立的应用能力，可用于开箱即用的消息通知，也可通过二次开发实现智能化协作流程，无需绑定特定应用或群组即可快速部署。

通过简单的配置或 API 调用，机器人可将业务系统的事件（如告警、审批提醒、订单变更）主动推送到钉钉群聊或个人会话中，并支持接收用户回复，实现双向交互。

### 适用对象

|  |  |
| --- | --- |
| 应用类型 | 是否支持 |
| 企业内部应用 | 是 |
| 第三方企业应用 | 是 |
| 个人应用 | 否 |

### 权限说明

* 普通成员：可接收机器人消息，参与交互。
* 群管理员：可添加/删除机器人，管理 Webhook 安全设置。
* 开发者：需在群设置中添加机器人并获取 Webhook 地址。

### 主要类型

* **自定义机器人**：通过 Webhook 地址向指定群发送消息，支持文本、富文本、Markdown、卡片等多种消息格式。
* **群助手机器人**：具备接收消息与响应能力，可实现问答、菜单选择、表单收集等交互逻辑。

### 典型用途

* **系统告警通知**：监控系统异常时自动发送告警消息至运维群。
* **每日日报汇总**：定时收集成员日报并整理成卡片发送至管理群。
* **审批提醒**：当有新审批待处理时，机器人自动私信提醒负责人。
* **任务分配与追踪**：通过机器人下发任务，成员确认后自动记录状态。
