更新于 2026-01-22在开始开发前，建议您先了解钉钉开放平台的核心基础概念。本文档旨在帮助开发者快速掌握关键术语的定义、用途及获取方式，适用于企业内部应用、第三方企业应用以及机器人、AI助理等各类开放能力的开发者。

## **概述**

本节内容按逻辑结构分为四类：**企业标识类**（如 `CorpId`）、**用户标识类**（如 `UserId`、`UnionId`）、**应用标识类**（如 `AgentId`、`MiniAppId`、`Unified App ID`）和**安全凭证类**（如 `Client ID/Secret`、`access_token`、`SSOSecret`）。每项术语均包含简明定义、典型使用场景说明及具体查看路径，便于查阅与实践。

## 企业管理后台

[钉钉企业管理后台](https://oa.dingtalk.com/)是企业管理员用于管理组织架构、成员权限、工作台配置的核心平台。通过该后台，管理员可完成通讯录导入、审批流程设置、子管理员授权等操作。

## 开发者后台

[钉钉开发者后台](https://open-dev.dingtalk.com/#/index)是管理应用开发的服务平台。开发者可以配置应用信息，查看应用运行情况等。

**重要**

只有企业管理员和授予了**开发者权限**和**工作台管理**权限的子管理员可访问开发者后台。子管理员权限可以在[钉钉企业管理后台](https://oa.dingtalk.com/)配置。

## 合作伙伴平台

[钉钉合作伙伴平台](https://partner.dingtalk.com/)是面向服务商的统一管理平台，提供技术对接、部署实施、服务等全链路服务。当前涵盖产品方案商、部署服务商和培训服务商三类角色。

## CorpId

`CorpId` 是企业在钉钉中的全局唯一标识符，每个企业对应一个唯一的 `CorpId`。

**使用场景**：调用大多数企业级OpenAPI时，需结合`CorpId` 与`access_token`进行身份验证。

**查看路径**：登录[钉钉开发者后台](http://open-dev.dingtalk.com/)，在**首页**下查看企业的`CorpId`。

![[development-development-basic-concepts_p386893.png]]

## UserId

`UserId` 是企业内员工的唯一身份标识，由钉钉系统生成并绑定至具体成员，创建后不可更改。

**使用场景**：在发送消息、发起审批、查询用户详情等接口中，通常以 `UserId` 作为目标用户的识别参数。

**查看路径**：登录[钉钉管理后台](http://oa.dingtalk.com/)，在**通讯录 > 成员管理**页面，单击员工姓名可查看员工的`UserId`。

![[development-development-basic-concepts_p507263.png]]

## UnionId

`UnionId` 是用户在同一个钉钉开放平台账号范围内的唯一标识。当一个开放平台账号下包含多个应用（包括企业自建应用、第三方企业应用、ISV套件应用等），同一用户在这些应用中拥有相同的 `UnionId`。

**使用场景**：跨应用用户身份识别与数据打通，尤其适用于多应用协同场景下的用户主数据统一。

**查看路径**：可通过调用[查询用户详情](/document/development/query-user-details)接口获取`unionid`参数值。

## **​Unified App ID**

`Unified App ID`（统一应用 ID）是钉钉开放平台为所有类型应用分配的全局唯一标识，无论应用形态如何（企业自建、第三方、机器人、AI助理等），均使用此ID作为标准标识。

相比以往不同场景使用不同ID体系的设计，`Unified App ID` 简化了跨系统沟通和技术对接流程，广泛应用于 OpenAPI 调用、事件订阅、AI 技能开发等场景。

**使用场景**：用于注册回调地址、配置技能接入、标识调用来源等需要统一识别应用身份的场合。

## Client ID/Client Secret

`Client ID` 和 `Client Secret` 是钉钉企业内部应用和第三方企业应用的身份凭证对，用于调用 OAuth2.0 认证流程及相关接口。

* `Client ID`：应用的公开标识。
* `Client Secret`：用于签名和获取 `access_token` 的私密密钥，需妥善保管。

**使用场景**：

* 企业内部应用通过 `Client ID/Secret` 获取 `access_token`。
* 第三方企业应用也使用该机制获取被授权企业的访问令牌。

**查看路径**：在[钉钉开发者后台](http://open-dev.dingtalk.com/)创建企业内部应用或第三方企业应用后，系统会自动生成一对 Client ID和 Client Secret。

**说明**

如果企业内部应用授权给服务商开发，服务商需要根据**CustomKey**和**CustomSecret**获取授权。

![[development-development-basic-concepts_p726090.png]]

## **AgentId**

`AgentId` 是企业内部应用的小程序或网页应用能力的唯一标识，企业在[钉钉开发者后台](http://open-dev.dingtalk.com/)添加应用能力时或企业授权开通第三方企业应用时，系统会自动生成一个AgentId。

**使用场景**：主要用于调用工作台管理类 API（如设置首页应用、更新菜单等）时标识目标应用。

**查看路径**：

* 企业内部应用网页应用AgentId

  ![[development-development-basic-concepts_p726226.png]]
* 企业内部应用小程序AgentId

  ![[development-development-basic-concepts_p726228.png]]

  + 登录[钉钉管理后台](https://oa.dingtalk.com/index.htm#/microApp/microAppList)，单击**工作台 > 应用管理**。
  + 在应用管理页面，选择要查看的官方应用，然后单击**设置**，即可查看应用的AgentId。

## SSOSecret

`SSOSecret` 与 `CorpId` 配合使用，用于获取微应用后台免登所需的 `access_token`，实现管理员从[钉钉管理后台](https://oa.dingtalk.com/)单点登录至第三方应用管理系统，无需重复输入账号密码。

![[development-development-basic-concepts_p380977.png]]

**使用场景**：常用于企业SaaS系统的后台集成，提升运维效率与安全性。

**查看路径**：登录[**钉钉开发者后台**](https://open-dev.dingtalk.com/#/) **> 基本信息 > 开发信息（旧版）**，找到**企业自用账号信息** > **点击查看**SSOSecret。

![[development-development-basic-concepts_p507291.png]]

## access\_token

`access_token` 是调用钉钉 OpenAPI 所必需的身份凭据，用于验证请求来源的合法性。每次调用受保护接口时，必须在请求头或参数中携带有效的 `access_token`。

**使用场景**：

* 企业内部应用，通过[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取access\_token。
* 第三方企业应用，通过[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取access\_token。

  **说明**

  默认access\_token的有效期为7200秒，有效期内重复获取，会返回相同的access\_token，并自动续期。

**权限粒度差异**：

* 企业内部应用的 `access_token` 具备该应用所申请的所有权限。
* 第三方应用的 `access_token` 权限受限于企业授权范围。

## MiniAppId

`MiniAppId` 是钉钉小程序的唯一身份标识，在[钉钉开发者后台](http://open-dev.dingtalk.com/)创建小程序后，系统会自动生成一个MiniAppId。

**使用场景**：用于小程序发布、调试、权限配置及 API 调用中的身份识别。

**查看路径**：

登录[钉钉开发者后台](http://open-dev.dingtalk.com/)，在**应用开发**页面，单击已创建的应用，然后单击**应用信息**查看MiniAppId。

## 附录 A：常见认证术语对照表

|  |  |  |
| --- | --- | --- |
| 术语 | 生成方式 | 主要用途 |
| `authCode` | 用户扫码或点击链接授权后由钉钉返回 | 用于换取用户身份信息（如 `userid`）。 |
| `access_token` | 使用 `Client ID/Secret` 或 `CustomKey/Secret` 调用接口获取 | 调用 OpenAPI 的主要访问凭据。 |
| `jsapi_ticket` | 使用 `access_token` 调用接口获取 | 用于生成前端 JS API 的签名。 |
| `UnionId` | 通过 调用接口获取 | 跨应用用户身份识别。 |
| `SSOToken` | 通过 `SSOSecret` 和 `CorpId` 获取 | 实现后台免登跳转。 |

## 附录 B：快速上手示例

以下伪代码展示如何使用 `CorpId` 和 `access_token` 发起一次最简单的 API 调用，验证环境配置是否正确：

```http
GET https://oapi.dingtalk.com/topapi/v2/user/get?access_token=ACCESS_TOKEN&userid=USERID
```

**步骤说明**：

1. 使用 `Client ID` 和 `Client Secret` 调用获取 `access_token` 接口。
2. 获取任意成员的 `UserId`（可通过管理后台查看）。
3. 构造上述请求，替换 `ACCESS_TOKEN` 和 `USERID`。
4. 若返回 `"errcode": 0, "errmsg": "ok"`，则表示配置成功，可继续后续开发。

> 提示：实际开发中请使用 HTTPS 并做好错误处理与重试机制。
