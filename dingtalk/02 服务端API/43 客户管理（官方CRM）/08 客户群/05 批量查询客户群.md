更新于 2025-10-09调用本接口，批量查询客户群的信息。

例如，有三个客户群，分别为核心客户1群、核心客户2群和核心客户3群，如下图所示。调用本接口，可获取三个客户群的详情信息，包括群名称、群主信息和群管理员信息等。

![[development-query-customer-groups-in-batches_O1CN01PHsBlr28ZmHoYHGb9_!!6000000007947-2-tps-2416-1176.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 客户管理客户群读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=crm_1.0%23GetCrmGroupChatMulti) |
| 第三方企业应用 | 支持 | 客户管理客户群读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=crm_1.0%23GetCrmGroupChatMulti) |
| 第三方个人应用 | 暂不支持 | 客户管理客户群读权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/crm/crmGroupChats/batchQuery HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "openConversationIds" : [ "String" ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| openConversationIds | Array of String | 否 | 客户群openConversationId，调用[查询客户群列表](/document/development/query-the-list-of-customer-groups)接口获取openConversationId参数值。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Array | 客户群列表。 |
| openConversationId | String | 客户群openConversationId。 |
| openGroupSetId | String | 客户群所在的群组openGroupSetId。   * 如果是客户群组裂变出的客户群，会返回该字段。 * 如果是调用[创建客户群](/document/development/create-a-customer-group)接口创建的客户群，不返回该字段。 |
| ownerUserId | String | 群主的userId。 |
| ownerUserName | String | 群主的名字。 |
| name | String | 客户群名。 |
| memberCount | Integer | 客户群成员数量。 |
| gmtCreate | Long | 客户群创建时间戳，单位毫秒。 |
| iconUrl | String | 群头像地址。 |
