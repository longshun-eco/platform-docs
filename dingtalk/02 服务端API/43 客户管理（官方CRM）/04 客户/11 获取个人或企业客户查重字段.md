更新于 2025-10-09调用本接口，获取客户管理中设置个人客户、企业客户的查重字段，包括查重字段的Id和查重字段的别名。

例如，企业客户管理中的客户关系元数据字段如下图所示，有**客户名称、客户标签、协同人、客户群**等字段，fieldId（字段Id）和bizAlias（别名）是表字段的两个属性值。

![[development-obtain-duplicate-check-fields_O1CN016m55v41PzHdEsc9gO_!!6000000001911-2-tps-2336-1108.png]]

企业客户管理后台设置的企业客户查重字段为**客户名称**。调用本接口可获取**客户名称**字段的bizAlias（别名）和fieldId（字段Id）。

![[development-obtain-duplicate-check-fields_O1CN01t28gMo1RoPSgdDic2_!!6000000002158-2-tps-2408-1040.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 获取CRM主数据的接口访问权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=crm_1.0%23GetRelationUkSetting) |
| 第三方企业应用 | 支持 | 获取CRM主数据的接口访问权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=crm_1.0%23GetRelationUkSetting) |
| 第三方个人应用 | 暂不支持 | 获取CRM主数据的接口访问权限 | 暂不支持 |

## 请求方法

```
GET /v1.0/crm/relationUkSettings?relationType=String HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| relationType | String | 是 | 客户类型。   * **crm\_customer**：企业客户 * **crm\_customer\_personal**：个人客户 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Array | 响应结果列表。 |
| fieldId | String | 查重字段的字段Id。 |
| bizAlias | String | 查重字段的别名。  **说明**   * 如果自定义添加的字段设置为查重字段，该字段返回为空。 * 如果预设字段设置为查重字段，该字段正常返回。 |
