更新于 2025-10-09调用本接口，批量新增个人客户、企业客户数据。

例如，在企业的客户管理中的客户如下图所示，只有一个名为**小钉**的客户。

![[development-add-multiple-relationship-data-in-batches_O1CN01PCd8Pd1hcX2iqjmQX_!!6000000004298-2-tps-2408-1098.png]]

调用本接口可以添加两个客户分别是**小钉2**和**小钉3**。接口调用成功后效果如下图所示。

![[development-add-multiple-relationship-data-in-batches_O1CN01xQl9S41w6fiw9K2iD_!!6000000006259-2-tps-2414-1250.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 维护CRM主数据的接口写入权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=crm_1.0%23BatchAddRelationDatas) |
| 第三方企业应用 | 支持 | 维护CRM主数据的接口写入权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=crm_1.0%23BatchAddRelationDatas) |
| 第三方个人应用 | 暂不支持 | 维护CRM主数据的接口写入权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/crm/relationDatas/batch HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "relationType" : "String",
  "operatorUserId" : "String",
  "skipDuplicateCheck" : Boolean,
  "relationList" : [ {
    "bizDataList" : [ {
      "key" : "String",
      "value" : "String",
      "extendValue" : "String"
    } ],
    "bizExtMap" : {
      "key" : "String"
    },
    "relationPermissionDTO" : {
      "principalUserIds" : [ "String" ],
      "participantUserIds" : [ "String" ]
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| relationType | String | 是 | 客户类型。   * **crm\_customer**：企业客户 * **crm\_customer\_personal**：个人客户 |
| operatorUserId | String | 是 | 操作人userId。 |
| skipDuplicateCheck | Boolean | 否 | 是否跳过查重，默认不跳过。   * **true**：是 * **false**：否   企业可在查重设置中设置查重字段，如下图所示。 ![[development-add-multiple-relationship-data-in-batches_O1CN01GLJ3lA1NwsJYjwWNw_!!6000000001635-2-tps-2240-1050.png]] |
| relationList | Array | 是 | 新增客户关系列表，最大值10。 |
| bizDataList | Array | 是 | 新增客户关系的模型数据列表，最大值256。 |
| key | String | 是 | 模型字段key，该参数传客户对象元数据信息中获取字段的name值，调用[获取个人或企业客户的元数据](/document/development/get-metadata-description-of-crm-customer-object-1)接口获取name参数值。  该参数是否必填，取决于调用获取客户对象的元数据信息接口，返回的nillable字段值：   * 若nillable是**true**：本接口参数key和value为非必填。 * 若nillable是**false**：本接口参数key和value为必填。 |
| value | String | 是 | 模型字段value，不同类型的组件value值格式不同，请参考[新增和更新客户字段格式说明V2](/document/development/the-model-fields-supported-by-the-interface-and-their-value)。 |
| extendValue | String | 否 | 模型字段extendValue。 |
| bizExtMap | Map<String, String> | 否 | 扩展业务字段。  **说明**  该参数暂不支持使用。 |
| relationPermissionDTO | Object | 否 | 负责人和协同人信息。 |
| principalUserIds | Array of String | 否 | 负责人userId列表。 |
| participantUserIds | Array of String | 否 | 协同人userId。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| results | Array | 批量新增结果列表，results返回结果和新增数据顺序是一致的，可以查看每条客户数据分别对应的结果是否成功。  **说明**  例如，调用本接口批量新增了两个客户，第一个客户字段被查重规则拦截导致写入失败；第二个客户写入正常。返回的信息格式为 ![[development-add-multiple-relationship-data-in-batches_O1CN01EKtqea1ir1BWb9uRp_!!6000000004465-2-tps-1010-698.png]] |
| success | Boolean | 数据是否保存成功。   * **true**：成功 * **false**：失败 |
| errorCode | String | 错误码。   * 如果保存失败，表示失败的错误码。 * 如果保存成功，该字段不返回。 |
| errorMsg | String | 错误信息。   * 如果保存失败，表示失败的错误原因。 * 如果保存成功，该字段不返回。 |
| relationId | String | 保存成功的客户Id。 |
| duplicatedRelationIds | Array of String | 新增客户反馈信息。  **说明**   * 如果新增客户成功，该字段返回空。 * 如果因为查重导致失败，该字段返回重复的客户Id列表。 * 如果是其他原因导致的失败，该字段返回空，可以查看errorMsg返回的错误详情。 |
