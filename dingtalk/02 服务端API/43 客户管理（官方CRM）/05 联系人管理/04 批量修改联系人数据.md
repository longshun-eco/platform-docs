更新于 2025-10-09调用本接口，批量修改联系人数据，可修改姓名、手机号、职位、邮箱等信息。

例如，企业客户管理中的联系人如下图所示，有两个客户小钉的联系人，分别是**客户小钉的联系人1**和**客户小钉的联系人2**。

![[development-modify-contact-data-in-batches_O1CN0141w5pD1iJ7sa71yV3_!!6000000004391-2-tps-2744-1262.png]]

调用本接口可批量修改这两个联系人的信息，例如修改联系人的姓名，将联系人姓名分别修改为**修改的联系人1**和**修改的联系人2**。接口调用成功后，效果如下图所示。

![[development-modify-contact-data-in-batches_O1CN01yYmejS1FFLl1iMK7Y_!!6000000000457-2-tps-2732-1242.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 维护CRM主数据的接口写入权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=crm_1.0%23BatchUpdateContacts) |
| 第三方企业应用 | 支持 | 维护CRM主数据的接口写入权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=crm_1.0%23BatchUpdateContacts) |
| 第三方个人应用 | 暂不支持 | 维护CRM主数据的接口写入权限 | 暂不支持 |

## 请求方法

```
PUT /v1.0/crm/contacts/batch HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "operatorUserId" : "String",
  "relationList" : [ {
    "bizDataList" : [ {
      "key" : "String",
      "value" : "String",
      "extendValue" : "String"
    } ],
    "bizExtMap" : {
      "key" : "String"
    },
    "relationId" : "String"
  } ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| operatorUserId | String | 是 | 操作人userId。 |
| relationList | Array | 是 | 联系人数据列表，最大值10。 |
| bizDataList | Array | 否 | 新增联系人的模型数据列表，最大值256。 |
| key | String | 是 | 模型字段key，该参数传[获取联系人的元数据](/document/development/gets-the-metadata-description-of-a-crm-contact-object)接口中获取的name字段值。  **说明**  该参数是否必填，取决于**获取联系人的元数据**接口，返回的nillable字段值。   * nillable是**true**：本接口参数key和value为非必填。 * nillable是**false**：本接口参数key和value为必填。 |
| value | String | 是 | 模型字段value，不同类型的组件value值格式不同，请参考[新增和更新联系人字段格式说明V2](/document/development/add-and-update-contact-field-format-description-v2)。 |
| extendValue | String | 否 | 模型字段extendValue。 |
| bizExtMap | Map<String, String> | 否 | 扩展业务字段。  **说明**  该参数暂不支持使用。 |
| relationId | String | 是 | 要修改的联系人ID，调用[根据指定条件查询联系人数据](/document/development/api-getcontacts)接口获取instance\_id参数值。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| results | Array | 批量修改结果列表，results返回结果和修改数据顺序是一致的，可以查看每条联系人数据分别对应的结果是否成功。  **说明**  例如，调用本接口批量修改了两个联系人，第一个联系人修改失败；第二个联系人修改成功。返回的信息格式如图所示。 ![[development-modify-contact-data-in-batches_O1CN01h1k8wG1rbI8DggHlF_!!6000000005649-2-tps-806-442.png]] |
| success | Boolean | 数据是否更新成功。   * **true**：成功 * **false**：失败 |
| errorCode | String | 错误码。   * 如果更新成功，该字段不返回。 * 如果更新失败，表示失败的错误码。 |
| errorMsg | String | 错误信息。   * 如果更新成功，该字段不返回。 * 如果更新失败，表示失败的错误原因。 |
| relationId | String | 联系人Id。   * 如果更新成功，表示联系人Id。 * 如果更新失败，该字段不返回。 |
