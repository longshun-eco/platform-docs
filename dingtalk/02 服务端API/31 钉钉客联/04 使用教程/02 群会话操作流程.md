更新于 2025-12-08本文档介绍了如何调用钉钉客联群相关接口实现群会话操作的相关流程。首先创建一个企业内部应用，再使用钉钉客联提供的API，实现创建群会话、获取群成员列表、群会话管理与解散群会话流程。

## **流程简介**

前提条件：完成[创建应用](/document/dingstart/create-application)的流程。

步骤一：获取应用凭证信息，获取应用 Client ID 和 Client Secret。

步骤二：申请接口权限，申请钉钉客联相关接口的权限。

步骤三：获取应用访问凭证[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)。调用接口时，通过accessToken鉴权调用者身份。

步骤四：进入钉钉工作台，在钉钉客联应用中创建群模板。

步骤五：调用钉钉客联相关API：

1. 调用服务端API-[创建钉钉客联普通互通群](/document/development/create-common-group-new-version)接口，获取群会话`OpenConversationId`。
2. 根据群会话`OpenConversationId`，调用服务端API-[获取钉钉客联H5页面地址](/document/development/get-the-dingtalk-guest-group-session-address)接口，钉外用户可通过该地址进入对应群内。
3. 根据群会话`OpenConversationId`，调用服务端API-[查询钉钉客联互通群成员列表](/document/development/queries-the-group-member-list)接口，获取互通群成员列表。
4. 根据群会话`OpenConversationId`，进行群会话管理操作：

   * 根据群会话`OpenConversationId`，调用服务端API-[添加钉钉客联互通群成员](/document/development/add-a-group-member-1)接口，添加互通群成员。
   * 根据群会话`OpenConversationId`，调用服务端API-[更换钉钉客联互通群群主](/document/development/change-group-owner)接口，更新互通群主。
   * 根据群会话`OpenConversationId`，调用服务端API-[移除钉钉客联互通群成员](/document/development/remove-group-members)接口，移除互通群成员。
   * 根据群会话`OpenConversationId`，调用服务端API-[修改钉钉客联互通群头像](/document/development/modify-the-avatar-of-a-communication-group)接口，修改互通群头像。
   * 根据群会话`OpenConversationId`，调用服务端API-[修改钉钉客联互通群名称](/document/development/modify-the-group-name)接口，修改互通群名称。
5. 根据群会话`OpenConversationId`，调用服务端API-[解散钉钉客联互通群](/document/development/disband-bc-interconnection-group)接口，解散互通群。
6. 调用服务端API-[批量查询跨钉两人互通群列表](/document/development/queries-the-session-information-of-two-population-groups)接口，获取跨钉两人群列表。

## **前提条件**

完成[创建应用](/document/dingstart/create-application)的流程。

## **步骤一：获取应用凭证**

1. 选择目标应用，进入应用详情页，单击**基础信息** > **凭证与基础信息**。
2. 获取应用 Client ID 和 Client Secret。

## **步骤二：添加接口权限**

1. 单击**开发配置** > **权限管理**，在权限搜索框中输入`InterConnect.Common.ReadWrite`，并申请权限。

## **步骤三：获取应用访问凭证accessToken**

**重要**

服务端API差异详情参见[旧版服务端API、新版服务端API的区别](/document/development/how-to-call-apis#section-8lr-id4-rbz)，以下接口均使用新版服务端API接口，SDK下载详情参见[服务端SDK下载](/document/development/download-the-server-side-sdk)。

根据步骤一中的 Client ID 和 Client Secret，获取应用访问凭证[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)。
