更新于 2025-09-10调用本接口，获取部门下的人员列表信息。

### **接口功能介绍**

调用本接口可以获取行业通讯录下某个部门的人员列表，具体信息如下图。

![[development-obtains-the-list-of-people-under-a-department_p490528.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 查询行业通讯录信息权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.industry.user.list) |
| 第三方企业应用 | 是 | 查询行业通讯录信息权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.industry.user.list) |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/industry/user/list`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | bE74xxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| dept\_id | Number | 是 | 1 | 部门id，可调用[获取部门列表](/document/development/obtain-the-department-list-v2)接口获取dept\_id参数值。 |
| role | String | 否 | Village | 行业相关，不同行业角色不一样。例如：   * 针对家校    + **teacher**: 老师   + **guardian**: 监护人   + **student**: 学生 * 针对农村    + **GroupManager**: 组长   + **HeadOfHouseHold**: 户主   + **HouseAdmin**: 家庭管理员   + **Villager**：村民   + **Leaseholder**：租客 |
| cursor | Number | 否 | 1 | 分页查询的游标。 |
| size | Number | 是 | 10 | 分页查询的大小，最大值1000。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/industry/user/list?access_token=ACCESS_TOKEN
```

请求正文

```json
{
    "dept_id": 1,
    "cursor": 1,
    "size": 10
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/industry/user/list");
OapiIndustryUserListRequest req = new OapiIndustryUserListRequest();
req.setDeptId(1L);
req.setSize(10L);
req.setCursor(1L);
OapiIndustryUserListResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```
