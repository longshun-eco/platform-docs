### **接口功能介绍**

调用本接口可以获取行业通讯录下，具体部门的用户的详情，具体信息如下。

![[development-queries-department-user-details_p490527.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 查询行业通讯录信息权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.industry.user.get) |
| 第三方企业应用 | 是 | 查询行业通讯录信息权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.industry.user.get) |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/industry/user/get`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | bE74xxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| dept\_id | Number | 是 | 1 | 部门ID，可调用[获取部门列表](/document/development/obtain-the-department-list-v2)接口获取dept\_id参数值。 |
| userid | String | 是 | user01 | 员工userId，可调用[获取部门下人员列表](/document/development/obtains-the-list-of-people-under-a-department)接口获取userid参数值。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/industry/user/get?access_token=ACCESS_TOKEN
```

请求正文

```json
{
    "dept_id": 1,
    "userid": "user01"

}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/industry/user/get");
OapiIndustryUserGetRequest req = new OapiIndustryUserGetRequest();
req.setDeptId(1L);
req.setUserid("user01");
OapiIndustryUserGetResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

## 错误码

| --- | --- | --- |
| 错误码（errcode） | 错误码描述（errmsg） | 解决方案 |

| 40009 | 无效的dept\_id | 请确认dept\_id是否正确 |
| 33012 | 无效的userid | 请确认员工id是否为空 |
| 400001 | 系统错误 | 请稍后重试 |
