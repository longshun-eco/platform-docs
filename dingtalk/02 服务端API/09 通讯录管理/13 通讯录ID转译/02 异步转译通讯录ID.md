更新于 2025-09-11调用本接口，发起异步通讯录ID内容转译，替换产品方案商通讯录权限范围内的用户ID和部门ID。

### 接口功能介绍

如果第三方企业应用选择了[通讯录加密](/document/development/address-book-encryption)部署方式，该应用调用通讯录等相关接口，无法获取授权企业员工姓名、职位和所在部门名称等信息。可以通过以下方法获取：

调用本文接口，第三方企业应用提供导出用户信息文件的功能，包含企业用户的名称、职位和所在部门名称。

### 实现步骤

例如，某企业授权开通一个第三方企业应用，使用应用导出本企业可以使用该第三方应用的员工列表及信息，包含员工userID、员工姓名、员工所在部门ID和员工所在部门名称。可按照以下步骤实现。

1.构造需要转译的文档，示例如下图所示。

**说明**

只支持xlsx，xls，csv，txt格式文件。

![[development-asynchronous-address-book-file-content-translation_O1CN01XvGYPv1NGjswLGLT7_!!6000000001543-2-tps-1198-344.png]]

2.调用[获取文件上传信息](/document/development/obtain-file-upload-informations)接口，获取media\_id。

3.调用本接口，获取转译任务ID。

**说明**

调用本接口后，如果文件转译成功，会触发异步任务回调通知。

4.调用[获取异步转译任务结果](/document/development/obtains-the-results-of-an-asynchronous-translation-task)接口，获取转译后的文件下载链接。

**说明**

返回的文件下载链接建议使用`<a>标签`引用，`download`属性不可指定（浏览器兼容性问题）。

5.用户下载得到的文件内容，如下图所示。

![[development-asynchronous-address-book-file-content-translation_O1CN01y9EIgQ1G8JI3rCEP6_!!6000000000577-2-tps-1192-344.png]]

在调用本接口的过程中，你可能会遇到以下问题：

* **问题**：使用`NodeJs`第三方库`exceljs`生成的`excel`文件，文件内容看上去正常，但是转译不成功。
* **问题原因**：`exceljs`在不开启`sharedString`的情况下，会默认使用`formula string`进行文件内容写入，这种行为不合规范，导致转译失败。
* **解决方法**：查看`exceljs`文档，强制开启`useSharedStrings`选项。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 暂不支持 | 成员信息读权限 | 暂不支持 |
| 第三方企业应用 | 支持 | 成员信息读权 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=contact_1.0%23TranslateFile) |
| 第三方个人应用 | 暂不支持 | 成员信息读权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/contact/files/translate HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "medias" : {
    "key" : "String"
  },
  "outputFileName" : "String",
  "unionId" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| medias | Map<String, String> | 是 | 需要转译的文件信息。   * Map类型，key为钉盘文件media\_id，调用[获取文件上传信息](/document/development/obtain-file-upload-informations)接口获取。 * value为文件名，需要包含文件扩展名，与调用[获取文件上传信息](/document/development/obtain-file-upload-informations)接口时传入的文件名称保持一致。   **说明**   * 只支持xlsx，xls，csv，txt类型文件。 * 最大数量20个文件。 |
| outputFileName | String | 否 | 转译完打包的文件名，不需带后缀。  **说明**  若medias中文件个数大于1，则该参数必填。 |
| unionId | String | 是 | 当前操作导出文档的用户的unionId，可以调用[通过免登码获取用户信息](/document/development/obtain-the-userid-of-a-user-by-using-the-log-free)接口获取。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| jobId | String | 异步转译任务ID，最大长度为64字符。  **说明**  **jobId**用于接口[获取异步转译任务结果](/document/development/obtains-the-results-of-an-asynchronous-translation-task)传递。 |
