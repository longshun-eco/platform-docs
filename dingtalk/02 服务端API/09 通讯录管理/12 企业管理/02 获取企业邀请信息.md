更新于 2025-09-11调用本接口，获取企业的邀请信息。

### 接口功能介绍

例如，本接口用于获取企业邀请信息的链接，被邀请者访问该链接展示的页面如下图。

![[development-obtain-invitation-information_O1CN01DXEOoO1TXa84MK5Ua_!!6000000002392-2-tps-1284-930.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录基本数据读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23GetApplyInviteInfo) |
| 第三方企业应用 | 支持 | 通讯录基本数据读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=contact_1.0%23GetApplyInviteInfo) |
| 第三方个人应用 | 暂不支持 | 通讯录基本数据读权限 | 暂不支持 |

## 请求方法

```
GET /v1.0/contact/invites/infos?inviterUserId=String&deptId=Long HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| inviterUserId | String | 否 | 邀请者的userId，详情参见[基础概念-UserId](/document/development/basic-concepts-beta#section-aa9-zk1-os6)。  **说明**  该参数不填写，默认邀请者为当前企业创建者的userId。 ![[development-obtain-invitation-information_userId信息.png]] |
| deptId | Long | 否 | 获取部门邀请链接的部门ID。 ![[development-obtain-invitation-information_O1CN01HpHFw71iID6zGnwwa_!!6000000004389-2-tps-2334-257.png]] |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| inviteSwitch | Boolean | 是否开启邀请。   * **true**：开启 * **false**：不开启 |
| searchNameInvite | Boolean | 是否开启通过企业名称搜索申请。   * **true**：开启 * **false**：不开启 |
| orgApplyCodeInvite | Boolean | 是否开启通过团队号申请加入。   * **true**：开启 * **false**：不开启 |
| linkInvite | Boolean | 是否开启通过链接邀请加入。   * **true**：开启 * **false**：不开启 |
| inviteUrl | String | 邀请链接。 |
| auditType | Long | 审核类型。   * **0**：无需审核 * **1**：有权限的子管理员   **说明**  仅部门邀请有效。 |
| empApplyJoinDept | Boolean | 是否允许员工扫码加入部门。   * **true**：允许 * **false**：不允许   **说明**  仅在无需审核的时候有效，已经在组织内的成员通过扫描部门二维码加入某个部门。 |
