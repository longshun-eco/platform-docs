更新于 2025-09-10调用本接口根据用户的userid删除指定用户。

例如下图，调用本接口的实现效果与直接删除员工效果一致。

![[development-delete-a-user_p407047.png]]

**说明**

调用本接口成功删除通讯录员工，同时会删除以下内容：

* 该员工所在的考勤组会同步删除该员工。
* 智能人事花名册会同步删除该员工，员工信息可以在智能人事已离职人员中查看。

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 维护通讯录的接口访问权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.v2.user.delete) |
| 第三方企业应用 | 否 | — | — |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/v2/user/delete`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | Be3xxxx | 调用该接口的应用凭证，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| userid | String | 是 | uxxxx3 | 员工的userid。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| request\_id | String | 4f9md9obopn2 | 请求ID。 |
| errcode | Number | 0 | 返回码。 |
| errmsg | String | 不合法的userid | 调用失败时返回的错误信息。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/v2/user/delete?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "userid":"user123"
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/v2/user/delete");
OapiV2UserDeleteRequest req = new OapiV2UserDeleteRequest();
req.setUserid("user123");
OapiV2UserDeleteResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
    "errcode": 0,
    "errmsg":"ok"
    "request_id": "4f9md9obopn2"
}
```

## 错误码

|  |  |  |
| --- | --- | --- |
| 错误码（errcode） | 错误码描述（errmsg） | 解决方案 |
| 33012 | 无效的userId | 请检查userId是否正确 |
| 400002 | 无效的参数 | 请确认参数是否按要求输入 |
| -1 | 系统繁忙 | 请稍后再试 |
