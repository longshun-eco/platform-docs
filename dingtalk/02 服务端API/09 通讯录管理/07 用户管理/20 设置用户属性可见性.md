更新于 2025-09-11调用本接口，设置用户属性可见性。

### 接口功能说明

调用本接口，可新增或更新企业员工属性字段隐藏设置，具体信息如下图所示。

![[development-add-or-update-the-hidden-settings-of-the-employee-property_O1CN01WCt0vH1q2CBjcf6St_!!6000000005437-2-tps-1954-1209.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录数据可见性的管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23UpdateEmpAttrbuteVisibilitySetting) |
| 第三方企业应用 | 支持 | 通讯录数据可见性的管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=contact_1.0%23UpdateEmpAttrbuteVisibilitySetting) |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/contact/staffAttributes/visibilitySettings HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "id" : Long,
  "name" : "String",
  "description" : "String",
  "objectStaffIds" : [ "String" ],
  "objectDeptIds" : [ Long ],
  "objectTagIds" : [ Long ],
  "hideFields" : [ "String" ],
  "excludeStaffIds" : [ "String" ],
  "excludeDeptIds" : [ Long ],
  "excludeTagIds" : [ Long ],
  "active" : Boolean
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| id | Long | 否 | 设置ID。  **说明**   * 新增时设置ID为空。 * 修改时设置ID为需要修改的ID。 |
| name | String | 否 | 设置的名称。 |
| description | String | 否 | 设置的描述信息。 |
| objectStaffIds | Array of String | 否 | 需要隐藏字段的员工userId，可通过以下方式获得。   * 企业内部应用，可通过[通过免登码获取用户信息](https://open.dingtalk.com/document/orgapp/obtain-the-userid-of-a-user-by-using-the-log-free)获得userid参数值。 * 第三方企业应用，可通过[通过免登码获取用户信息](https://open.dingtalk.com/document/isvapp/obtain-the-userid-of-a-user-by-using-the-log-free)获得userid参数值。 |
| objectDeptIds | Array of Long | 否 | 需要隐藏字段的部门deptId，可通过以下方式获得。   * 企业内部应用，可通过[获取部门列表](https://open.dingtalk.com/document/orgapp/obtain-the-department-list-v2)获得dept\_id参数值。 * 第三方企业应用，可通过[获取部门列表](https://open.dingtalk.com/document/isvapp/get-department-list)获得dept\_id参数值。 |
| objectTagIds | Array of Long | 否 | 需要隐藏字段的角色roleId，可通过以下方式获得：   * 企业内部应用，可通过[获取角色列表](https://open.dingtalk.com/document/orgapp/obtains-a-list-of-enterprise-roles)获得id参数值。 * 第三方企业应用，可通过[获取角色列表](https://open.dingtalk.com/document/isvapp/get-role-list)获得id参数值。 |
| hideFields | Array of String | 否 | 需要隐藏的用户属性ID列表。   * 当需要隐藏的用户属性是企业自定义属性时，传入该属性的属性ID。 * 当需要隐藏的用户属性是系统默认属性时，传入系统默认属性列表。   **说明**  系统默认属性列表如下：   * **name**： 姓名 * **department**： 部门 * **email**： 邮箱 * **manager**： 直属主管 * **title**： 职位 * **mobile**： 手机号 * **ext\_number**： 分机号 * **work\_station**： 办公地点 * **remark**： 备注 * **hire\_date**： 入职时间 * **job\_number**： 工号 |
| excludeStaffIds | Array of String | 否 | 可见隐藏字段的员工userId。 |
| excludeDeptIds | Array of Long | 否 | 可见隐藏字段的部门deptId。 |
| excludeTagIds | Array of Long | 否 | 可见隐藏字段的角色roleId。 |
| active | Boolean | 否 | 是否生效。   * **true**：生效 * **false**：不生效 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Long | 设置ID。 |
