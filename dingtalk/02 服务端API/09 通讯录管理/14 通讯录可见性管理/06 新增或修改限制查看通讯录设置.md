更新于 2025-09-11调用本接口，新增或修改员工、部门、角色限制查看通讯录的设置。

### 接口功能介绍

钉钉OA管理后台中，可以限制本部门成员查看通讯录，本接口的能力是对限制本部门成员查看通讯录的拓展，不仅能实现限制部门内成员查看通讯录，也可以实现限制员工和角色查看通讯录。

![[development-add-or-modify-visibility-settings-for-address-book-restrictions_O1CN01c1IHaB1maJG9jwkSO_!!6000000004970-0-tps-2400-1350.jpg]]

**说明**

本接口的限制查看设置与OA后台的限制查看设置是相互独立存储，最终生效结果两边的设置是或的关系。比如：同一个部门，本接口或者OA后台有任意一方设置了仅能查看自己，最终这个部门就只能查看自己。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录数据可见范围管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23UpdateContactRestrictSetting) |
| 第三方企业应用 | 暂不支持 | 通讯录数据可见范围管理权限 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 通讯录数据可见范围管理权限 | 暂不支持 |

## 请求方法

```
PUT /v1.0/contact/restrictions/settings HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "id" : Long,
  "name" : "String",
  "description" : "String",
  "subjectUserIds" : [ "String" ],
  "subjectDeptIds" : [ Long ],
  "subjectTagIds" : [ Long ],
  "type" : "String",
  "excludeUserIds" : [ "String" ],
  "excludeDeptIds" : [ Long ],
  "excludeTagIds" : [ Long ],
  "active" : Boolean,
  "restrictInUserProfile" : Boolean,
  "restrictInSearch" : Boolean
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| id | Long | 否 | 设置ID。   * 如果需要新增设置，该参数不传。 * 如果需要修改已有的设置，需要指定该参数，通过调用[获取限制查看通讯录设置列表](/document/development/gets-a-list-of-address-book-limit-visibility-settings)接口获取。 |
| name | String | 否 | 设置名称。 |
| description | String | 否 | 设置描述。 |
| subjectUserIds | Array of String | 否 | 需要限制查看通讯录的用户userId列表，可调用[获取部门用户userid列表](/document/development/query-the-list-of-department-userids)接口获取userId。  **说明**  subjectUserIds、subjectDeptIds、subjectTagIds三个参数内元素个数之和不能超过50。 |
| subjectDeptIds | Array of Long | 否 | 需要限制查看通讯录的部门ID列表，可调用[获取部门列表](/document/development/obtain-the-department-list-v2)接口获取dept\_id。  **说明**  subjectUserIds、subjectDeptIds、subjectTagIds三个参数内元素个数之和不能超过50。 |
| subjectTagIds | Array of Long | 否 | 需要限制查看通讯录的角色ID，通过调用[获取角色列表](/document/development/obtains-a-list-of-enterprise-roles)接口获取。  **说明**  subjectUserIds、subjectDeptIds、subjectTagIds三个参数内元素个数之和不能超过50。 |
| type | String | 是 | 限制类型，有以下取值：   * **onlySelf**：只能查看自己，不能看自己之外的其他部门和人。 * **onlySelfDeptAndChild**：只能看到自己所在的部门及子部门，不能看到其他部门和人。 * **excludeNode**：默认值，只能看到白名单列表中的部门和人。   **说明**  当该参数值为excludeNode时，设置的白名单才生效。 |
| excludeUserIds | Array of String | 否 | 白名单用户userId，可调用[获取部门用户userid列表](/document/development/query-the-list-of-department-userids)接口获取userId。  **说明**   * excludeUserIds、excludeDeptIds、excludeTagIds三个参数内元素个数之和不能超过50。 * 当type参数值为excludeNode时，设置的白名单才生效。 |
| excludeDeptIds | Array of Long | 否 | 白名单部门ID，通过调用[获取部门列表](/document/development/obtain-the-department-list-v2)接口获取。  **说明**   * excludeUserIds、excludeDeptIds、excludeTagIds三个参数内元素之和不能超过50。 * 当type参数值为excludeNode时，设置的白名单才生效。 |
| excludeTagIds | Array of Long | 否 | 白名单角色ID，通过调用[获取角色列表](/document/development/obtains-a-list-of-enterprise-roles)接口获取。  **说明**   * excludeUserIds、excludeDeptIds、excludeTagIds三个参数内元素个数之和不能超过50。 * 当type参数值为excludeNode时，设置的白名单才生效。 |
| active | Boolean | 否 | 本次设置是否生效。   * true：生效 * false：不生效 |
| restrictInUserProfile | Boolean | 否 | 是否同时限制查看个人资料页。   * true：是 * false：否   **说明**  如果限制查看，在钉钉客户端点击不在可看见范围内的员工头像，不会展示当前组织的资料信息。 |
| restrictInSearch | Boolean | 否 | 是否同时限制搜索。   * true：是 * false：否   **说明**  如果限制搜索，在钉钉客户端搜索不在可看见范围内的员工，会搜索不到结果。 |

## 返回参数

|  |  |  |

| 名称 | 类型 | 描述 |
| result | Long | 限制设置ID。 |
