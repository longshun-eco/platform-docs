更新于 2025-09-10调用本接口，查询指定用户所属的所有父级部门。

**说明**

本接口不受通讯录权限范围限制。

例如，员工A的所属部门组织如下图所示。当传入员工A的userId时，返回的结果按顺序依次为其所有父部门的ID，直到根部门，在本示例中为[[456,123,1],[789,1]]。

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |

| 第三方企业应用 | 是 | 通讯录部门信息读权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.v2.department.listparentbyuser) |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/v2/department/listparentbyuser`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | bE74xxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| userid | String | 是 | 100 | 要查询的用户的userid。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| errcode | Number | 0 | 返回码。 |
| request\_id | String | nfe328zf67ec | 请求ID。 |
| errmsg | String | 不合法的userid | 调用失败时返回的错误信息。 |
| result | DeptListParentByUserResponse |  | 返回结果。 |
| parent\_list | DeptParentResponse[] |  | 父部门列表集合。 |
| parent\_dept\_id\_list | Number[] | [456, 123, 1] | 父部门列表。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/v2/department/listparentbyuser?access_token=ACCESS_TOKEN
```

请求正文

```json
{
 "userid":"100"
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/v2/department/listparentbyuser");
OapiV2DepartmentListparentbyuserRequest req = new OapiV2DepartmentListparentbyuserRequest();
req.setUserid("100");
OapiV2DepartmentListparentbyuserResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
    "errcode": 0,
    "errmsg":"ok",
    "result": {
        "parent_list": [
            {
                "parent_dept_id_list": [

}
```
