更新于 2025-09-11授权当前组织的企业账号在加入其他组织之后，在其他组织查看组织的企业账号信息的具体字段

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业账号加入其他组织后数据可见性权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23OrgAccountMobileVisibleInOtherOrg) |
| 第三方企业应用 | 暂不支持 | 暂不支持 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
PUT /v1.0/contact/orgAccounts/mobiles/visibleInOtherOrg HTTP/1.1
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "toCorpIds" : [ "String" ],
  "optUserId" : "String",
  "fields" : [ "String" ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| toCorpIds | Array of String | 是 | 被授权组织对应corpId列表。  **说明**  被授权组织的corpId列表和调用接口的调用者对应 userId，都不能为空且参数出错会导致调用报错。 |
| optUserId | String | 是 | 当前调用接口的调用者对应userId。  **说明**  用于数据审计，需填写当前组织下真实的员工userId，否则接口调用会失败。 |
| fields | Array of String | 否 | 授权字段:  **说明**  如果fields为空或者空集合表示仅授权mobile（历史版本兼容）   * **mobile**：企业账号手机号。 * **status**： 企业账号停用状态。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Boolean | 接口调用结果 |
