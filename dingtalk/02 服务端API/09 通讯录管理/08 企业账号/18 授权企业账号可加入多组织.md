更新于 2025-09-11调用本接口，授权企业账号可以加入多个组织，只有被授权后企业账号才能加入外部组织。

例如，钉钉组织1内存在企业账号a，企业账号a想被添加到钉钉组织2，需要按照以下步骤实现：

1. 钉钉组织1，调用本接口授权企业账号a可加入钉钉组织2。
2. 钉钉组织2，调用 [邀请其他组织企业账号加入](/document/development/invite-other-organization-specific-accounts-to-join)接口，将企业账号a添加到本组织。

**说明**

授权成功后，只支持接口操作添加进入企业。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录企业账号安全数据读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23MultiOrgPermissionGrant) |
| 第三方企业应用 | 暂不支持 | 通讯录企业账号安全数据读写权限 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 通讯录企业账号安全数据读写权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/contact/orgAccounts/multiOrgPermissions/auth HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "joinCorpId" : "String",
  "grantDeptIdList" : [ Long ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| joinCorpId | String | 是 | 被授权的组织CorpId。 |
| grantDeptIdList | Array of Long | 否 | 授权的部门列表。   * 如果该参数不传，授权整个企业。 * 如果该参数传值，授权参数值对应的部门。 |
