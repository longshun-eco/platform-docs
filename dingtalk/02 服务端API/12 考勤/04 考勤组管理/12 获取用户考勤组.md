更新于 2025-09-10调用本接口，获取员工的考勤组信息，一个员工在一个企业中只能属于一个考勤组。

例如，用户在**测试考勤组**，**测试考勤组**部分信息如下图所示。调用本接口，可以获取到**测试考勤组**相关信息，包括考勤组名称、考勤类型等。

![[development-queries-a-user-attendance-group_p960299.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 支持 | 默认开通，无需申请 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.attendance.getusergroup) |
| 第三方企业应用 | 支持 | 默认开通，无需申请 | [调试](https://open-dev.dingtalk.com/apiExplorer?spm=ding_open_doc.document.0.0.18332355WN7mYG#/?devType=isv&api=dingtalk.oapi.attendance.getusergroup) |
| 第三方个人应用 | 暂不支持 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/attendance/getusergroup`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | 6d1bxxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| userid | String | 是 | user123 | 员工在企业内的userId。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/attendance/getusergroup?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "userid":"user123"
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/attendance/getusergroup");
OapiAttendanceGetusergroupRequest req = new OapiAttendanceGetusergroupRequest();
req.setUserid("user123");
OapiAttendanceGetusergroupResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```
