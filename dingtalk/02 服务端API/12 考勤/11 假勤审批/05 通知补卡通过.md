例如，员工小钉在11月14日的打卡统计信息如下图所示，上下班均未打卡。

![[development-make-up-the-card-after-approval_p520538.png]]

调用本接口，可以实现补卡操作，在早上8:58补卡，调用效果如下图所示。

![[development-make-up-the-card-after-approval_p520540.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 考勤组管理权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.attendance.approve.check) |
| 第三方企业应用 | 是 | 考勤组管理权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.attendance.approve.check) |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/attendance/approve/check`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | 6d1bxxxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| userid | String | 是 | manager4220 | 员工的userId。 |
| work\_date | String | 是 | 2019-08-15 | 要补卡的时间，可通过[批量查询人员排班信息](/document/development/query-batch-scheduling-information)接口获取的work\_date值。 |
| punch\_id | Number | 是 | 176877195378 | 要补的排班ID，可通过[批量查询人员排班信息](/document/development/query-batch-scheduling-information)接口获取的shift\_id值。 |
| punch\_check\_time | String | 是 | 2019-08-15 08:00 | 排班的打卡时间，可通过[批量查询人员排班信息](/document/development/query-batch-scheduling-information)接口获取的plan\_check\_time值。 |
| user\_check\_time | String | 是 | 2019-08-15 07:59 | 用户打卡时间。 |
| approve\_id | String | 是 | aasdv13124 | 审批单ID，自定义值。 |
| jump\_url | String | 是 | https://open.dingtalk.com/ | 审批单跳转地址。 |
| tag\_name | String | 是 | 补卡 | 审批单名称。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| errcode | Number | 0 | 返回码。 |
| errmsg | String | ok | 返回码描述。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/attendance/approve/check?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "punch_check_time":"2019-08-15 08:00",
        "jump_url":"https://open.dingtalk.com/",
        "work_date":"2019-08-15",
        "tag_name":"补卡",
        "user_check_time":"2019-08-15 07:59",
        "punch_id":"176877195378",
        "approve_id":"aasdv13124",
        "userid":"manager4220"
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/attendance/approve/check");
OapiAttendanceApproveCheckRequest req = new OapiAttendanceApproveCheckRequest();
req.setUserid("manager4220");
req.setWorkDate("2019-08-15");
req.setPunchId(176877195378L);
req.setPunchCheckTime("2019-08-15 08:00");
req.setUserCheckTime("2019-08-15 07:59");
req.setApproveId("aasdv13124");
req.setJumpUrl("https://open.dingtalk.com/");
req.setTagName("补卡");
OapiAttendanceApproveCheckResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
        "errcode":0,
        "errmsg":"ok"
}
```
