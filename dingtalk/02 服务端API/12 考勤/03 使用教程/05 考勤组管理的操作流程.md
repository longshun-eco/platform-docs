**登录**[**钉钉管理后台**](https://oa.dingtalk.com/) **> 工作台 > 应用管理 > 考勤打卡 > 考勤组管理**

## **接入流程简介**

本文介绍了创建一个企业内部应用，使用**考勤组管理**提供的API，实现创建、获取、更新和删除考勤组等。

前提条件：完成[创建应用](/document/orgapp/create-an-application)的流程

步骤一：获取应用凭证信息，获取应用 Client ID 和 Client Secret。

步骤二：申请接口权限，申请考勤相关接口权限。

步骤三：获取应用访问凭证[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)。调用接口时，通过accessToken鉴权调用者身份。

步骤四：调用考勤相关API：

1. 调用服务端API-[创建考勤组](/document/development/attendance-group-write)接口，获取考勤组`id`。

   **说明**

   本文档示例采用固定班制考勤组信息。

   如果需要创建**固定班制考勤组**，调用创建考勤组接口前，需确保已设置了班次：

   * 如果无班次，调用服务端API-[创建班次](/document/development/create-modify-shifts)接口，获取班次ID。
   * 如果已有班次，调用服务端API-[获取班次摘要信息](/document/development/enterprise-shift-query-in-batches)接口，获取已有班次ID。
2. 获取考勤组信息。

   * 获取企业下所有的考勤组信息

     + 调用服务端API-[批量获取考勤组摘要](/document/development/batch-query-of-simple-information-of-the-attendance-group)，获取企业下所有的考勤组摘要信息。
     + 调用服务端API-[批量获取考勤组详情](/document/development/batch-obtain-attendance-group-details)，获取企业下所有的考勤组详情。
   * 调用服务端API-[搜索考勤组摘要](/document/development/attendance-group-search)接口，根据考勤组名称模糊查询考勤组信息。
   * 获取单个考勤组信息

     + 根据考勤组`id`，调用服务端API-[获取考勤组详情](/document/development/query-a-single-attendance-group)，获取单个考勤组的考勤组详情信息。
     + 根据groupKey获取考勤组详情

       1. 根据考勤组`id`，调用服务端API-[groupId转换为groupKey](/document/development/groupid-to-groupkey)接口，获取考勤组`groupKey`。
       2. 根据考勤组`groupKey`，调用服务端API-[根据groupKey查询考勤组信息](/document/development/queries-attendance-group-information-by-id)接口，获取考勤组信息。
3. 根据考勤组`id`，调用服务端API-[更新考勤组](/document/development/attendance-group-update-interface)，修改考勤组名称、考勤组班次等信息。
4. 根据考勤组`groupKey`，调用服务端API-[删除考勤组](/document/development/delete-attendance-group)接口，实现删除考勤组。

## **前提条件**

完成[创建应用](/document/dingstart/create-application)的流程。

## **步骤一：获取应用凭证**

1. 选择目标应用，进入应用详情页，单击**基础信息** > **凭证与基础信息**。
2. 获取应用 Client ID 和 Client Secret。

## 步骤二：添加接口权限

单击**开发配置** > **权限管理**，在权限搜索框中输入`qyapi_attendance_group_manage`和`qyapi_attendance_group_read`，并申请权限。

## 步骤三：获取应用访问凭证accessToken

**重要**

服务端API差异详情参见[旧版服务端API、新版服务端API的区别](/document/development/how-to-call-apis#section-8lr-id4-rbz)。以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK下载](/document/development/download-the-server-side-sdk)。

根据步骤一中 的 Client ID 和 Client Secret，获取应用访问凭证[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)。

```java
public void getAccessToken() throws ApiException {
        DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/gettoken");
        OapiGettokenRequest req = new OapiGettokenRequest();
        req.setAppkey("dingxxxxxxxxxhgn");
        req.setAppsecret("9G_xxxxxxxxxxxxxxx1JDf0Qq3nexxxxxxxxGIO");
        req.setHttpMethod("GET");
        OapiGettokenResponse rsp = client.execute(req);
        System.out.println(rsp.getBody());
    }
```
