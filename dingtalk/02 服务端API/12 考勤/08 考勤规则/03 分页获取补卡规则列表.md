更新于 2025-09-11调用本接口，分页获取考勤打卡中设置的补卡规则列表，包括规则名称和规则ID。

例如，企业的考勤规则管理内有两个补卡规则，分别为**测试补卡规则**和**默认补卡规则**，如下图所示。

调用本接口可获取两个补卡规则的规则名称和规则ID。

![[development-retrieve-a-list-of-replenishment-rules-by-page_p961085.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 考勤组查询权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=attendance_1.0%23GetAdjustments) |
| 第三方企业应用 | 支持 | 考勤组查询权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=attendance_1.0%23GetAdjustments) |
| 第三方个人应用 | 暂不支持 | 考勤组查询权限 | 暂不支持 |

## 请求方法

```
GET /v1.0/attendance/adjustments?pageNumber=Long&pageSize=Long HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| pageNumber | Long | 是 | 分页起始页。  **说明**  该参数值从1开始。 |
| pageSize | Long | 是 | 分页大小。  **说明**  该参数最大不能超过50。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Object | 获取的结果。 |
| pageNumber | Long | 当前页码。 |
| totalPage | Long | 总页数。 |
| items | Array | 补卡规则列表。 |
| id | Long | 补卡规则ID。 |
| name | String | 补卡规则名称。 |
| settingId | Long | 补卡规则settingId。 |
