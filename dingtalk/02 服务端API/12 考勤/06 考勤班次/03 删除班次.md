更新于 2025-09-10调用本接口，根据班次ID删除考勤班次。

调用本接口，可删除考勤班次，删除后该班次不在列表展示，如下图所示。

![[development-delete-shift_p961043.png]]

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 考勤组管理权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.attendance.shift.delete) |
| 第三方企业应用 | 否 | — | — |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/attendance/shift/delete`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | af21xxx | 调用该接口的应用凭证，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| op\_user\_id | String | 是 | user123 | 操作人userId。 |
| shift\_id | Number | 是 | 2423 | 班次ID，可通过[获取班次摘要信息](/document/development/enterprise-shift-query-in-batches)接口获取id参数值。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| errmsg | String | 系统错误 | 调用失败时返回的错误信息。 |
| errcode | Number | 0 | 返回码。 |
| success | Boolean | true | 调用是否成功。   * **true**：成功 * **false**：失败 |
| request\_id | String | 16apb7hw85jjw | 请求ID。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/attendance/shift/delete?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "op_user_id":"user456",
        "shift_id":"2423"
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/attendance/shift/delete");
OapiAttendanceShiftDeleteRequest req = new OapiAttendanceShiftDeleteRequest();
req.setOpUserId("user456");
req.setShiftId(2423L);
OapiAttendanceShiftDeleteResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

**返回示例**

```json
{
    "errcode": 0,
    "success": true,
    "request_id": "3lupjm5o0j7g"
}
```
