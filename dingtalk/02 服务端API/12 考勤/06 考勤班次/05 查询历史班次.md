更新于 2025-09-10调用本接口，根据班次ID和version查询历史班次信息。

调用本接口，可查询历史班次信息，如班次名称、班次设置、打卡时间、休息时间等。

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 是 | 考勤组查询权限 | [调试](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.attendance.shift.history.query) |
| 第三方企业应用 | 否 | — | — |
| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/attendance/shift/history/query`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | 73e9xxx | 调用该接口的应用凭证，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| op\_user\_id | String | 是 | user123 | 操作者userId。 |
| shift\_id | Number | 是 | 628380875 | 班次ID，可通过[获取班次摘要信息](/document/development/enterprise-shift-query-in-batches)接口获取id参数值。 |
| version | Number | 是 | 624190327 | 班次版本，可通过[批量查询人员排班信息](/document/development/query-batch-scheduling-information)接口获取shift\_version参数值。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/attendance/shift/history/query?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "op_user_id":"user456",
        "shift_id":715390132,
        "version":624190327
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/attendance/shift/history/query");
OapiAttendanceShiftHistoryQueryRequest req = new OapiAttendanceShiftHistoryQueryRequest();
req.setOpUserId("user456");
req.setShiftId(715390132L);
req.setVersion(624190327L);
OapiAttendanceShiftHistoryQueryResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```
