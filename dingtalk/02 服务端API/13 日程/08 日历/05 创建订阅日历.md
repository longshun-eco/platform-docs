更新于 2025-09-11调用本接口，创建订阅日历。

### 接口功能介绍

例如，员工的日历中，可订阅日历下有一个**淘票票电影日历**，如下图所示。

![[development-create-subscription-calendar_O1CN01YfjbII1YkTNl0NrkX_!!6000000003097-2-tps-2152-1416.png]]

调用本接口可创建一个名为**测试订阅**的日历。接口调用成功后，实现效果如下图所示。

![[development-create-subscription-calendar_O1CN01EUO2gd1CE1m7dXnz2_!!6000000000048-2-tps-2150-1412.png]]

### 接口调用说明

调用本接口创建订阅日历时，可指定可订阅该日历的员工、群组或组织。

在指定可订阅范围内的员工、群组或组织，可按照如下图所示步骤订阅该日历。

![[development-create-subscription-calendar_O1CN01eQ5WK11ehcbtQNjEM_!!6000000003903-2-tps-2464-1274.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=calendar_1.0%23CreateSubscribedCalendar) |
| 第三方企业应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=calendar_1.0%23CreateSubscribedCalendar) |
| 第三方个人应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=personal&api=calendar_1.0%23CreateSubscribedCalendar) |

## 请求方法

```
POST /v1.0/calendar/users/{userId}/subscribedCalendars HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "name" : "String",
  "description" : "String",
  "managers" : [ "String" ],
  "subscribeScope" : {
    "unionIds" : [ "String" ],
    "openConversationIds" : [ "String" ],
    "corpIds" : [ "String" ]
  }
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 * 第三方个人应用，调用[获取用户token](/document/development/obtain-user-token)接口获取。 |

## Path参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 日历创建者的unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| name | String | 是 | 订阅日历的名称。 |
| description | String | 否 | 订阅日历的描述，最大长度1024字符。 |
| managers | Array of String | 否 | 订阅日历共同编辑人的unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| subscribeScope | Object | 是 | 可订阅该日历的对象。 |
| unionIds | Array of String | 否 | 可订阅该日历的人员unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| openConversationIds | Array of String | 否 | 可订阅该日历的群组OpenConversationId列表。   * 企业内部应用，调用[获取群会话的OpenConversationId](/document/development/obtain-group-openconversationid)接口获取openConversationId参数值。 * 第三方企业应用，调用[创建群](/document/development/create-a-scene-group-v2)接口获取open\_conversation\_id参数值。   **说明**   * 日程创建者unionId要求在该参数openConversationIds群组内或者参数corpId组织下。 * 第三方个人应用无需传此参数。 |
| corpIds | Array of String | 否 | 可订阅该日历的组织corpId列表。 登录[钉钉开发者后台](https://open-dev.dingtalk.com/#/)，在首页下查看企业的corpId。 ![[development-create-subscription-calendar_O1CN019mpQUK1CNe7mDKcx8_!!6000000000069-2-tps-2716-1230.png]]  **说明**  日程创建者unionId要求在该corpId组织下或者参数openConversationIds群组内。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| calendarId | String | 可订阅日历的ID。 |
