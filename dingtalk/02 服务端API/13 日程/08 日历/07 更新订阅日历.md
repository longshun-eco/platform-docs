更新于 2025-09-11调用本接口，更新单个订阅日历信息。

例如，创建的订阅日历信息如下图所示，日历标题为**测试订阅**，可订阅人群为**小明**，描述和共同编辑人为空。

![[development-update-subscription-calendar_O1CN019tUxn21jHaAaHO26m_!!6000000004523-2-tps-2152-1410.png]]

调用本接口可更新该订阅日历信息。将日历标题更新为**测试订阅1**，描述更新为**测试描述**，共同编辑人添加了**小明**，可订阅人群添加了**测试群**群组。

接口调用成功后，该订阅日历信息如下图所示。

![[development-update-subscription-calendar_O1CN01MW1XPJ1z17n9rkJ02_!!6000000006653-2-tps-2144-1396.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=calendar_1.0%23UpdateSubscribedCalendars) |
| 第三方企业应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=calendar_1.0%23UpdateSubscribedCalendars) |
| 第三方个人应用 | 支持 | 日历应用中日历写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=personal&api=calendar_1.0%23UpdateSubscribedCalendars) |

## 请求方法

```
PUT /v1.0/calendar/users/{userId}/subscribedCalendars/{calendarId} HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "name" : "String",
  "description" : "String",
  "managers" : [ "String" ],
  "subscribeScope" : {
    "unionIds" : [ "String" ],
    "openConversationIds" : [ "String" ],
    "corpIds" : [ "String" ]
  }
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 * 第三方个人应用，调用[获取用户token](/document/development/obtain-user-token)接口获取。 |

## Path参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| calendarId | String | 是 | 订阅日历id，可调用[创建订阅日历](/document/development/create-subscription-calendar)接口获取calendarId参数值。 |
| userId | String | 是 | 日历作者或共同编辑人的unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| name | String | 否 | 订阅日历的标题。 |
| description | String | 否 | 订阅日历的描述。 |
| managers | Array of String | 否 | 日历共同编辑人的unionId列表。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| subscribeScope | Object | 否 | 日历的可订阅范围。 |
| unionIds | Array of String | 否 | 可订阅该日历的人员unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| openConversationIds | Array of String | 否 | 可订阅该日历的群OpenConversationId列表。   * 企业内部应用，调用[获取群会话的OpenConversationId](/document/development/obtain-group-openconversationid)接口获取openConversationId参数值。 * 第三方企业应用，调用[创建群](/document/development/create-a-scene-group-v2)接口获取open\_conversation\_id参数值。   **说明**   * 需要传该订阅日程创建者所在的群OpenConversationId。 * 第三方个人应用无需传此参数。 |
| corpIds | Array of String | 否 | 可订阅该日历的组织corpId列表。 登录[钉钉开发者后台](https://open-dev.dingtalk.com/#/)，在首页下查看企业的corpId。 ![[development-update-subscription-calendar_O1CN019mpQUK1CNe7mDKcx8_!!6000000000069-2-tps-2716-1230.png]]  **说明**  需要传该订阅日程创建者所在的组织corpId。 |
