更新于 2025-09-11调用本接口，查询一个用户给定时间范围内的日程，支持翻页和增量查询。

**说明**

* 对于循环日程，会返回所有过去的循环日程实例，但是未来的不会返回，例如有一个从1.1号开始的每日循环日程，查询时已经是1.4号，那么会返回1.1～1.4号的循环日程实例，但是不会返回1.4号之后的 。
* 可以根据返回的日程对象recurrenceId是不是为空，来判断该日程是否为一个循环日程的实例。
* 默认不会返回已删除的日程，如果需要返回可以设置`showDeleted=true`，当返回的是已删除的日程时，此时调用[查询单个日程详情](/document/development/query-details-about-an-event)接口会返回**instanceNotExist**的错误。

调用本接口，查询某用户在某段时间内的日程列表，可以获取到日程标题、日程的开始日期、开始时间、结束日期、结束时间等信息。

| 第三方企业应用 | 支持 | 日历应用中日程读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=calendar_1.0%23ListEvents) |

## 请求方法

```
GET /v1.0/calendar/users/{userId}/calendars/{calendarId}/events?timeMin=String&timeMax=String&showDeleted=Boolean&maxResults=Integer&maxAttendees=Integer&nextToken=String&syncToken=String HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 * 第三方个人应用，调用[获取用户token](/document/development/obtain-user-token)接口获取。 |

## Path参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 查询目标用户的unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| calendarId | String | 是 | 日程所属的日历id，统一为primary，表示用户的主日历。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| timeMin | String | 否 | 日程开始时间的最小值，格式为ISO-8601的date-time格式，可不填。  **说明**  `timeMin`和 `timeMax`最大差值为一年。 |
| timeMax | String | 否 | 日程开始时间的最大值，格式为ISO-8601的date-time格式，可不填。  **说明**  `timeMin`和 `timeMax`最大差值为一年。 |
| showDeleted | Boolean | 否 | 是否返回已删除的日程：   * **true**：返回 * **false**（默认）：不返回 |
| maxResults | Integer | 否 | 最大返回记录数，最大值100，默认值100。 |
| maxAttendees | Integer | 否 | 每个日程的参与者查询个数，默认100，最大100。 如果参会人数超过100人，需要拉取全部参会人请使用[获取日程参与者](/document/development/get-the-participants-of-a-schedule)接口获取。 |
| nextToken | String | 否 | 分页游标。  **说明**  如果一个查询条件一次无法全部返回结果，会返回分页token，下次查询带上该token后会返回后续数据，直到分页token为null表示数据已经全部查询完毕。 |
| syncToken | String | 否 | 同步token，用于增量数据同步场景，一个查询条件已经返回全部数据后（如果数据比较多会分页返回，调用方需要连续使用nextToken查询，直到所有分页都查完nextToken返回null之后），会返回一个syncToken，下次带着该token来查询会返回两次查询之间发生变更的增量数据。  **说明**  如果传入了timeMax，则不会返回syncToken。 |
| seriesMasterId | String | 否 | 重复日程的主日程id，非重复日程为空。 |
