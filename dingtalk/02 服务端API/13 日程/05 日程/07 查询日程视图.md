更新于 2025-09-11调用本接口，查询用户的钉钉主日历在某时间范围内的日程视图（循环日程展开）。

**说明**

* 本接口与**查询日程列表**接口的区别在于对循环日程的处理。本接口会根据循环规则展开查询区间内的所有循环日程，但不会实际在数据库中生成对应实例。
  例如，有一个从1.1号开始的每日循环日程，查询时已经是1.4号，查询区间为1.1号至1.10号，那么本接口会返回1.1～1.10号的所有循环日程，而[查询日程列表](/document/development/query-an-event-list)接口是不会返回1.4号之后的日程。
* 可以根据返回的日程对象seriesMasterId是否为空，来判断该日程是否为一个循环日程的实例。

调用本接口，查询某用户在某段时间内的日程列表，可以获取到日程标题、日程的开始日期、开始时间、结束日期、结束时间等信息。

| 第三方企业应用 | 支持 | 日历应用中日程读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=calendar_1.0%23ListEventsView) |

## 请求方法

```
GET /v1.0/calendar/users/{userId}/calendars/{calendarId}/eventsview?timeMin=String&timeMax=String&maxResults=Long&nextToken=String HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 * 第三方个人应用，调用[获取用户token](/document/development/obtain-user-token)接口获取。 |

## Path参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 要查询的用户unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| calendarId | String | 是 | 日程所属的日历ID，调用[查询日历](/document/development/query-a-calendar)接口查询当前用户日程，使用**primary**关键字，表示当前用户的主日历。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| timeMin | String | 否 | 查询日程结束时间 >= TimeMin，格式为ISO-8601的date-time格式。 |
| timeMax | String | 否 | 查询日程开始时间 < timeMax，格式为ISO-8601的date-time格式。 |
| maxResults | Integer | 否 | 最大返回记录数，最大值100，默认值100。 |
| maxAttendees | Integer | 否 | 每个日程的参与者查询个数，默认100，最大100。 如果参会人数超过100人，需要拉取全部参会人请使用[获取日程参与者](/document/development/get-the-participants-of-a-schedule)接口获取。 |
| nextToken | String | 否 | 分页游标。  **说明**  如果一个查询条件一次无法全部返回结果，会返回分页token，下次查询带上该token后会返回后续数据，直到分页token为null表示数据已经全部查询完毕。 |
