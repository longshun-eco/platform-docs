更新于 2025-09-11调用本接口，从日程参与者中删除指定的用户。

**说明**

每次日程参与者操作最大支持500人，最大支持操作5000人的日程。

例如，某日程参与者有3人，如下图所示。

![[development-delete-schedule-participant_O1CN010lfcLK27ScWgt8Ozm_!!6000000007796-2-tps-1474-568.png]]

调用本接口，可从日程参与者中删除指定的用户。接口调用成功后，日程参与者为2人，如下图所示。

![[development-delete-schedule-participant_O1CN01esPrNv1Z9fC6fdqhf_!!6000000003152-2-tps-1464-508.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 日历应用中日程写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=calendar_1.0%23RemoveAttendee) |
| 第三方企业应用 | 支持 | 日历应用中日程写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=calendar_1.0%23RemoveAttendee) |
| 第三方个人应用 | 支持 | 日历应用中日程写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=personal&api=calendar_1.0%23RemoveAttendee) |

## 请求方法

```
POST /v1.0/calendar/users/{userId}/calendars/{calendarId}/events/{eventId}/attendees/batchRemove HTTP/1.1
Host:api.dingtalk.com
x-client-token:String
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "attendeesToRemove" : [ {
    "id" : "String"
  } ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-client-token | String | 否 | 幂等校验。  **说明**   * 相同的`x-client-token`表示同一次请求。 * 过期失效，1天。 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 * 第三方个人应用，调用[获取用户token](/document/development/obtain-user-token)接口获取。 |

## Path参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 日程创建者unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
| calendarId | String | 是 | 日程所属的日历id，统一为primary，表示用户的主日历。 |
| eventId | String | 是 | 日程ID，可调用[创建日程](/document/development/create-event)接口或[查询日程列表](/document/development/query-an-event-list)接口获取id参数值。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| attendeesToRemove | Array | 否 | 需要被删除的日程参与者列表。 |
| id | String | 否 | 用户unionId。   * 企业内部应用和第三方企业应用，调用[查询用户详情](/document/development/query-user-details)接口获取unionid参数值。 * 第三方个人应用，调用[获取用户通讯录个人信息](/document/development/dingtalk-retrieve-user-information)接口获取unionId参数值。 |
