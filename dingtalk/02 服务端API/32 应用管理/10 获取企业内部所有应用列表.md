更新于 2026-01-22通过本接口可获取企业内部所有自建应用的详细信息，包括应用名称、描述、图标、访问地址等，便于企业管理员统一管理应用资产。

### 使用场景

企业管理员在以下场景中可使用该接口：

* 需要查看企业内所有自建应用的列表及其当前状态（如启用或停用）时，可通过该接口快速获取完整清单。
* 在进行企业应用资产管理与维护时，可利用该接口了解应用分布情况，辅助制定应用整合或优化策略。
* 当需要批量获取应用的基础信息（如名称、描述、访问链接、图标）用于系统对接或展示时，该接口提供标准化数据输出。
* 在实施应用生命周期管理过程中，监控各应用的启用状态，及时识别并处理已停用或异常的应用实例，保障企业应用环境的安全与合规。

上述能力为企业实现集中化、可视化的应用治理提供了技术支撑。

### 接口功能介绍

调用本接口，可从钉钉[开发者后台](https://open-dev.dingtalk.com/fe/app#/corp/app) 的企业内部开发应用列表中，获取当前企业下所有自建应用的详细信息。相关信息包括应用AgentId、名称、描述、图标、移动端和PC端首页链接、管理后台地址、应用状态及类型等。具体如下图：

## 权限

要调用此API，需要以下权限之一。

| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| --- | --- | --- | --- |
| 企业内部应用 | 支持 | 企业已安装的应用列表查询权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=microApp_1.0%23ListAllInnerApps) |
| 第三方企业应用 | 暂不支持 | 暂不支持 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
GET /v1.0/microApp/allInnerApps HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](https://open.dingtalk.com/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## 返回参数

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| appList | Array | 应用列表。 |
| agentId | Long | 应用AgentId。 |
| name | String | 应用名称。 |
| desc | String | 应用描述。 |
| icon | String | 应用图标。 |
| homepageLink | String | 应用移动端首页地址。 |
| pcHomepageLink | String | 应用PC端首页地址。 |
| ompLink | String | 应用管理后台地址。 |
| appId | Long | 应用ID。  **说明**  企业内部应用，AppId值是0。 |
| appStatus | Integer | 应用状态，取值：   * **0**：停用 * **1**：启用 |
| developType | Integer | 应用类型：   * **0**：表示H5微应用。 * **1**：表示小程序。 |
| robotInfo | Object | 应用下的机器人信息 |
| name | String | 机器人名称。 |
| robotCode | String | 机器人code。 |
| coolAppInfo | Array | 应用下的所有已发布酷应用列表。 |
| name | String | 酷应用名称。 |
| coolAppCode | String | 酷应用code。 |
