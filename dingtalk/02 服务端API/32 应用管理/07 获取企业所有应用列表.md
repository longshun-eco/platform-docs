更新于 2025-09-23调用本接口，获取企业所有应用的信息，包括应用名称、应用描述、应用图标、应用访问地址等。

### 应用场景

本接口适用于企业管理员需要获取企业应用完整信息的场景。例如：

* 需要查看企业所有已安装应用的列表和状态时；
* 需要了解企业应用资产分布，进行应用管理和维护时；
* 需要获取应用的基础信息（名称、描述、图标、访问地址等）时；
* 需要监控应用状态（启用、停用、过期）进行应用生命周期管理时。

通过本接口，管理员可获取企业所有应用的详细信息，包括基础应用、自建应用和第三方企业应用，为应用管理和决策提供数据支持。

### 接口功能介绍

例如，调用本接口可以获取[企业管理后台](https://oa.dingtalk.com/)-应用管理列表页，所有基础应用、自建应用和第三方企业应用的应用信息，如应用名称、应用描述、应用图标等。

只有以下情况，在企业管理后台-应用管理列表页查看应用信息列表时，和接口调用的应用列表相同：

* 如果是企业主管理员，在企业管理后台-应用管理列表页，可以查看到所有的应用信息。
* 如果是企业子管理员，必须同时拥有全部应用管理权限，在企业管理后台-应用管理列表页，可以查看所有应用的信息。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业已安装的应用列表查询权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=microApp_1.0%23ListAllApp) |
| 第三方企业应用 | 暂不支持 | 暂不支持 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
GET /v1.0/microApp/allApps HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| appList | Array | 应用列表。 |
| agentId | Long | 应用AgentId。 |
| name | String | 应用名称。 |
| desc | String | 应用描述。 |
| icon | String | 应用图标。 |
| homepageLink | String | 应用移动端首页地址。 |
| pcHomepageLink | String | 应用PC端首页地址。 |
| ompLink | String | 应用管理后台地址。 |
| appId | Long | 应用ID。   * 基础应用，返回应用对应的AppId值。 * 企业自建应用，AppId值是0。 * 第三方企业应用，返回应用对应的AppId值。 |
| appStatus | Integer | 应用状态，取值：   * **0**：停用 * **1**：启用 * **3**：过期   **说明**  过期是指企业开通第三方企业应用，超出了开通规格内的服务时长。 |
| developType | Integer | 应用类型。   * 0：表示H5微应用。 * 1：表示小程序。 |
