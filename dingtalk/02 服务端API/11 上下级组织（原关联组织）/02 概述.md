更新于 2025-09-10上下级组织是钉钉推出一项有关组织架构管理的新功能，旨在帮助大型组织解决人员臃肿带来的管理难题，实现管理职责下放，同时与上级实现跨组织的沟通、协同。

## 什么是上下级组织

钉钉上的组织（企业、团队）可以通过上下级组织功能，实现“集团总公司/分公司”架构，完成“上、下级单位”互联。

以集团公司场景举例：

每个分公司作为【分支组织】，建立独立的钉钉组织。集团总公司作为【主干组织】，可以邀请分公司进行关联。

关联成功后，分支组织成员（全部或部分）将自动同步到主干组织的通讯录架构，主干成员可以直接和分支成员相互沟通。分支组织成员可以访问主干工作台，使用应用，实现协同。后续分公司的人员变动也将自动与集团架构同步，无须在集团和分公司分别维护架构人员。

| 原来 | 现在 |
| --- | --- |
| 组织庞大，全部人员集中管理，主管理员疲于奔命 | 下级单位作为独立组织，关联到上级单位，管理职责下放，大幅提升效率 |
| 下级单位作为部门，无法自建工作台，无法设置审批考勤，缺乏灵活自主性 | 下级单位 自主管理通讯录、工作台，灵活自治 |
| 上级单位联系下级单位、合作伙伴，还需先加好友 | 上级单位、下级单位、合作伙伴——实现跨组织通讯录互通，不加好友直接沟通 |

## 如何开通上下级组织功能

1. 组织管理员登录[OA管理后台](https://oa.dingtalk.com/index.htm#/service/market)。
2. 单击**增值服务**，在产品专区，找到上下级组织功能，然后单击**立即购买**。

## 常见问题

1. **什么是主干组织、分支组织？**

   在上下级组织中：

   * 上级单位、总公司 统称为主干组织
   * 下级单位、分公司 统称为分支组织
2. **使用上下级组织是否需要主干/分支组织均为认证组织？**

   目前非认证组织也可以申请。
3. **2个组织成功关联后，会有什么效果？**

   * 主干组织：基于权限控制，主干成员可以在通讯录中看到关联过来的分支人员。
   * 分支组织：被关联进入主干组织的成员，可在主干通讯录查看到权限范围内可见的人员，访问主干的工作台、使用应用。
4. **关联成功后，分支组织如果想修改关联到主组织的部门，可以修改吗？**

   可以，如果分支组织开通了上下级组织，可在【管理】-【组织关联】-【我是分支】路径进入修改。

   如果分支组织未开通上下级组织，可在关联成功后【钉钉管理助手】推送的关联成功信息点击进入修改。
5. **2个组织成功关联后，分支成员加入主干组织架构，可以实现哪些场景？**

   |  |  | | --- | --- | | **场景**  **和功能** | **分支成员在主干架构中可以** | | 第三方加密 | 主干组织钉钉架构中开启第三方加密后，分支组织成员可以在主干组织中聊天。第三方加密的费用的人数计算按照关联后的整体人数进行计算。 | | 创建群聊 | 可以与主干组织的成员创建内部群聊。 | | 工作台 | 分支组织成员能看到主干组织的工作台。 | | 通讯录 | 分支成员能查看主干组织通讯录，主干可设置其可见范围。 | | 角色 | 可为分支组织成员单独设置角色。 | | 审批 | 分支组织成员能够发起主干审批流、能作为审批节点，并且能针对分支成员设置审批可见范围。分支组织发起工作流不可添加主干成员。 | | 公告 | 分支成员能够接收主干组织发出的公告。 | | DING | 主干、分支成员能够相互发DING，分别使用各自的额度。 | | 微应用 | 分支组织成员可以使用主干组织的微应用，主干可设置其可见范围。 | | 企业认证 | 主干组织和分支组织需单独认证。 | | 主企业 | 分支组织成员不能选择主干组织为主企业。 | | 人数统计 | 暂不支持对上下级组织的人数进行统计。 |
6. **主干组织能否看到分支组织的工作台？**

   不可以。只支持分支看主干的工作台。
7. **分支组织之间，是否可以设置相互不可见？**

   可以。主干管理员可以灵活设置分支之间的相互可见性。
8. **关联时，分支组织是否可以只让部分成员加入主干？**

   目前已支持4层关联，A关联B，B关联C，C关联D。例如：省教育厅-市教育局-区教育局-学校。
9. **分支组织是否可以关联到多个主干组织？**

   可以，一个分支组织可以关联/挂靠到5个主干组织，不管是否审批通过、最多发起5个请求。
10. **一个主干组织可以关联多少分支组织？**

    一个主干可以关联300分支组织。
11. **主干组织能否管理、编辑分支组织的部门、人员？**

    不可以。
12. **主干组织工作台的应用，能否设置不被分支人员访问?**

    可以针对每个应用，单独设置访问权限，限制分支人员查看。
13. **子管理员可以进行组织关联操作吗？**

    具有上下级组织权限点的子管理员可以进行关联操作。
14. **关联之后可以解除关联吗？**

    可以。解除关联后，分支人员全部脱离主干通讯录，无法再访问主干工作台。
15. **分支组织的人员变动了，会实时同步到主干组织吗？**

    会。分支组织独立维护自己的通讯录，自动实时同步到主干。
16. **同一个员工，是否可以同时加入主干、分支2个组织？**

    可以。
17. **分支组织能否覆盖主干的某个部门，还是只支持挂在主干的某个部门下？**

    只支持挂在主干组织的某个部门下，不能覆盖部门。
18. **主干组织、分支组织之间，如果存在人员调动，是否可以组织内调整？**

    不支持，目前只能离职再入职。
19. **分支组织发起工作流是否可以选择到主干中的成员？**

    不可以，只有在主干中发起的工作流可以选到包括主干和分支的成员，可以理解为分支组织成员加入了主干组织。
20. **分支组织成员是否可以加入主干成员内部群？**

    可以。
