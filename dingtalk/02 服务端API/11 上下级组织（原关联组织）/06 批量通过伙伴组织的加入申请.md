更新于 2025-10-21调用本接口，批量通过分支组织加入主干组织申请。

例如，某主干组织的通讯录目前没有任何分支组织关联，如下图所示。

![[development-batch-through-the-application-of-partner-organizations-to-join-contact_p449716.png]]

该组织收到两条分支组织的关联申请，分别为**关联组织**和**体验组织**，申请信息如下图所示。

![[development-batch-through-the-application-of-partner-organizations-to-join-contact_p449717.png]]

调用本接口可同时同意这两条关联主干组织的申请。接口调用成功后，主干组织的通讯录新增了**关联组织**和**体验组织**，如下图所示。

![[development-batch-through-the-application-of-partner-organizations-to-join-contact_p449718.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录合作空间写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23BatchApproveUnionApply) |
| 第三方企业应用 | 支持 | 通讯录合作空间写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=contact_1.0%23BatchApproveUnionApply) |
| 第三方个人应用 | 暂不支持 | 通讯录合作空间写权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/contact/cooperateCorps/unionApplications/approve HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

[ {
  "branchCorpId" : "String",
  "unionRootName" : "String",
  "linkDeptId" : Long
} ]
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
|  | Array | 否 | 需要处理的申请列表，一次请求最多处理10个申请。 |
| branchCorpId | String | 否 | 申请的分支组织CorpId，详情参见[CorpId](/document/development/development-basic-concepts#section-bbk-mv0-oxd)。 |
| unionRootName | String | 否 | 分支组织在主干组织内的名称。 |
| linkDeptId | Long | 否 | 分支组织在主干组织内的位置，可调用[获取部门列表](/document/development/obtain-the-department-list-v2)接口获取dept\_id参数值。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Boolean | 是否处理成功。   * **true**：成功 * **false**：失败 |
