更新于 2025-09-11面向第三方个人应用调用本接口向指定的用户发送服务窗消息。

**说明**

* 调用此接口前需要先通过授权组件获得用户`OfficialAccount.SnsMessage.Send`授权，获取授权code方式请参考[钉钉统一授权套件](/document/development/function-description)。
* 此接口一天最多允许调用次数等于服务窗粉丝数量。
* 每位粉丝用户一天最多允许接收三条来自服务窗的消息（包括服务窗后台群发、批量发送接口及单人消息接口）。
* 服务窗为减少内容相同消息对用户的打扰，默认场景下会对相同内容的消息推送会进行前去重处理，相同内容消息同一用户一天内仅会收到一条。
* 目前支持的消息格式有文本、链接、卡片、markdown等。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 暂不支持 | 服务窗个人消息发送权限 | 暂不支持 |
| 第三方企业应用 | 暂不支持 | 服务窗个人消息发送权限 | 暂不支持 |
| 第三方个人应用 | 支持 | 服务窗个人消息发送权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=personal&api=crm_1.0%23SendOfficialAccountSNSMessage) |

## 请求方法

```
POST /v1.0/crm/officialAccounts/snsMessages/send HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "detail" : {
    "msgType" : "String",
    "uuid" : "String",
    "messageBody" : {
      "text" : {
        "content" : "String"
      },
      "markdown" : {
        "title" : "String",
        "text" : "String"
      },
      "link" : {
        "picUrl" : "String",
        "messageUrl" : "String",
        "title" : "String",
```
