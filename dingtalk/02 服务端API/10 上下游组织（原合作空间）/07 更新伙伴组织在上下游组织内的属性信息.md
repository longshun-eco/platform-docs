更新于 2025-09-11调用本接口，更新伙伴组织在上下游组织内内的属性信息，包括新伙伴组织的别名，新伙伴组织在上下游组织内的通讯录位置等。

**说明**

如果更新伙伴组织在通讯录的位置是一级目录，调用本接口的挂载部门参数linkDeptId，需要传-1。

例如，测试组织演示是上下游组织钉钉上下游联盟内的伙伴组织，调用本接口可以更新测试组织演示的别名和在通讯录内的位置，如图。

![[development-update-properties-of-branches-in-alibaba-group-1_O1CN01kAYPkt24zBn8b0ua4_!!6000000007461-2-tps-1192-866.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 通讯录合作空间写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=contact_1.0%23UpdateBranchAttributesInCooperate) |
| 第三方企业应用 | 支持 | 通讯录合作空间写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=contact_1.0%23UpdateBranchAttributesInCooperate) |
| 第三方个人应用 | 暂不支持 | 通讯录合作空间写权限 | 暂不支持 |

## 请求方法

```
PUT /v1.0/contact/cooperateCorps/branchAttributes HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

[ {
  "branchCorpId" : "String",
  "unionRootName" : "String",
  "linkDeptId" : Long
} ]
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。   **说明**  调用本接口，需要使用上下游组织的访问凭证，不能使用所属组织的访问凭证。 |

|  |  |  |  |
