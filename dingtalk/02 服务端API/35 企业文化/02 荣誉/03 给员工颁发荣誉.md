更新于 2026-03-14调用本接口，用于给组织内的员工颁发荣誉。

在荣誉应用页面，左侧选择**荣誉 > 荣誉颁发**，单击**颁发荣誉**。调用本接口与下图操作实现效果一致。

![[development-award-of-honor_O1CN01UjjJ8F1lAN9ithrSz_!!6000000004778-2-tps-2280-918.png]]

例如，给员工小明颁发**创造之星**荣誉，小明收到荣誉推送信息后，点击立即佩戴。佩戴完成后，小明的头像上会带上**创造之星**的挂件。如下图所示。

![[development-award-of-honor_O1CN01ojF22i1YhGbDbJfFi_!!6000000003090-2-tps-2288-1330.png]]

员工小明在钉钉客户端点击头像，可查看佩戴中的荣誉挂件。如下图所示。

![[development-award-of-honor_O1CN01qduR061NFp5lUmrMn_!!6000000001541-2-tps-1564-934.png]]

**说明**

同一个企业corpid、同一个颁发人userId，不允许并发执行，防止颁发顺序出错。

## 权限

要调用此API，需要以下权限之一。

| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| --- | --- | --- | --- |
| 企业内部应用 | 支持 | 组织文化荣誉信息写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=orgCulture_1.0%23GrantHonor) |
| 第三方企业应用 | 支持 | 组织文化荣誉信息写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=orgCulture_1.0%23GrantHonor) |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/orgCulture/honors/{honorId}/grant HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "senderUserId" : "String",
  "grantReason" : "String",
  "granterName" : "String",
  "expirationTime" : Long,
  "noticeSingle" : Boolean,
  "noticeAnnouncer" : Boolean,
  "receiverUserIds" : [ "String" ],
  "openConversationIds" : [ "String" ]
}
```

## Header参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](https://open.dingtalk.com/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](https://open.dingtalk.com/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Path参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| honorId | String | 是 | 荣誉Id，可调用[查询当前企业下可颁发的荣誉列表](https://open.dingtalk.com/document/development/query-the-list-of-honors-that-can-be-issued-under)接口获取honorId参数值。 |

## Body参数

| 名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| senderUserId | String | 是 | 发送人userId。  **说明**  如果是同一个发送人给多个员工颁发荣誉，请分别按照顺序执行，不允许并发执行，防止颁发顺序出错。 |
| grantReason | String | 是 | 颁奖词，最多100个字符。 |
| granterName | String | 是 | 颁奖人名字，最多50个字符。 |
| expirationTime | Long | 否 | 荣誉有效期到期时间戳，单位毫秒。   * 该参数值可不传，代表永久有效。 * 该参数值不允许传当天的时间戳。   **说明**  有效期时间范围要求1~366天后，例如调用本接口的时间为2022-01-01 12:00:00，有效期时间范围是2022-01-01 12:00:00——2023-01-02 12:00:00。 |
| noticeSingle | Boolean | 否 | 是否发送单聊通知。   * **true**：发送 * **false**：不发送   **说明**  如果该参数传true，荣誉接收人会收到发送人的单聊消息。 ![[development-award-of-honor_O1CN01qf032m1oUTNxPr3vj_!!6000000005228-2-tps-2292-1072.png]] |
| noticeAnnouncer | Boolean | 否 | 是否使用官宣号通知获奖人。   * **true**：通知 * **false**：不通知   **说明**  如果该参数传true，荣誉接收人会收到工作通知消息。 ![[development-award-of-honor_O1CN013QoXEg1zu5MSgaqQJ_!!6000000006773-2-tps-728-1196.png]] |
| receiverUserIds | Array of String | 是 | 接受人userId列表，最大值10。 |
| openConversationIds | Array of String | 否 | 接收荣誉消息的群openConversationId列表，最大值。   * 企业内部应用，可调用[创建群](https://open.dingtalk.com/document/development/session-management-creates-groups)接口获取openConversationId。 * 第三方企业应用，可调用[创建群](https://open.dingtalk.com/document/development/create-a-scene-group-v2)接口获取openConversationId。   **说明**  荣誉发送人senderUserId，必须是群成员。 |

## 返回参数

| 名称 | 类型 | 描述 |
| --- | --- | --- |
| success | Boolean | 是否调用成功，true表示调用成功。 |
| result | Object | 响应结果。 |
| successUserIds | Array of String | 返回成功的userId。 |
| failedUserIds | Array of String | 返回失败的userId。 |
