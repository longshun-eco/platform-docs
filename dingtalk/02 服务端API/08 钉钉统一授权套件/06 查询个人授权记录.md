更新于 2026-01-23当需要查询用户的授权记录信息时，可以调用本接口获取用户的个人授权详情。该接口适用于用户隐私权限审计、第三方应用授权状态同步等场景，帮助开发者在用户登录后检查其已授予的敏感权限列表，用于前端界面展示或安全合规审查。

例如下图所示，用户已授权手机号和个人基本信息两项权限，调用本接口可获取当前用户已授权的具体权限点，返回结果如下：

`{ "result": [ { "resource": "rpc_auth", "authItems": [ "Contact.User.Read" ] }, { "resource": "Contact.User", "authItems": [ "mobile" ] } ] }`

## **使用场景**

* **用户登录后权限校验**：在用户完成登录流程后，调用本接口检查其已授权的敏感数据权限（如手机号、组织架构信息），用于控制功能可见性或数据访问范围。
* **隐私合规审计**：企业安全团队可通过定期调用此接口，收集员工对第三方应用的授权情况，确保符合内部信息安全策略。
* **第三方应用授权管理**：第三方企业应用可在设置页中展示用户当前授权状态，提示用户是否已开启必要权限，提升用户体验与透明度。

| 第三方企业应用 | 支持 | 调用SNS API时需要具备的基本权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=oauth2_1.0%23GetPersonalAuthRule) |
| 第三方个人应用 | 支持 | 调用SNS API时需要具备的基本权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=personal&api=oauth2_1.0%23GetPersonalAuthRule) |

## 请求方法

```
GET /v1.0/oauth2/authRules/user HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 获取用户个人身份访问凭证，调用[获取用户token](https://open.dingtalk.com/document/development/obtain-user-token)接口获取。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Array | 授权记录列表。 |
| resource | String | 授权实体。 |
| authItems | Array of String | 授权项。 |
