更新于 2025-12-26调用本接口，获取企业组织公告的钉盘空间信息，用于发公告时可携带附件信息。

### 接口调用流程

步骤一：调用本接口，获取上传附件需要的 `spaceId`参数值。

步骤二：为当前操作人添加权限，需要传入步骤一中的`spaceId`。

* 企业内部应用，调用[添加权限](/document/development/add-permissions-file)接口进行授权。注意上传操作需要**EDITOR**（编辑者）及以上的权限，不能是**DOWNLOADER**或者**READER**权限。

步骤三：获取附件上传信息并完成上传，拿到附件id。

* 企业内部应用，参考[获取文件上传信息](/document/development/obtain-file-upload-informations)接口文档「接口调用流程」中描述的三个步骤，完成附件上传。上传完成后接口返回参数中文件信息的`id`就是附件id。

步骤四：调用创建公告接口，传入附件id，进行公告发送。

* 企业内部应用，调用[创建公告](/document/development/create-an-enterprise-announcement)接口进行公告发送。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 读取钉钉公告微应用数据的权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=blackboard_1.0%23QueryBlackboardSpace) |
| 第三方企业应用 | 暂不支持 | 读取钉钉公告微应用数据的权限 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 读取钉钉公告微应用数据的权限 | 暂不支持 |

## 请求方法

```
GET /v1.0/blackboard/spaces?operationUserId=String HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的应用凭证，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| operationUserId | String | 是 | 操作人userId。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| spaceId | String | 当前组织的公告对应的钉盘空间id。 |
