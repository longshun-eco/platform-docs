## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 智能人事待入职员工管理权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=hrm_1.0%23AddHrmPreentry) |
| 第三方企业应用 | 暂不支持 | 智能人事待入职员工管理权限 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 智能人事待入职员工管理权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/hrm/preentries HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "preEntryTime" : Long,
  "name" : "String",
  "mobile" : "String",
  "agentId" : Long,
  "groups" : [ {
    "groupId" : "String",
    "sections" : [ {
      "oldIndex" : Integer,
      "empFieldVOList" : [ {
        "value" : "String",
        "fieldCode" : "String"
      } ]
    } ]
  } ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| preEntryTime | Long | 否 | 预计入职时间戳，单位毫秒。 |
| name | String | 是 | 待入职员工的姓名。 |
| mobile | String | 是 | 待入职员工的手机号。 |
| agentId | Long | 否 | 应用的agentId，请参考[基础概念-AgentId](/document/development/basic-concepts-beta#884d363067bnq)。 |
| groups | Array | 否 | 待入职员工花名册分组列表，建议不超过5个。 |
| groupId | String | 否 | 分组ID，可调用[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口获取group\_id参数值。 |
| sections | Array | 否 | 分组内字段列表，建议不超过5个。 |
| oldIndex | Integer | 否 | 当前字段在分组内的序列号。 |
| empFieldVOList | Array | 否 | 分组内字段信息列表，建议不超过5个。 |
| value | String | 否 | 分组内字段的值。 |
| fieldCode | String | 否 | 分组内字段fieldCode，可调用[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口获取的field\_code参数值。 |
| needSendPreEntryMsg | Boolean | 否 | 是否发送完善入职登记表的IM消息给员工本人：   * true：发送 * false：不发送 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| tmpUserId | String | 待入职员工的的userId。  **说明**  待入职员工的userId为临时userId，正式入职后该员工userId会发生变化。 |
