更新于 2025-09-10调用本接口，更新员工档案信息，支持明细分组。

## **接口功能介绍**

**说明**

* 调用本接口可更新员工档案信息。
* 该接口如果传入的字段是不支持更新的字段（如sys00分組内的字段，部门和直属主管），会返回 *參数错误* 的报错信息。

以更新员工花名册内的身份证姓名为例，员工小钉在智能人事花名册内的身份证姓名是**小钉**，如下图所示。

![[development-intelligent-personnel-update-employee-file-information_p449990.png]]

调用本接口可将员工小钉的身份证姓名更新为**小钉1**。接口调用成功后，个人信息如下图所示。

![[development-intelligent-personnel-update-employee-file-information_p449993.png]]

## 权限

|  |  |  |  |

| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |

| 第三方个人应用 | 否 | — | — |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/smartwork/hrm/employee/v2/update`

## Query参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| access\_token | String | 是 | af21axxx | 调用该API的应用凭证。   * 企业内部应用，通过[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)接口获取。 * 第三方企业应用，通过[获取第三方企业的access\_token](/document/development/obtain-isvapp-token)接口获取。 |

## Body参数

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 示例值 | 描述 |
| agentid | Number | 是 | 23470561 | 应用的AgentId。   * 企业内部应用，应用详情页获取[应用 AgentId](https://open.dingtalk.com/document/orgapp/basic-concepts-beta#813cbd7067yn0)。 * 第三方企业应用，通过[获取企业授权信息](/document/development/obtains-the-basic-information-of-an-enterprise)接口获取agentid传说中。 |
| param | EmpUpdateByCustomParam | 是 |  | 员工信息。 |
| groups | EmpGroupFieldVo[] | 否 |  | 花名册分组。 |
| sections | EmpListFieldVo[] | 否 |  | 分组下明细，非明细分组仅一条明细。 |
| section | EmpFieldVo[] | 否 |  | 分组下字段列表。 |
| field\_code | String | 否 | sys01-birthTime | 更新的字段code，调用[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口获取field\_code参数值。  **说明**   * sys00分组内字段code值为汉字，如姓名，更新时直接传入**"姓名"**作为该参数值。 * sys00分组内的字段，**部门**和**直属主管**暂不支持接口更新。 |
| value | String | 否 | 2020-10-10 | 更新的字段值。   * 如果field\_code对应字段的field\_type为**TextField**，该字段值自定义。 * 如果field\_code对应字段的field\_type为**DDDateField**，该字段值传日期格式为"yyyy-MM-dd"。 * 如果field\_code对应字段的field\_type为**DDSelectField**，该字段值传option\_text内的value。结果显示为value对应的label值。  **说明**  例如“民族”字段获取的元数据信息如截图，此时value传1，该员工的民族会更改为“汉族”。 ![[development-intelligent-personnel-update-employee-file-information_p409085.png]] |
| old\_index | Number | 否 | 0 | 明细下标。   * 传入该值时，表示当前传入的section为编辑员工花名册现有的第**old\_index**条明细，此时只编辑该条明细中传入的字段。 * 不传入该值时，表示当前传入的section为新增明细，此时会保存该条明细传入的字段，未传字段会清空。 |
| group\_id | String | 否 | sys01 | 分组标识，调用[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口获取group\_id参数值。 |
| userid | String | 是 | 123456 | 被更新字段信息的员工userid。  **说明**  确保该userId是当前本企业内正确的值，否则接口会报错**系统错误**。 |

## 返回参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 示例值 | 描述 |
| result | Boolean | true | 更新是否成功。   * **true**：成功 * **false**：失败 |
| success | Boolean | true | 接口调用是否成功。   * **true**：成功 * **false**：失败 |
| errcode | Number | 0 | 返回码。 |

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/smartwork/hrm/employee/v2/update?access_token=ACCESS_TOKEN
```

请求正文

```
  {
  "agentid": "23470561",
  "param": {
    "groups": [
      {
        "group_id": "sys03",
        "sections": [
          {
            "old_index": 0,
            "section": [
              {
                "field_code": "sys03-highestEdu",
                "value": "1"
              },
              {
                "field_code": "sys03-graduateSchool",
                "value": "测试大学"
              },
              {
                "field_code": "sys03-graduationTime",
                "value": "2020-09-01"
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/smartwork/hrm/employee/v2/update");
OapiSmartworkHrmEmployeeV2UpdateRequest request = new OapiSmartworkHrmEmployeeV2UpdateRequest();
request.setAgentid(23470561L);
EmpUpdateByCustomParam param = new EmpUpdateByCustomParam();
List groups = new ArrayList<>();
EmpGroupFieldVo group = new EmpGroupFieldVo();
List sections = new ArrayList<>();
EmpListFieldVo section = new EmpListFieldVo();
List fieldVos = new ArrayList<>();
EmpFieldVo fieldVo = new EmpFieldVo();
fieldVo.setFieldCode("test");
fieldVo.setValue("测试");
fieldVos.add(fieldVo);
section.setSection(fieldVos);
section.setOldIndex(0L);
sections.add(section);
group.setGroupId("asdsad");
group.setSections(sections);
groups.add(group);
param.setUserid("user456");
param.setGroups(groups);
```

**返回示例**

```json
{
    "result": true,
    "errcode": 0,
    "success": true,
    "errmsg" : "ok"
    "request_id": "8badquf9r90f"
}
```
