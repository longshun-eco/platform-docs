更新于 2025-09-12调用本接口，根据操作员工 ID、离职人员 ID，实现企业员工离职并删除的功能。支持设置交接人。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 智能人事流程读写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=hrm_1.0%23HrmProcessTerminationAndHandover) |
| 第三方企业应用 | 暂不支持 | 暂不支持 | 暂不支持 |
| 第三方个人应用 | 暂不支持 | 暂不支持 | 暂不支持 |

## 请求方法

```
POST /v1.0/hrm/processes/terminateAndHandOver HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "userId" : "String",
  "optUserId" : "String",
  "lastWorkDate" : Long,
  "dismissionReason" : Integer,
  "terminationReasonVoluntary" : [ "String" ],
  "terminationReasonPassive" : [ "String" ],
  "dismissionMemo" : "String",
  "aflowHandOverUserId" : "String",
  "docNoteHandoverUserId" : "String",
  "dingPanHandoverUserId" : "String",
  "permissionHandoverUserId" : "String",
  "directSubordinatesHandoverUserId" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| userId | String | 是 | 要离职员工的 UserId。  **说明**   * 本接口调用成功后企业员工将直接离职并从企业通讯录删除。 * 本接口不支持主管理员的离职， 需要先撤销主管理员权限；且不支持上下级组织员工和行业通讯录员工。 |
| optUserId | String | 是 | 离职的操作人。  **说明**  必须是本组织员工，且不能是离职员工本人。 |
| lastWorkDate | Long | 是 | 离职日期，Unix事件戳，单位毫秒。 |
| dismissionReason | Integer | 否 | **说明**  废弃字段， 建议使用`terminationReasonPassive` 和 `terminationReasonVoluntary` 离职原因字段。  离职原因：   * 1：家庭原因 * 2：个人原因 * 3：发展原因 * 4：合同到期不续签 * 5：协议解除 * 6：无法胜任工作 * 7：经济性裁员 * 8：严重违法违纪 * 9：其他 |
| terminationReasonVoluntary | Array of String | 否 | 主动离职原因的id，调用[获取企业已有的所有离职原因](/document/development/api-getalldismissionreasons)接口获取离职原因id。  **说明**  `terminationReasonPassive` 和 `terminationReasonVoluntary` 至少填写一个。 |
| terminationReasonPassive | Array of String | 否 | 被动离职原因的id，调用[获取企业已有的所有离职原因](/document/development/api-getalldismissionreasons)接口获取离职原因id。  **说明**  `terminationReasonPassive` 和 `terminationReasonVoluntary` 至少填写一个。 |
| dismissionMemo | String | 是 | 离职原因备注。 |
| aflowHandOverUserId | String | 否 | 审批离职交接人。  **说明**  离职人和交接人不能相同。 |
| docNoteHandoverUserId | String | 否 | 团队文档离职交接人。  **说明**  离职人和交接人不能相同。 |
| dingPanHandoverUserId | String | 否 | 钉盘离职交接人。  **说明**  离职人和交接人不能相同。 |
| permissionHandoverUserId | String | 否 | 权限离职交接人。  **说明**  离职人和交接人不能相同。 |
| directSubordinatesHandoverUserId | String | 否 | 直属下属交接人。  **说明**  离职人和交接人不能相同。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Boolean | 是否离职成功。 |
