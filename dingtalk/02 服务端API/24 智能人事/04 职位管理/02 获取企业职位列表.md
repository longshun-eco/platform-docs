更新于 2025-09-12调用本接口，根据特定条件分页查询企业的职位相关信息，包括职位ID、职位名称和职位类别ID等信息。

### 调用说明

调用本接口前，需要对智能人事产品进行升级。智能人事升级**职位管理**功能。

**说明**

智能人事升级职位管理功能后，需注意以下：

调用通讯录[更新用户信息](/document/development/user-information-update)接口更新员工部门或者员工职位时，接口会出现报错，报错信息如下图所示。如果需要更新员工部门或者员工职位，请参考使用[智能人事员工调岗](/document/development/intelligent-personnel-staff-transfer)接口。

![[development-obtain-enterprise-position-information_O1CN01aL7Jss1qVxuTak12v_!!6000000005502-2-tps-2044-246.png]]

### 接口功能介绍

例如，在**钉钉工作台 > 智能人事 > 人事 > 职位管理**页面， 职位管理列表中有名为**测试职位1**的职位，职位名称是测试职位1，职位类别是测试职位类别1。

调用本接口可获取**测试职位1**的职位名称、职位类别等信息，如下图所示。

![[development-obtain-enterprise-position-information_O1CN01nuO5Qz1wigbYEqbUO_!!6000000006342-2-tps-2398-726.png]]

| 第三方企业应用 | 支持 | 智能人事个人信息读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=hrm_1.0%23QueryPositions) |

## 请求方法

```
POST /v1.0/hrm/positions/query?nextToken=Integer&maxResults=Integer HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "positionName" : "String",
  "inCategoryIds" : [ "String" ],
  "inPositionIds" : [ "String" ],
  "deptId" : Long
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| nextToken | Integer | 是 | 分页游标。   * 首次调用，该参数传0。 * 非首次调用，该参数传上次调用本接口返回的nextToken。 |
| maxResults | Integer | 是 | 每页最大条目数，最大值200。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| positionName | String | 否 | 职位名称。   * **首次查询**：需先调用本接口获取positionName。 * **非首次查询**：从首次查询的信息中获取需要的positionName值。   **说明**  该参数为非必填参数。 |
| inCategoryIds | Array of String | 否 | 职位类别ID列表。   * **首次查询**：需先调用本接口获取inCategoryId。 * **非首次查询**：从首次查询的信息中获取需要的inCategoryId值。   **说明**  该参数为非必填参数。 |
| inPositionIds | Array of String | 否 | 职位ID列表。   * **首次查询**：需先调用本接口获取inPositionId。 * **非首次查询**：从首次查询的信息中获取需要的inPositionId值。   **说明**  该参数为非必填参数。 |
| deptId | Long | 否 | 部门ID。  **说明**  编辑职位时可设置所属部门，如下图所示。 ![[development-obtain-enterprise-position-information_O1CN01WV2SFA1ntphceBhSJ_!!6000000005148-2-tps-2402-1274.png]]  调用接口时，根据是否填写deptId，接口调用结果获取的职位列表不同。   * 填写deptId，根据参数deptId对应的部门是否被设置为职位所属部门，调用接口获取的职位列表不同。    + **是**：调用接口，获取deptId是所属部门的职位列表。   + **否**：调用接口，获取企业内所有的职位列表。 * 未填写deptId，调用接口，获取企业内所有的职位列表。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| nextToken | Long | 分页游标。  **说明**  如果该字段未返回，表示数据已经读取完毕。 |
| hasMore | Boolean | 是否有更多数据。   * **true**：是 * **false**：否 |
| list | Array | 职位列表。 |
| positionId | String | 职位ID。 |
| positionName | String | 职位名称。 |
| positionCategoryId | String | 职位类别ID。  **说明**  只有升级职位管理，调用接口才可以获取该字段。 |
| jobId | String | 所属职务ID。  **说明**  只有升级职位管理，调用接口才可以获取该字段。 |
| positionDes | String | 职位描述。  **说明**  只有升级职位管理，调用接口才可以获取该字段。 |
| rankIdList | Array of String | 职级ID。  **说明**  只有升级职位管理，调用接口才可以获取该字段。 |
| status | Integer | 职位状态。   * **0**：启用。 * **1**：停用。   **说明**  只有升级职位管理，调用接口才可以获取该字段。 |
