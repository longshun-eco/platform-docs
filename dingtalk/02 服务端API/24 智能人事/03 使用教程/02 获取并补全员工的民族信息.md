更新于 2025-10-09本文档介绍了如何调用智能人事相关接口实现获取员工的民族信息，如果民族信息为空，通过接口实现更新。首先创建一个企业内部应用，再使用智能人事提供的API，实现获取员工民族信息、更新员工民族、获取员工花名册元数据字段信息期流程。

## 流程简介

前提条件：完成[创建应用](/document/dingstart/create-application)的流程。

步骤一：获取应用凭证信息，获取应用 Client ID 和 Client Secret。

步骤二：申请接口权限，申请智能人事相关接口的权限。

步骤三：获取应用访问凭证[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)，通过accessToken鉴权调用者身份。

步骤四：调用服务端智能人事和工作通知相关API：

1. 调用服务端API-[获取员工花名册字段信息](/document/development/api-getemployeerosterbyfield)接口，获取员工的民族。
2. 如果获取员工民族的字段为空，说明该员工的民族字段没有填写，调用服务端API-[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口，获取员工花名册字段元数据值。
3. 调用服务端API-[更新员工花名册信息](/document/development/intelligent-personnel-update-employee-file-information)接口，更新员工的民族信息。

## **前提条件**

完成[创建应用](/document/dingstart/create-application)的流程。

## **步骤一：获取应用凭证**

1. 选择目标应用，进入应用详情页，单击**基础信息** > **凭证与基础信息**。
2. 获取应用 Client ID 和 Client Secret。

## **步骤二：添加接口权限**

单击**开发配置** > **权限管理**，在权限搜索框中分别输入`qyapi_hrm_read_user`和`qyapi_hrm_manager`，并申请权限。

## **步骤三：**获取应用访问凭证accessToken

**重要**

服务端API差异详情参见[旧版服务端API、新版服务端API的区别](/document/development/how-to-call-apis#section-8lr-id4-rbz)，以下接口均使用服务端API接口，SDK下载详情参见[服务端SDK下载](/document/development/download-the-server-side-sdk)。

根据步骤一中的 Client ID 和 Client Secret，获取应用访问凭证[获取企业内部应用的access\_token](/document/development/obtain-orgapp-token)。

```java
public void getAccessToken() throws ApiException {
        DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/gettoken");
        OapiGettokenRequest req = new OapiGettokenRequest();
        req.setAppkey("dingxxxxxxxxxhgn");
        req.setAppsecret("9G_xxxxxxxxxxxxxxx1JDf0Qq3nexxxxxxxxGIO");
        req.setHttpMethod("GET");
        OapiGettokenResponse rsp = client.execute(req);
        System.out.println(rsp.getBody());
    }
```

## 步骤四：调用服务端智能人事和工作通知相关API

1. 调用服务端API-[获取员工花名册字段信息](/document/development/api-getemployeerosterbyfield)接口，获取员工的年龄。

   例如，获取员工张三的年龄信息如下

   ```json
   {"field_code":"sys02-nationType","field_name":"民族","field_value_list":[{"item_index":0,"label":"阿昌族","value":"40"}],"group_id":"sys02"}
   ```

   解析label值，可以拿到员工的民族信息。
2. 如果未获取到民族字段信息，调用服务端API-[获取花名册元数据](/document/development/intelligent-personnel-roster-metadata-query)接口，获取的民族字段元数据信息如下
