更新于 2025-12-08如果你需要使用群插件（快捷入口），你可以参考本文档操作步骤完成群插件的创建。

## **前提条件**

完成[成为钉钉开发者](/document/dingstart/dingtalk-developer)的流程。

## **操作步骤**

1. 登录[开发者后台](https://open-dev.dingtalk.com/#/)，单击顶部导航栏**开放能力** > **场景群。**选择侧边导航栏**群插件**，并单击**创建群插件。**
2. 配置群插件基本信息：

   |  |  | | --- | --- | | **配置项** | **说明** | | 企业类型 | 企业类型分为：  * 企业三方（第三方企业应用） * 企业内部（企业内部应用） | | 可选应用 | 选择已经创建完成的应用。 | | 中文名称 | 填写插件中文名称。 | | 英文名称 | 填写插件英文名称。 | | 中文描述 | 填写插件中文描述。 | | 英文描述 | 填写插件英文描述。 | | 移动端群会话底部图标 | 移动端底部图标展示。 | | Mac端侧边栏图标 | Mac端侧边栏图标展示。 | | 更多插件列表图标 | 更多插件列表的图标展示。 | | windows端图标选择 | windows端图标展示。 |
3. 配置群插件 URL：

   > 链接和菜单只能 2 选 1。

   * 配置链接：

     |  |  | | --- | --- | | **配置项** | **说明** | | 移动端链接 | 移动端单击插件打开的 URL 链接。 | | 桌面端链接 | 桌面端单击插件打开的 URL 链接。 |

     **说明**

     配置链接时支持通过配置特定的**地址动参**来实现打开链接时通过参数获取群openConversationId等参数，详情参考：[设置快捷入口（群插件）URL](/document/development/create-a-swarm-plug-in-1)。
   * 配置菜单：

     |  |  | | --- | --- | | **配置项** | **说明** | | 子菜单中文名称 | 填写插件名称。 | | 子菜单英文名称 | 填写插件英文名称。 | | 移动端 url | 移动端单击插件打开的 URL 链接。 | | PC端 url | PC 端单击插件打开的 URL 链接。 |

     配置一个子菜单完成后，你可以**添加子菜单**，也可以单击下方**新建**进行保存。
4. 单击左侧导航栏群插件，此时目标群插件正处于审核中，审核会在 1～2 个工作日通过。
