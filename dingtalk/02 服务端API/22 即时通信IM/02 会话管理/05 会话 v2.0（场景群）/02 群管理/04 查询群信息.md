更新于 2025-09-10调用本接口根据群ID查询群的信息。

根据群ID获取群名称、群图标、群主id、入群链接、群设置项等信息，适用于需要查看群详细信息的场景，如群管理、群数据分析等，具体示例如下图所示：

![[development-queries-the-basic-information-of-a-scenario-group_p530773.png]]

**说明**

支持场景：基于群模板创建的群，详情参见[创建群](/document/development/create-a-scene-group-v2)。

## 权限

服务端API是以应用维度授权的，在调用接口前，确保已经为应用添加了接口权限。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持调用 | 权限申请方式 | API Explorer调试 |
| 企业内部应用 | 支持 | 钉钉群基础信息读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=dingtalk.oapi.im.chat.scenegroup.get) |
| 第三方企业应用 | 支持 | 钉钉群基础信息读权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=dingtalk.oapi.im.chat.scenegroup.get) |
| 第三方个人应用 | 暂不支持 | 钉钉群基础信息读权限 | 暂不支持 |

## 基本信息

**请求方式**：POST

**请求地址**：`https://oapi.dingtalk.com/topapi/im/chat/scenegroup/get`

## 示例

**请求示例（HTTP）**

```http
POST https://oapi.dingtalk.com/topapi/im/chat/scenegroup/get?access_token=ACCESS_TOKEN
```

请求正文

```json
{
        "open_conversation_id":"cid9FTRQSLo+s*****=="
}
```

**请求示例（JAVA SDK）**

```java
DingTalkClient client = new DefaultDingTalkClient("https://oapi.dingtalk.com/topapi/im/chat/scenegroup/get");
OapiImChatScenegroupGetRequest req = new OapiImChatScenegroupGetRequest();
req.setOpenConversationId("cid9FTRQSLo+s*****==");
OapiImChatScenegroupGetResponse rsp = client.execute(req, access_token);
System.out.println(rsp.getBody());
```

## 错误码

|  |  |  |
| --- | --- | --- |
| 错误码（errorcode） | 错误码描述（errmsg） | 解决方案 |
| 40035 | 无效的请求参数 | 请确认参数是否符合上述入参要求 |
| 400020 | 无访问权限 | 请确认群内是否安装群模板，或者安装的群模板是否属于当前token对应的应用名下 |
| 400001 | 系统错误 | 优先确认群ID是否正确，请稍后重试 |
