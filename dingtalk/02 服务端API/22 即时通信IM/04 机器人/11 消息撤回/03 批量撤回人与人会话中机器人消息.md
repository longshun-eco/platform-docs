更新于 2025-09-12调用本接口批量撤回人与人会话中机器人消息。

### 接口功能介绍

调用本接口，根据会话ID，机器人编码和消息ID列表，实现批量撤回人与人会话中的机器人消息，适用于撤回企业消息发送错误的场景，示例如下：

![[development-batch-withdrawal-of-single-chat-robot-messages-in-person-to-person-conversations_撤回消息.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=robot_1.0%23BatchRecallPrivateChat) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=robot_1.0%23BatchRecallPrivateChat) |
| 第三方个人应用 | 暂不支持 | 企业内机器人发送消息权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/robot/privateChatMessages/batchRecall HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "openConversationId" : "String",
  "robotCode" : "String",
  "processQueryKeys" : [ "String" ]
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| openConversationId | String | 是 | 会话ID，需要与[人与人会话中机器人发送普通消息](/document/development/the-robot-sends-ordinary-messages-in-a-person-to-person-conversation)接口或[创建并投放卡片](/document/development/create-and-deliver-cards)接口使用的openConversationId保持一致。 |
| robotCode | String | 是 | 机器人的编码，可参考[机器人 ID](/document/development/robot-overview)。  需要与[人与人会话中机器人发送普通消息](/document/development/the-robot-sends-ordinary-messages-in-a-person-to-person-conversation)接口使用的robotCode保持一致。 |
| processQueryKeys | Array of String | 是 | 消息id。   * 企业内部应用，可通过[创建并投放卡片](/document/development/create-and-deliver-cards)接口或[人与人会话中机器人发送普通消息](/document/development/the-robot-sends-ordinary-messages-in-a-person-to-person-conversation)接口，获取`processQueryKey`参数值。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| successResult | Array of String | 撤回成功的消息id。 |
| failedResult | Map<String, String> | 撤回失败的消息id列表及原因。 |
