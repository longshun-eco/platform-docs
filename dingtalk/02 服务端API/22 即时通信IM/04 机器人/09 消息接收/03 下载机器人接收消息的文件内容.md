更新于 2025-09-12调用本接口下载机器人接收消息的文件内容。

### 接口功能

1. 当用户向机器人发送图片消息、语音消息、视频消息、文件消息、富文本消息时，通过机器人回调给开发者的信息中获取临时下载码`downloadCode`。
2. 通过临时下载码`downloadCode`，调用本接口获取消息中的图片、语音、文件、视频等文件的临时下载链接。
3. 访问临时下载链接，获取文件内容。

**说明**

* 群聊会话中：群成员@机器人，机器人不支持接收语音、文件、视频类型。
* 人与人的会话中：机器人不支持接收语音、文件、视频类型。
* 人与机器人的会话中：机器人支持接收语音、文件、视频类型。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=robot_1.0%23RobotMessageFileDownload) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=robot_1.0%23RobotMessageFileDownload) |
| 第三方个人应用 | 暂不支持 | 企业内机器人发送消息权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/robot/messageFiles/download HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "downloadCode" : "String",
  "robotCode" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| downloadCode | String | 是 | 用户向机器人发送文件消息后，机器人回调给开发者消息中的下载码。详情参考[机器人接收消息](/document/dingstart/receive-message)。 |
| robotCode | String | 是 | 机器人的编码，详情参考[机器人 ID](/document/development/robot-overview)。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| downloadUrl | String | 文件的临时下载链接。  **说明**  访问临时下载链接，获取下载文件，在文件被下载到本地后，需开发者替换本地下载的文件扩展名。例如替换视频文件：   * 替换前：`iAEI******IVAiYgXNCKMGzSsRB85j_Z7zCM0CYA.file`。 * 替换后：`iAEI******IVAiYgXNCKMGzSsRB85j_Z7zCM0CYA.mp4`。 |
