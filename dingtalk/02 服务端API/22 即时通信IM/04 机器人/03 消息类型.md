## 消息类型和数据格式

发送机器人消息的方式有2种，通过**接口发送**机器人消息和通过**Webhook发送**机器人消息。不同的方式，支持的消息类型和数据格式不同。

**说明**

推荐使用接口方式发送消息。

|  |  |  |
| --- | --- | --- |
| 消息类型 | 接口方式发送机器人消息 | Webbook方式发送机器人消息 |
| Text文本类型 | ✅ | ✅ |
| Markdown类型 | ✅ | ✅ |
| 图片Image类型 | ✅ | ❌ |
| ActionCard类型 | ✅ | ✅ |
| FeedCard类型 | ❌ | ✅ |
| Link链接消息 | ✅ | ✅ |

> 如果上述消息类型均无法满足你的要求，可以使用[互动卡片](/document/development/overview-card)。

## 接口方式支持的消息类型和数据格式

**重要**

* 人与机器人会话中机器人消息：支持图片、语音、文件**收发**能力。
* 群聊会话中机器人消息：图片、语音、视频、文件**发送**能力，**但群聊中用户无法@机器人发送语音、视频、文件给机器人**。

### 适用接口

* [批量发送人与机器人会话中机器人消息](/document/development/chatbots-send-one-on-one-chat-messages-in-batches)
* [机器人发送群聊消息](/document/development/the-robot-sends-a-group-message)

## Webhook方式支持的消息类型和数据格式

### 适用范围

**说明**

webhook方式只支持在群聊会话。

Webhook发送消息的实现方式，请参考[机器人回复/发送消息](/document/dingstart/robot-reply-and-send-messages)和[创建自定义机器人](/document/dingstart/custom-bot-creation-and-installation)。

### 数据格式

* **文本text类型**

  ```json
  {
      "at": {
          "atMobiles": [
              "180xxxxxx"
          ],
          "atUserIds": [
              "user123"
          ],
          "isAtAll": false
      },
      "text": {
          "content": "我就是我, @180xxxxxx 是不一样的烟火"
      },
      "msgtype": "text"
  }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **是否必填** | **类型** | **说明** | | msgtype | 是 | String | text。 | | text.content | 是 | String | 消息文本。 | | at.atMobiles | 否 | Array | 被@人的手机号。  **说明**  消息内容content中要带上"@手机号"，跟atMobiles参数结合使用，才有@效果，如上示例。 | | at.atUserIds | 否 | Array | 被@人的用户userid。 | | at.isAtAll | 否 | Boolean | @所有人是true，否则为false。 |

  ![[development-robot-message-type_p131215.png]]
* **链接Link类型**

  ```json
  {
      "msgtype": "link",
      "link": {
          "text": "这是Link消息",
          "title": "这是一个Link消息",
          "picUrl": "https://img.alicdn.com/tfs/TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png",
          "messageUrl": "https://open.dingtalk.com/document/"
      }
  }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **参数类型** | 是否必填 | **说明** | | msgtype | String | 是 | 消息类型，此时固定为：link。 | | link.title | String | 是 | 消息标题。 | | link.text | String | 是 | 消息内容。如果太长只会部分展示。 | | link.messageUrl | String | 是 | 点击消息跳转的URL。 | | link.picUrl | String | 否 | 图片URL。 |

  ![[development-robot-message-type_p385786.png]]
* **Markdown类型**

  ```json
  {
       "msgtype": "markdown",
       "markdown": {
           "title":"杭州天气",
           "text": "#### 杭州天气 @150XXXXXXXX \n> 9度，西北风1级，空气良89，相对温度73%\n> ![[development-robot-message-type_TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png]]\n> ###### 10点20分发布 [天气](https://www.dingalk.com) \n"
       },
        "at": {
            "atMobiles": [
                "150XXXXXXXX"
            ],
            "atUserIds": [
                "user123"
            ],
            "isAtAll": false
        }
   }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **是否必填** | **类型** | **说明** | | msgtype | 是 | String | markdown。 | | markdown.title | 是 | String | 首屏会话透出的展示内容。 | | markdown.text | 是 | String | Markdown格式的消息内容。 | | at.atMobiles | 否 | Array | 被@人的手机号。  **说明**  消息内容text内要带上"@手机号"，跟atMobiles参数结合使用，才有@效果，如上示例。 | | at.atUserIds | 否 | Array | 被@人的用户userid。 | | at.isAtAll | 否 | Boolean | @所有人是true，否则为false。 |

  ![[development-robot-message-type_p131216.png]]

  目前只支持Markdown语法的子集，支持的元素如下：

  ```
  标题
  # 一级标题
  ## 二级标题
  ### 三级标题
  #### 四级标题
  ##### 五级标题
  ###### 六级标题

  引用
  > A man who stands for nothing will fall for anything.

  文字加粗、斜体
  **bold**
  *italic*

  链接
  [this is a link](https://www.dingtalk.com/)

  图片
  ![[development-robot-message-type_pic.jpg]]
  ```
* **整体跳转ActionCard类型**

  ```json
  {
      "msgtype": "actionCard",
      "actionCard": {
          "title": "打造一间咖啡厅",
          "text": "![[development-robot-message-type_TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png]] \n #### 乔布斯 20 年前想打造的苹果咖啡厅 \n\n Apple Store 的设计正从原来满满的科技感走向生活化，而其生活化的走向其实可以追溯到 20 年前苹果一个建立咖啡馆的计划",
          "singleTitle" : "阅读全文",
          "singleURL" : "https://www.dingtalk.com/"
      }
  }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **是否必填** | **类型** | **说明** | | msgtype | 是 | String | actionCard。 | | actionCard.title | 是 | String | 首屏会话透出的展示内容。 | | actionCard.text | 是 | String | markdown格式的消息内容。 | | actionCard.singleTitle | 是 | String | 单个按钮的标题。 | | actionCard.singleURL | 是 | String | 单个按钮的跳转链接。 |

  ![[development-robot-message-type_p131217.png]]
* **独立跳转ActionCard类型**

  ```json
  {
      "msgtype": "actionCard",
      "actionCard": {
          "title": "乔布斯 20 年前想打造一间苹果咖啡厅，而它正是 Apple Store 的前身",
          "text": "![[development-robot-message-type_TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png]] \n\n #### 乔布斯 20 年前想打造的苹果咖啡厅 \n\n Apple Store 的设计正从原来满满的科技感走向生活化，而其生活化的走向其实可以追溯到 20 年前苹果一个建立咖啡馆的计划",
          "btnOrientation": "0",
          "btns": [
              {
                  "title": "内容不错",
                  "actionURL": "https://www.dingtalk.com/"
              },
              {
                  "title": "不感兴趣",
                  "actionURL": "https://www.dingtalk.com/"
              }
          ]
      }
  }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **是否必填** | **类型** | **说明** | | msgtype | 是 | String | actionCard。 | | actionCard.title | 是 | String | 首屏会话透出的展示内容。 | | actionCard.text | 是 | String | markdown格式的消息内容。 | | actionCard.btns | 是 | Array | 按钮。 | | btns.title | 是 | String | 按钮标题。 | | btns.actionURL | 是 | String | 点击按钮触发的URL。 | | btns.btnOrientation | 否 | String | 按钮排列顺序。  **0**：按钮竖直排列  **1**：按钮横向排列 |

  ![[development-robot-message-type_p131218.png]]
* **FeedCard类型**

  ```json
  {
      "msgtype": "feedCard",
      "feedCard": {
          "links": [
              {
                  "title": "时代的火车向前开1",
                  "messageURL": "https://www.dingtalk.com/",
                  "picURL": "https://img.alicdn.com/tfs/TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png"
              },
              {
                  "title": "时代的火车向前开2",
                  "messageURL": "https://www.dingtalk.com/",
                  "picURL": "https://img.alicdn.com/tfs/TB1NwmBEL9TBuNjy1zbXXXpepXa-2400-1218.png"
              }
          ]
      }
  }
  ```

  |  |  |  |  | | --- | --- | --- | --- | | **参数** | **是否必填** | **类型** | **说明** | | msgtype | 是 | String | feedCard。 | | links.title | 是 | String | 单条信息文本。 | | links.messageURL | 是 | String | 单条信息跳转链接。 | | links.picURL | 是 | String | 单条信息后面图片的URL。 |

  ![[development-robot-message-type_p131219.png]]

  当不想回复消息到群里时，回复格式如下：

  ```json
  {
      "msgtype": "empty"
  }
  ```
