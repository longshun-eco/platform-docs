更新于 2025-09-12调用本接口批量查询人与机器人会话机器人消息是否已读。

### 接口功能介绍

机器人批量发送消息后，查询各个消息接收者是否已读消息情况的能力，适用于企业内部沟通中24小时内需要确认消息是否被阅读的场景。

**说明**

* 一次最多可以查询一条单聊信息20名接收者的消息是否已读数据。
* 若直接发起http请求，需要先对processQueryKey参数进行URLEncoding转义，然后再将转义后的processQueryKey拼入URL中。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=robot_1.0%23BatchOTOQuery) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=robot_1.0%23BatchOTOQuery) |
| 第三方个人应用 | 暂不支持 | 企业内机器人发送消息权限 | 暂不支持 |

## 请求方法

```
GET /v1.0/robot/oToMessages/readStatus?robotCode=String&processQueryKey=String HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Query参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| robotCode | String | 是 | 机器人的编码，详情参考[机器人 ID](/document/development/robot-overview)。 |
| processQueryKey | String | 是 | 消息唯一标识，可通过[批量发送人与机器人会话中机器人消息](/document/development/chatbots-send-one-on-one-chat-messages-in-batches)接口返回参数中`processQueryKey`字段获取。  **说明**  在发送消息24小时内可以通过processQueryKey查询消息已读状态，超过24小时则无法查询。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| sendStatus | String | 消息发送状态：   * **SUCCESS**：成功 * **RECALLED**：已撤回 * **PROCESSING**： 处理中 |
| messageReadInfoList | Array | 消息已读情况。 |
| name | String | 消息接收者名称。 |
| userId | String | 消息接收者的userId。 |
| readStatus | String | 已读状态：   * **READ**：已读 * **UNREAD**：未读 |
| readTimestamp | Long | 暂不支持。 |
