更新于 2025-09-12调用本接口，查询企业机器人群聊消息用户已读状态。

### 接口功能介绍

调用本接口可用于获取机器人消息发送状态、消息已读成员userId等信息，例如当企业机器人在内部群会话发送群聊消息后，使用本接口，能查询到已读用户的userId。

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=robot_1.0%23OrgGroupQuery) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=robot_1.0%23OrgGroupQuery) |
| 第三方个人应用 | 暂不支持 | 企业内机器人发送消息权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/robot/groupMessages/query HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "openConversationId" : "String",
  "robotCode" : "String",
  "processQueryKey" : "String",
  "maxResults" : Long,
  "nextToken" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| openConversationId | String | 否 | 群ID，可通过客户端 [chooseChat](/document/development/jsapi-choose-chat)获取群会话 openConversationId。  需要与[机器人发送群聊消息](/document/development/the-robot-sends-a-group-message)接口时使用的openConversationId一致。 |
| robotCode | String | 否 | 机器人的编码，详情参考[机器人 ID](/document/development/robot-overview)。  需要与[机器人发送群聊消息](/document/development/the-robot-sends-a-group-message)接口时使用的robotCode一致。 |
| processQueryKey | String | 是 | 消息唯一标识，通过[机器人发送群聊消息](/document/development/the-robot-sends-a-group-message)接口返回参数processQueryKey字段中获取。 |
| maxResults | Long | 否 | 分页查询每页的数量，最大值200。 |
| nextToken | String | 否 | 分页游标，置空表示从首页开始查询。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| sendStatus | String | 消息发送状态：   * SUCCESS：成功 * RECALLED：已撤回 * PROCESSING：处理中 |
| readUserIds | Array of String | 消息已读人的userId。 |
| nextToken | String | 下次分页查询的游标。 |
| hasMore | Boolean | 分页查询是否还有人员可查询消息已读状态。 |
