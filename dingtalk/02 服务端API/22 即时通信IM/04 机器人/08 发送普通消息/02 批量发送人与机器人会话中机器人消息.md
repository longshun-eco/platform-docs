更新于 2025-09-12调用本接口批量发送人与机器人会话（人与机器人单聊）中机器人消息。

本接口适用于需要向多个用户（最多20个）批量发送机器人消息的场景，如企业通知、系统告警推送等场景。

**说明**

本接口支持企业内部应用机器人，详情参考[创建企业内部应用机器人](/document/dingstart/configure-the-robot-application)。

### 接口功能介绍

调用本接口，可以给批量发送单聊消息，具体示例如下图所示：

![[development-chatbots-send-one-on-one-chat-messages-in-batches_单聊助手机器人发消息.png]]

## 权限

要调用此API，需要以下权限之一。

|  |  |  |  |
| --- | --- | --- | --- |
| 应用类型 | 是否支持 | 权限 | API Explorer调试 |
| 企业内部应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=robot_1.0%23BatchSendOTO) |
| 第三方企业应用 | 支持 | 企业内机器人发送消息权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=isv&api=robot_1.0%23BatchSendOTO) |
| 第三方个人应用 | 暂不支持 | 企业内机器人发送消息权限 | 暂不支持 |

## 请求方法

```
POST /v1.0/robot/oToMessages/batchSend HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "robotCode" : "String",
  "userIds" : [ "String" ],
  "msgKey" : "String",
  "msgParam" : "String"
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过以下获取：   * 企业内部应用，调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 * 第三方企业应用，调用[获取第三方应用授权企业的accessToken](/document/development/obtain-the-access-token-of-the-authorized-enterprise-1)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| robotCode | String | 是 | 机器人的编码，本接口只支持企业内部应用机器人调用，该参数使用企业内部应用机器人的robotCode，详情参考[机器人 ID](/document/development/robot-overview)。 |
| userIds | Array of String | 是 | 接收机器人消息的用户的userId列表。  **说明**  每次最多传20个。 |
| msgKey | String | 是 | 消息模板key，详情参考[企业机器人发送消息的消息类型](/document/dingstart/types-of-messages-sent-by-robots)。 |
| msgParam | String | 是 | 消息模板参数，消息模板参数，详情参考[企业机器人发送消息的消息类型](/document/dingstart/types-of-messages-sent-by-robots)。 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| processQueryKey | String | 消息id，根据此id，可用于查询消息是否已读和撤回消息，调用[批量查询人与机器人会话机器人消息是否已读](/document/development/chatbot-batch-query-the-read-status-of-messages)和[批量撤回人与机器人会话中机器人消息](/document/development/batch-message-recall-chat)接口。 |
| invalidStaffIdList | Array of String | 无效的用户userId列表。 |
| flowControlledStaffIdList | Array of String | 被限流的userId列表。 |
