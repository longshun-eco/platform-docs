更新于 2025-09-11调用本接口，获取审批单流程中的节点信息。

### 接口功能介绍

例如，审批单在审批后台设置的流程如下图所示。调用本接口，可获取到节点的以下信息：

* 自选节点，可以获取节点名称、审批类型、设置审批人信息、选择范围、多人审批时采用的审批方式、节点激活类型等。
* 非自选节点，可以获取节点名称、审批类型、设置审批人信息、多人审批时采用的审批方式、节点激活类型等。

  ![[development-approval-process-prediction_O1CN01LpxV4b1f6M20NzZNY_!!6000000003957-2-tps-1916-950.png]]

### 使用场景

1. 用于判断是否有自选节点，和[发起审批实例](/document/development/create-an-approval-instance)接口配合使用。

   **说明**

   仅适用于调用**发起审批实例**接口时，不指定审批流程，复用审批后台设置的审批流程情况。

   ![[development-approval-process-prediction_O1CN01JClTWm1lU4EXAZn3f_!!6000000004821-2-tps-1906-920.png]]
2. 企业希望获取某审批模板设置的节点信息，如获取节点类型、节点名称等。

### 如何判断是否有自选节点

#### 有自选节点

**说明**

调用本文接口，返回信息中存在activityType字段值为target\_select的节点。

在和**发起审批实例**接口配合使用时，需注意以下：

* 将当前节点内的actorKey字段值作为发起审批接口的actionerKey参数。
* actorSelectionRange内的workNo字段值做为发起审批实例接口的actionerUserIds参数。

  ![[development-approval-process-prediction_O1CN010b2wGy1QuzaU1MHJb_!!6000000002037-2-tps-1904-916.png]]

#### 无自选节点

**说明**

调用本文接口，返回信息中不存在activityType字段值为target\_approval的节点。

![[development-approval-process-prediction_O1CN01t0X30U28ZJwNFoWOa_!!6000000007946-2-tps-1896-918.png]]

|  |  |  |  |
| --- | --- | --- | --- |

| 企业内部应用 | 支持 | 工作流实例写权限 | [API Explorer](https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=workflow_1.0%23ProcessForecast) |
