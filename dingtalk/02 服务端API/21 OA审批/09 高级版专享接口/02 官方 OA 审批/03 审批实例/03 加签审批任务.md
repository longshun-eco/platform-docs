### 特别提醒

**说明**

当前接口为OA高级版专享接口，升级OA高级版可用，可满足更高级的开发需求，响应个性化的业务场景查看全部[专享OpenAPI](/document/development/description-of-new-oa-approval-premium-exclusive-openapi-and-solutions)。

### 接口功能介绍

调用本接口，当前审批人可对审批任务进行加签操作，加签方式支持前加签、后加签，审批方式支持或签、会签、依次审批等。接口调用成功后，实现效果如下图所示。

## 请求方法

```
POST /v1.0/workflow/premium/tasks/append HTTP/1.1
Host:api.dingtalk.com
x-acs-dingtalk-access-token:String
Content-Type:application/json

{
  "taskId" : Long,
  "processInstanceId" : "String",
  "operateUserId" : "String",
  "appenderUserIds" : [ "String" ],
  "remark" : "String",
  "activateType" : "String",
  "type" : "String",
  "agreeAll" : Boolean
}
```

## Header参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| x-acs-dingtalk-access-token | String | 是 | 调用该接口的访问凭证，通过调用[获取企业内部应用的accessToken](/document/development/obtain-the-access-token-of-an-internal-app)接口获取。 |

## Body参数

|  |  |  |  |
| --- | --- | --- | --- |
| 名称 | 类型 | 是否必填 | 描述 |
| taskId | Long | 是 | 审批任务ID。可通过调用[获取单个审批实例详情](/document/development/obtains-the-details-of-a-single-approval-instance-pop)接口获取`taskId`参数值。 |
| processInstanceId | String | 是 | 审批实例ID：   * 调用[发起审批实例](/document/development/create-an-approval-instance)接口获取`InstanceId`参数值。 * 调用[获取审批实例ID列表](/document/development/obtain-an-approval-list-of-instance-ids)接口获取`list`参数值。 |
| operateUserId | String | 是 | 操作人的用户ID，需要跟任务的当前执行人保持一致，否则无法通过校验。 |
| appenderUserIds | Array of String | 是 | 加签人用户ID。 |
| remark | String | 否 | 加签备注信息。 |
| activateType | String | 是 | 审批方式：   * **ALL**：或签/或签 * **ONE\_BY\_ONE**： 依次审批 |
| type | String | 是 | 加签方式：   * **before**：前加签 * **after**：后加签 |
| agreeAll | Boolean | 否 | 是否需要全部同意，多人审批时需指定：   * **true**：依次审批 * **true**：会签 * **false**： 或签 |

## 返回参数

|  |  |  |
| --- | --- | --- |
| 名称 | 类型 | 描述 |
| result | Boolean | 是否加签成功。 |
