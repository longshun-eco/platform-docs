更新于 2025-09-10OA审批提供了丰富的开放能力，开发者通过API接口、事件回调、前端AppLink页面等集成可以实现OA审批和企业业务系统打通。本文汇总了OA审批标准版和OA审批高级版所有的开放能力供开发者参考。

## OA审批标准版开放的接口能力

全量开放30+标准接口、2个标准流程对接方案、2类基础审批事件开放。

|  |  |  |  |
| --- | --- | --- | --- |
| **Open API**  [**全量开放30+**](/document/development/workflow-overview#section-fs4-8ov-8im)**：**  **官方OA审批 & 流程中心**    ![[development-oa-approval-open-capacity-summary_p925322.png]] | **事件**  [**全量开放2类**](/document/development/event-bpms-instance-change)**：**  **审批实例变更 & 审批任务变更**     * 审批实例开始、结束、终止、删除 * 审批任务开始、结束、取消 | **AppLink**  **暂未开放**        **钉钉应用标准链接协议，可以让钉钉的应用或功能直接通过链接即可访问**，如跳转某个审批页面、打开审批详情页、发起页等功能等。 | **解决方案**  [**2个标准流程对接方案**](/document/development/dingtalk-oa-approval-process-center-opening-scheme)       * **官方OA审批对接方案** * **自有OA审批对接方案** |
