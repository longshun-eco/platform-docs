## 免费规格说明

针对付费应用，可提供免费规格，价格为0元，周期无限（100年）。 免费规格是针对那些使用后付费意愿没有那么强的组织，也能有基本可用的功能，在工作台上的应用入口一直存在，保持应用的活跃，以便后续再次能转化为付费用户。

针对免费规格，可对功能/人数做一定的限制，如只能使用最基础的功能，或者只允许6人使用。具体如何限制ISV可自行设定。

## 配置免费规格

1. 登录[开发者后台](https://open-dev.dingtalk.com/)，单击**应用运营**，然后单击进入需要配置免费规格应用的应用详情页，最后选择**收费模式配置**并单击后方**编辑**按钮。
2. 勾选**体验规格配置**中的**提供免费规格**。
3. 根据自身需求**选择免费模式**并**设置免费人数**。

   可选择是否限制免费规格人数，不设置最大人数代表免费人数不做限制，全员可用。

   **说明**

   人数限制如果要变更，可以增加人数或者变成不限人数，但是不能缩小。

   免费规格在应用内的是否做功能的限制以及怎么做功能限制，可由服务商自定控制。如果配置了最大支持人数，钉钉侧应用开通选择使用人员范围时会做控制，不允许选择全员或者大于人数限制的使用范围。
4. 根据需求选择**优先开通顺序**。

   优先开通顺序，可选项有三种：

   * 优先开通试用，试用到期后不付费自动转免费规格
   * 优先试用，试用到期后用户自主选择开通免费或付费
   * 优先开通免费规格，用户自主选择开通试用或付费规格
5. 单击页面下方**保存**退出页面，然后单击**提交审核**。

   提交审核之后，联系对应的类目运营小二去后台审核规格更新。

   **说明**
   * 开发者后台完成配置且通过审核后，免费规格即会完成上线。线上展示免费规格（包括应用内授权和SKU页）生效。
   * 免费规格不仅仅是针对个人的，是这个应用相关的，从各个流量入口（工作台、广场等）开通时，管理员和非管理员开通的时候，默认开通的是免费规格。

![[development-free-specifications_p361074.png]]

**重要**

* 免费规格配置后，目前在天元系统不会展示（天元侧目前不支持开通时选择人员），无法代客下免费规格订单。
* **目前在开发者后台中支持删除免费规格（由勾选“提供免费规格”改为不勾选），但是由于删除免费规格后再重新创建免费规格（由不勾选“提供免费规格”改为勾选），新免费规格的规格码会和原来的不一样。如果ISV的实现中是通过规格码来识别免费规格的话这种场景会有问题，因此请谨慎进行删除操作。**

## 免费规格和试用规格的切换

增加免费规格之后，首次开通应用时，支持**优先开通试用**或**优先开通免费**。

试用规格到期（不包括个人版试用到期）后，客户可以自己再次开通免费规格。 钉钉侧也可以帮客户自动切换为免费规格。

试用到期自动切换为免费规格的实现方式是：在试用到期当天，以上一笔订单的下单人名义再下一笔免费规格订单，会有正常的订单消息推送给服务商。

**订单消息格式请参考**：

* **RDS/SyncHTTP推送**：[biz\_type=17](/document/development/data-format#section-5je-fzy-8p5)
* **HTTP推送请参考**：[应用市场下单事件](/document/development/application-market-order)

试用规格自动切换为免费规格，应用使用范围的变化规则是：

（1）当前使用范围 ≤ 免费规格的上限，当前的使用范围不变。

（2）如果当前使用范围 > 免费规格的上限，使用范围设置为开通人这一个人。

建议免费规格人数不做限制，功能做限制，这样降为免费规格之后，使用范围中的人，在工作台仍可以看到和使用应用。

**重要**

* 如果试用订单的下单人已经没有下单权限（如在试用期间离职或者从管理员变成员工），则无法再次下单，无法自动切换免费。
* 由于自动切换是以客户名义去下免费(100年)订单，**可能会涉及到商机客保等问题，建议谨慎考虑**。

## 优先试用和优先免费区别

### 优先试用

* 如果企业首次开通，在SKU页或者应用内授权看不到免费规格，只能看到试用规格。
* 如果企业之前开通过试用或者付费，开通到期之后，在SKU页会看到免费规格。

### 优先免费

* 如果企业首次开通，在SKU页或者应用内授权就会看到免费规格。
* 如果企业之前开通过试用或者付费，开通到期之后，也会看到免费规格。

**重要**

**有部分应用曾经接入过“个人试用” (不是“个人版”，两者区别在个人版接入文档里有说明)， “个人试用”在应用内授权页面只会开通试用规格，不支持“优先免费”。如果需要“优先免费”，请升级到“个人版”（升级方式参见**[成员授权接入](/document/development/document-on-authorization-of-application-market-members)**）。**

## 自测

1. 服务商判断是否需要对自己应用的免费规格做人数或者功能限制。
2. 在开发者后台配置免费规格，提交审核，并联系对应的类目运营小二审核通过。
3. 由于部分商品线上已经接入了应用内授权，首次开通不跳转到SKU页; 提供下面SKU页测试链接, 链接里面**$GOODS$**替换为要测试的商品码。

```
https://h5.dingtalk.com/open-market/skuDetail.html?
dd_share=false
&showmenu=false
&goodsCode=$GOODS$
&funnelsource=freeItem
&bizCode=isv
```
