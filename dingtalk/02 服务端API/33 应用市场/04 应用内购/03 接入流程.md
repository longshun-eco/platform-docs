更新于 2026-02-27本文适用于希望在钉钉应用中集成内购功能的ISV开发者，需已完成应用创建并具备相应开发能力。

## 前提条件

* 应用类型为第三方企业应用，并已在钉钉开放平台创建。
* 企业已完成认证（认证企业）。
* 已获取应用的 Client ID 和 Client Secret。
* 开发者已掌握钉钉服务端 API 调用方式，具备接收异步回调的能力。
* 应用服务器可访问公网，能够接收来自钉钉的支付成功回调通知。

## 内购接口权限申请

在调用内购相关接口前，，需要先申请接口权限：

1. 登录[开发者后台](https://open-dev.dingtalk.com/#/appMgr/provider/eapp/53642/1)，单击目标应用打开详情页。
2. 在**权限管理**页面，单击**应用内购**。

   ![[development-lg1nb7_p264471.png]]
3. 选择应用内购相关接口权限，单击**申请权限**。

   ![[development-lg1nb7_p264472.png]]

## **内购商品购买流程（非iOS）**

非iOS用户内购系统间交互流程如下图所示，请注意以下关键点：

1. ISV在接收到支付成功回调通知后进行业务的开通处理，并在处理成功后调用`/appstore/internal/order/finish`接口通知钉钉该笔订单已完成处理。
2. 支付成功回调通知时如果通知失败(如网络异常)，钉钉侧将进行通知重试，因此ISV对支付成功回调通知的处理需要保持幂等。
3. 第四步“购买成功”跳转的目标页面地址为ISV调用`/appstore/internal/skupage/get`接口时传入的callback\_page参数。

   1. 微应用场景：`callback_page`必须与微应用首页页面同域名。
   2. 小程序场景：`callback_page`必须为小程序内的页面。

      **重要**

      跳转到目标页面不能作为“购买成功”的判断依据，判断购买成功需要根据服务端“支付成功回调通知”结果为准。

![[development-lg1nb7_p173361.png]]

## 内购商品购买流程（iOS）

iOS用户内购系统间交互如下图所示。

内购商品目前暂不支持iOS用户在手机端直接完成支付购买。用户可以通过DRP分销平台，联系代理服务商为iOS用户进行内购商品的代下单。下单后用户将收到支付提醒消息，完成支付后，钉钉侧将通过回调通知ISV。

## **内购商品核销**

对于消耗型商品，用户在使用商品时，ISV完成自身的业务逻辑处理后，需要调用钉钉的商品消耗接口，对指定订单商品进行消费。

![[development-lg1nb7_p173373.png]]

## 异常场景处理

由于异步支付成功回调通知在特殊异常场景，例如网络异常导致多次重试仍未正常处理，可能导致ISV未处理用户已支付的订单。为防止内购商品掉单，确保用户支付的每一笔内购订单ISV都有进行处理，钉钉侧提供服务端接口给ISV，用于查询所有未处理的已支付订单信息。建议ISV每日定时调用该接口获取已支付订单进行补偿处理。

接口名：`/appstore/internal/unfinishedorder/list`

## 通知回调

* 数据推送

  + **触发条件：**用户的内购订单支付成功后触发，插入表open\_sync\_biz\_data中。
  + 订单信息，biz\_type = 17，详情参见[应用市场下单](/document/development/application-market-order)事件。

    在原订单数据的基础上，内购订单数据信息新增如下字段：

    appId：内购应用ID。
  + 订单到期或者用户退款等导致的服务关闭，目前仅推送因用户退款导致的服务关闭，biz\_type = 37，详情参见[钉钉交易订购开启](/document/development/dingtalk-transaction-ordering-on)/[钉钉交易订购关闭](/document/development/dingtalk-transaction-ordering-closed)事件。
