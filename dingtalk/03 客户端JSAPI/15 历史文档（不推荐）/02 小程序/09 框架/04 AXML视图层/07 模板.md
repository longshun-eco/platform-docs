更新于 2025-09-17AXML提供模板（template），可以在模板中定义代码片段，在不同的地方调用。

**说明**

此处 template 区别于 slot，slot 可参考[自定义组件](/document/dingstart/mini-app-development-process)。

## 定义模板

使用`name`属性，作为模板的名字，然后在`<template/>`内定义代码片段。

```json
     {{index}}: {{msg}}
     Time: {{time}}
```

## 使用模板

使用`is`属性，声明需要使用的模板，然后将该模板所需要的`data`传入，例如：

```
Page({
  data: {
    item: {
      index: 0,
      msg: 'this is a template',
      time: '2016-09-15'
    }
  }
})
```

`is`属性可以使用`Mustache`语法，来动态决定具体需要渲染哪个模板。

```
   odd

   even
```

## 模板作用域

模板拥有自己的作用域，只能用`data`传入的数据，但可以通过 `onXX` 绑定页面的逻辑处理函数。

推荐用 template 方式来引入模板片段，因为 template 会指定自己的作用域，只使用`data`传入的数据，因此应用会对此进行优化。如果该 template 的 data 没有改变，该片段 UI 并不会重新渲染。
