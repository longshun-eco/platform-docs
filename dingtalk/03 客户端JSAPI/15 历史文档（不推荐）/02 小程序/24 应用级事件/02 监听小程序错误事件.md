更新于 2025-09-17调用**dd.onError**监听小程序错误事件。目前仅指JS执行错误，触发时机和参数与[App.onError](/document/orgapp/app-js-registration-mini-program)的一致。

**说明**

* 开发者可以通过[dd.canIUse](/document/orgapp/dd-caniuse)函数判断端上是否支持此能力。
* 使用**dd.onError**监听到的报错，[App.onError](/document/orgapp/app-js-registration-mini-program)方法也会监听到。
* 使用本接口**dd.onError**监听页面报错，如果在多个页面开启监听没有关闭，则页面报错时会触发多个监听事件，建议在页面关闭时调用[dd.offError](/document/orgapp/cancels-the-listening-applet-error-event)关闭监听。

### .axml示例代码

```java
Page({
  onLoad() {
    dd.onError(this.onErrorHandler);
  },
  onErrorHandler(error){
    dd.alert({
        title:'触发error',
        content:JSON.stringify(error),
      });
  },
  handleTap(){
    throw new Error('global error');
  },
  offOnError(){
    dd.offError(this.onErrorHandler);
  },
});
```
