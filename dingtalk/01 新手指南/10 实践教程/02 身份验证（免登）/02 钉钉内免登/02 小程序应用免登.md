本功能适用于企业内部自建应用的小程序免登场景，即企业员工在钉钉工作台中使用小程序时，无需输入账号密码即可完成身份认证，实现无缝登录体验。

## 适用对象

本文档适用于具备服务端开发能力的钉钉开发者，用于实现企业内部应用在钉钉小程序环境下的免登功能。需熟悉前端小程序开发及后端接口调用流程。

## **前提条件**

1. 成为[钉钉开发者](/document/dingstart/get-developer-permissions)。
2. 创建[钉钉企业应用](/document/dingstart/create-application)，获取应用凭证信息 Client ID 和 Client Secret。
3. 完成[添加小程序应用](/document/dingstart/add-application)能力。
4. 已经下载[小程序开发工具](/document/download/miniapp-tool)和服务端开发工具，服务端工具可根据实际开发环境自由下载。

## **开发流程**

建议将用户信息保存在前端缓存中（如 `dd.setStorage`）或 Cookie 中，避免每次进入应用都重复调用钉钉接口进行免登验证，提升性能与用户体验。

1. 调用`dd.getAuthCode`接口获取免登授权码，小程序免登可参考[小程序免登](/document/development/jsapi-get-auth-code)。

   若在小程序中使用`dd.httpRequest`API，需将请求域名添加至【小程序开发设置 > 安全设置】中的 HTTP 可信域名列表，否则线上发布后请求将被拦截。

   ![[dingstart-small-program-application-free-of-registration_p963889.png]]

   ```javascript
   dd.getAuthCode({
     success: (res) => {
       const { authCode } = res;
     },
     fail: () => {},
     complete: () => {},
   });
   ```
2. 调用[获取应用的 Access Token](/document/development/api-gettoken)接口，获取应用级别的访问凭证。
3. 根据免登授权码`code`和应用级`access_token`，调用[通过免登码获取用户信息](/document/development/obtain-the-userid-of-a-user-by-using-the-log-free)接口，获取当前用户的`userId`和`unionId`等身份信息。

**说明**

开发完成后必须发布应用，免登功能仅支持在钉钉客户端内运行，外部浏览器或非钉钉环境无法正常调用免登组件。

### **操作步骤**

1. 确保完成上述准备工作，满足 Demo 运行条件。
2. 你可以下载示例 [mini-app-sso.zip](https://help-static-aliyun-doc.aliyuncs.com/file-manage-files/zh-CN/20250604/sjjrjm/mini-app-sso.zip) Demo。

   > 解压包含两个目录：`backend`（服务端代码）和`mini-app-front`（小程序前端代码）。
3. 打开服务端开发工具，导入`backend`目录中的服务端项目。

   ![[dingstart-small-program-application-free-of-registration_p1018972.png]]
4. 打开`application.properties`文件，填写应用的`clientId`和`clientSecret`。

   > **注意**：请删除原字段值末尾可能存在的空格，确保配置格式正确。

   ![[dingstart-small-program-application-free-of-registration_p1018974.png]]
5. 启动小程序免登后端服务即可。

   > **注意**：启动前请确认 8080 端口未被其他进程占用，否则会导致服务启动失败。
6. 打开钉钉小程序开发工具，单击右上角【打开项目】，选择解压后的`mini-app-front`目录。项目类型应选择 **钉钉** 的 **企业内部应用**。
7. 单击左上角【登录以选择关联应用】，并选择已在前述步骤中完成配置的小程序应用。
8. 登录成功且应用绑定完成后，可在 IDE 模拟器中点击【免登登录】按钮进行测试。

   ![[dingstart-small-program-application-free-of-registration_p963927.png]]
9. 成功调用后，将在页面展示当前用户的`userId`和`unionId`等信息。

   ![[dingstart-small-program-application-free-of-registration_p963934.png]]
10. 测试无误后，单击右上角【上传版本】，将小程序代码上传至钉钉平台。
11. 进入对应应用详情页，依次单击**应用发布** > **版本管理与发布**。

    ![[dingstart-small-program-application-free-of-registration_p963945.png]]
12. 单击【创建新版本】，填写版本信息，选择已上传的小程序版本，提交发布。
13. 发布完成后，你需要在钉钉工作台打开应用。

    ![[dingstart-small-program-application-free-of-registration_p963974.png]]
14. 单击【免登登录】按钮，查看是否能正常获取用户信息。

    ![[dingstart-small-program-application-free-of-registration_p963975.png]]

### **常见问题**

**Q：在钉钉客户端访问如果出现“网络请求失败：4”，如何解决？**

![[dingstart-small-program-application-free-of-registration_p964632.png]]

A：造成这个原因可能是小程序缓存导致的，你需要清理一下钉钉客户端的缓存，步骤如下：

1. 打开钉钉客户端设置，单击**存储空间**。

   ![[dingstart-small-program-application-free-of-registration_p964650.png]]
2. 单击**缓存数据** > **前往清理**，进入缓存数据页面，勾选小程序，确认清理即可。

   ![[dingstart-small-program-application-free-of-registration_p964661.png]]
3. 清理后，重新在工作台访问小程序应用。
