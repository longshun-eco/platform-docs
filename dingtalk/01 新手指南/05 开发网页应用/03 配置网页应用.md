更新于 2026-01-22为了进行网页应用的开发，你需要完善网页应用的配置信息。你可以依据本文档操作流程来完成网页应用信息的配置。

## 适用对象与权限说明

本文档适用于**需要开发网页应用**的开发者。

关键信息如应用 ID 可在【开发者后台】>【应用详情】>【凭证与基础信息】中查看。

更多关于应用类型的说明，请参考：[应用类型介绍](https://open.dingtalk.com/document/development/introduction-to-application-types)。

## **前提条件**

在配置网页应用信息前，需要完成以下流程：

1. 需要完成[创建应用](/document/dingstart/create-application)流程。
2. 需要完成[添加应用能力](/document/dingstart/add-application)流程

## **操作步骤**

1. 登录[开发者后台](https://open-dev.dingtalk.com/#/)，单击目标应用，进入应用详情页。
2. 单击**应用能力** > **网页应用** > **编辑标识，**进入网页应用配置信息页面。
3. 根据业务需求填写以下配置项：

   |  |  |  | | --- | --- | --- | | **配置项** | **是否必选** | **配置说明** | | 应用首页地址 | 是 | 输入应用首页 URL，在移动端工作台点击应用图标会跳转到此页面。支持输入以下两种类型的链接：  * 以 http 或 https 开头的链接，例如`[https://www.dingtalk.com](https://www.dingtalk.com/)`。 * 后端服务部署的服务器的IP或域名，例如：`http://公网IP:8080`。 | | PC端首页地址 | 否 | 输入可以在钉钉 PC 端的工作台上打开网页应用的地址。链接地址必须以 http 或 https 开头。  **说明**  如果未输入，只支持在钉钉客户端使用该网页应用。在钉钉PC端工作台点击应用图标时，会提示“电脑版暂不支持显示，请用手机钉钉扫描下方二维码查看”。 | | 管理后台地址 | 否 | 输入管理员在[钉钉管理后台](https://oa.dingtalk.com/)访问该应用的地址。 |
4. 配置完成后，单击**保存**按钮提交更改。
5. （可选）启用网页应用的酷应用场景。

   **说明**

   启用前，必须确保应用首页地址已输入。

   如果应用首页地址发生变更，必须手动更新对应酷应用配置。关于如何配置，可参考[接入链接增强](/document/dingstart/access-link-enhancements-coolapp)。

   * 应用名片：启用后，系统会自动创建一个链接增强酷应用，并自动将应用名称、应用简介和应用首页地址解析在卡片内容中。如需自定义应用名片的卡片样式，点击**查看**跳转到链接增强酷应用，然后点击**前往卡片搭建器**更新卡片使用的模板。相关信息，可参考[互动卡片模板编辑器](/document/development/card-template-overview)。
   * 应用捷径：如果想在输入框（如聊天输入框、搜索输入框）内通过快捷指令“/”唤起H5微应用，点击**启用**应用捷径。

     启用后，系统会自动创建一个快捷指令酷应用，并自动将应用名称、应用简介和应用首页地址解析为指令名称、指令简介和指令执行后打开链接。

## **后续步骤**

配置完成后，你可以进行[发布应用](/document/dingstart/publish-dingtalk-application)，使其对组织内成员可见并可用。如需进一步开发网页应用前端或服务端功能，请参考：

* [开发网页应用前端](/document/dingstart/develop-webapp-frontend)
* [开发网页应用服务端](/document/dingstart/develop-webapp-backend)
