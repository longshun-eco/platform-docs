应用类型介绍

钉钉为企业和组织提供了丰富的基础办公应用，例如审批、日志、视频会议等。此外，企业可基于钉钉开放平台的能力，根据实际业务需求开发定制化应用，提升办公效率与协作体验。

## **应用类型**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **应用类型** | **开发者** | **使用人员** | **支持的能力** | **是否支持上架到钉钉应用广场** |
| 企业内部应用 | 企业内部开发者或委托的服务商开发者 | 安装了该应用的企业内部人员 | * 小程序    + 支持移动端   + 支持PC端 * H5微应用    + 支持移动端   + 支持PC端 | 否 |
| 第三方企业应用 | 产品方案商的开发者 | 购买开通该三方应用的企业内部人员 | * 小程序    + 支持移动端   + 支持PC端 * H5微应用    + 支持移动端   + 支持PC端 | 是，需要满足上架要求，上架流程请参考[合作全流程指引](/document/services/isv-cooperation-guide)。 |
| 第三方个人应用 | 产品方案商的开发者 | 钉钉的个人用户 | * 小程序    + 支持移动端   + 支持PC端 | 否 |
| 移动应用接入 | 产品方案商的开发者  企业内部开发者 | 要接入应用的用户 | * 登录（钉钉扫码和使用钉钉账号登录） * 分享（将应用分享到钉钉） | — |

## **企业内部应用**

如果所开发的应用仅服务于单一企业内部用户，且不对外部组织开放，建议选择创建**企业内部应用**。此类应用适用于企业自研系统集成、内部工具开发等场景。

* 如何开发企业内部应用

  详细的开发流程，请参考[企业内部应用学习指南](/document/dingstart/org-learning-map)。
* 企业内的员工如何使用应用

  以“测试H5微应用”为例，员工可在工作台中查找并使用该应用。

  例如，小明开发了企业内部应用，名称为“测试H5微应用”，可以在**工作台 > 全部应用**中找到该应用。

  **说明**
  + 新创建并发布的应用，需要在**全部应用**中找到该应用。
  + 如果希望应用在工作台正常展示，需要操作添加应用到工作台。

## **第三方企业应用**

若开发的应用面向多个不同企业客户，提供通用解决方案，并计划推广至更广泛用户群体，应选择创建**第三方企业应用**。该类型应用可上架至钉钉应用广场，供其他企业搜索、试用和购买。

* 如何开发第三方企业应用

  详细开发流程请参考[第三方企业应用学习指南](/document/dingstart/isv-learning-map)，上架流程详见[合作全流程指引](/document/services/isv-cooperation-guide)。
* 其他企业如何开通使用第三方企业应用

  当第三方企业应用成功上架后，目标企业可通过以下方式开通使用：

  以上架到广场为例，第三方企业应用满足上架标准上架后，会在**工作台**> **右上角**> **进入广场**，展示该应用。安装后，该应用将出现在企业的工作台中，员工即可正常使用。
