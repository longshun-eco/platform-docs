更新于 2026-01-22发布应用完成后，您需要完成上架前的应用自检。本指南适用于第三方ISV应用的开发者，在提交上架前进行功能与稳定性检查。

## 准备工作

在开始自检前，请确认已完成以下配置和准备工作：

* 已登录[开发者后台](https://open-dev.dingtalk.com/)并拥有目标应用的管理或开发权限。
* 应用类型为“第三方企业应用”，且已完成开发与内部测试。
* 已在应用的 **开发配置 > 安全中心** 中配置正确的服务器出口公网IP。
* 应用首页及管理后台地址已启用HTTPS协议。
* 推送类型已设置为SyncHTTP或Stream，并完成回调URL的安全域名配置，参考[配置并更新安全域名](/document/dingstart/configure-update-secure-domain-name)。
* 已接入监控中心，并持续运行至少24小时，确保有完整的API调用与异常监控数据。
* 服务端推荐部署于阿里云环境。

完成上述准备后，方可进入正式的自检流程。

## **操作步骤**

1. 登录[开发者后台](https://open-dev.dingtalk.com/)，单击目标应用，进入应用详情页。
2. 在应用详情页，单击**应用发布 > 应用自检**。
3. 单击 **开始检测**，系统将自动完成功能项检查。

   该页面主要验证应用的基础配置是否合规，各检查项说明如下：

   |  |  | | --- | --- | | 检查项 | 说明 | | 是否设置服务器出口ip | 需要在应用的开发配置 > 安全中心中设置服务器出口 IP。 | | 应用首页或管理后台地址是否https协议 | 应用首页或管理后台地址需要使用https协议。 | | 是否已设置有效推送类型 | 需要在应用的开发信息中完成推送设置。若使用SyncHTTP推送，回调URL地址需要使用[配置并更新安全域名](/document/dingstart/configure-update-secure-domain-name)。 | | 是否已完成监控中心接入 | 需要在监控中心中完成接入，并且产生监控数据。 | | 服务端是否已部署在阿里云 | 选择正确的部署方式。 |
4. 功能自检完成后，单击**下一步**进入安全审核环节。请参考[安全准入评估](/document/services/submit-your-product-to-the-shelves#undefined)，完成所需材料填写并提交审核。
5. 提交后进入稳定性监控阶段，如下图所示，具体要求参考[稳定性认证标准](/document/services/submit-your-product-to-the-shelves)。

   ![[dingstart-selfcheck-dingtalk-app_p700376.png]]

   |  |  |  | | --- | --- | --- | | **数据指标** | **说明** | **指标要求** | | 监控中心的API成功率 | 可以在监控中心的**监控指标** > **API请求** > **成功率**处查看。 | ≥98.99% | | 监控中心的API平均耗时 | 可以在监控中心的**监控指标** > **API请求** > **请求耗时**处查看。 | <300 | | 监控中心的js报错率 | 可以在监控中心的**监控指标** > **JS异常** > **异常次数PV比**处查看。 | <1.99% | | 监控中心的白屏率 | 可以在监控中心的**监控指标** > **白屏异常** > **白屏率**处查看。 | <1.99% | | dd.config成功率 | 调用 d**d.config** SAPI的日平均成功率。 | ≥98.99% | | Http推送成功率 | 如果推送类型是Stream或SyncHttp时，成功率是统计日期内钉钉给回调地址推送的成功率（**耗时超过1500ms的推送算超时失败**）。 | ≥98.99% |
6. 稳定性监控完成后，单击**自检完成，**并**提交上架。**

   ![[dingstart-selfcheck-dingtalk-app_p700389.png]]

## **后续步骤**

应用提交上架后，需继续完善商品信息。请参考[步骤三：提交应用商品上架](/document/services/submit-your-product-to-the-shelves)，完成应用名称、图标、描述、分类等信息的填写，以便进入审核队列。

**提示**：如自检失败或指标不达标，请根据错误提示修复问题后重新运行检测。
