更新于 2026-01-22本文档介绍如何在第三方个人应用中实现免登接入。通过钉钉开放平台提供的能力，用户在使用应用时可无需输入账号密码，快速完成身份验证。

## 适用对象与限制条件

本功能适用于以下场景：

* 应用类型：**第三方个人应用**
* 运行环境：仅支持**小程序**
* 认证方式：基于临时授权码（authCode）的授权机制

## 流程说明

### 前端发起授权并获取 authCode

用户进入E 小程序后，前端通过钉钉客户端提供的 JSAPI 发起授权请求，获取[免登临时授权码](/document/personalapp/mini-program-free-login)（authCode），随后将该code传递给应用后端。该操作通常使用`dd.getAuthCode` 接口完成。

> **关键术语说明**：`authCode` 是通过钉钉客户端 JSAPI（如 `dd.getAuthCode`）获取的临时授权码，有效期短暂且仅能使用一次，用于后端换取用户基本信息。

### 服务端调用接口获取用户信息

应用后端接收到`authCode`后，调用钉钉服务端接口 `sns/getuserinfo_bycode`[获取用户基本信息](/document/personalapp/queries-basic-user-information)，完成登录流程。

**说明**

* 调用频率每分钟会有限制，请合理控制请求频次。
* 建议对异常情况进行重试机制与日志记录。

## **示例代码**

### **前端示例**

```javascript
dd.getAuthCode({
    success: function(res) {
        const authCode = res.authCode;
        // 将 authCode 发送到应用后端
    },
    fail: function(err) {
        console.error('获取 authCode 失败', err);
    }
});
```

### **服务端示例**

接口调用成功后，返回用户信息包括 `nick`（昵称）、`openid`、`unionid` 等字段。


## 缓存策略建议

为提升用户体验并减少重复请求，建议对用户登录状态进行本地缓存管理：

* 使用 `dd.setStorage` 在客户端存储用户信息及时间戳。
* 下次进入页面时优先读取缓存。
* 推荐缓存有效期设置为 2 小时，避免长时间无效登录状态。

## 常见问题

* **Q: authCode 获取失败有哪些常见原因？**

  A: 可能原因包括：未正确引入 dd-sdk-js、网络异常、未在钉钉客户端内打开等。
* **Q: 用户信息接口返回 errcode=40001？**

  A: 表示凭证无效，请检查填写的参数 是否正确，并确认authCode是否已过期。
* Q: **是否支持 PC 端免登？**

  A: 当前方案主要适用于移动端 小程序；PC 端建议结合扫码登录或其他认证方式。
