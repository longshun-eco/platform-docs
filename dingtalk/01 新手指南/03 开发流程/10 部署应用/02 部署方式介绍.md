## **什么是计算巢**

计算巢服务是一个面向服务商（包括企业应用服务商、IT集成服务商、交付服务商和管理服务提供商等）和终端用户的PaaS管理平台。通过计算巢，服务商可以在阿里云上实现服务的标准化部署、自动化交付和集中化运维；用户则可统一管理和监控在阿里云上订阅的各类第三方服务，提升资源使用效率与治理能力。

## 适用对象

本部署文档适用于以下角色：

* **企业管理员**：负责在钉钉后台配置应用权限、绑定AgentId，并启用安全能力。
* **服务商技术负责人**：负责在计算巢控制台创建服务实例，配置ECS、RDS等基础设施资源。
* **开发者**：负责对接钉钉API、实现安全渲染逻辑或接入数据安全中心。

## 权限说明

|  |  |
| --- | --- |
| 角色 | 操作范围 |
| 阿里云主账号 | 可登录计算巢控制台，创建/管理服务实例，配置云资源。 |
| 钉钉企业超级管理员 | 可安装应用、授权通讯录等敏感权限、启用安全渲染组件。 |
| 普通员工 | 仅能使用已部署的应用功能，无配置权限。 |

## 部署方式

企业通讯录（用户名、用户职位、部门名称）是企业的重要敏感数据。服务商必须保障企业通讯录数据安全，否则无法上架应用市场。

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **方案介绍** | **部署方式** | **资源要求** | **优劣势** | **开通方式** |
| 自行接入钉钉安全渲染组件实现通讯录加密 | 计算巢 + 钉钉安全渲染组件 | * ECS * SLB（目前只支持CLB） * RDS * EIP | * 优势：无需购买数据安全中心。 * 劣势：存在接入改造成本。 | * 方式一：管理员在应用广场开通 * 方式二：商品二维码线下开通 |
| 启用数据安全中心，全面保障包括通讯录数据安全在内的所有数据 | 计算巢 +[数据安全中心](https://help.aliyun.com/product/88674.html?spm=ding_open_doc.document.0.0.7ea04a97BcYnJr) | * ECS * SLB（目前只支持CLB） * RDS * EIP * DSC | 优势：无需进行接入改造。 |
| 不上架应用广场  （不推荐） | 计算巢 | * ECS * SLB（目前只支持CLB） * RDS * EIP | 劣势：只可通过自有渠道进行推广。 | 商品二维码线下开通 |
