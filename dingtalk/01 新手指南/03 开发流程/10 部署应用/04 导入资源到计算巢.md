更新于 2026-01-22本文档指导如何将云资源导入至钉钉计算巢服务，实现资源的统一管理和安全协作。适用于已完成企业认证、具备系统管理员权限，并已创建具备API调用能力的第三方企业应用的开发者。

## 前置条件

在开始操作前，请确保满足以下条件：

1. 已完成钉钉企业认证；
2. 已创建**第三方企业应用**，并获取该应用的 `AgentId` 和调用凭证（Client ID、Client Secret）；
3. 当前操作账号为钉钉企业**系统管理员**，具备应用配置与部署权限；
4. 已在阿里云侧拥有对应权限的账号，且支持HTTPS协议访问；
5. 目标云资源（如ECS、RDS等）位于支持的地域，建议优先使用华东1（杭州）、华北3（张家口）等主流区域。

> **提示**：如何确认已关联阿里云账号？请参考文档[钉钉账号绑定阿里云账号](/document/dingstart/bind-a-dingtalk-account-to-an-alibaba-cloud-account)。

## 准备工作

* 确保企业通讯录已完成数据安全保障设置；
* 推荐提前梳理需导入的云资源列表，包括SLB、ECS、RDS、EIP等核心实例；
* 确认目标应用已通过安全检测，并具备调用计算巢相关API的权限。

## **操作步骤**

1. 登录[开发者后台](https://open-dev.dingtalk.com/)，单击**应用开发** > **第三方企业应用** > **企业应用**，选择目标应用，进入应用详情页。

   > **注意**：请选择已通过安全检测且具备API调用权限的第三方企业应用。
2. 单击**应用发布** > **应用部署**，选择合适的部署方式。
3. 关联阿里云账号，单击**选择账号**：

   * 如果已关联阿里云账号，你可以直接选择检测到的阿里云账号并单击**确认**。
   * 如果未关联阿里云账号**，**此时会打开一个新的页面，你可以单击**关联阿里云账号** > **确认。**然后在应用部署页面，重新选择检测到的阿里云账号并单击**确**认。详情参考[钉钉账号绑定阿里云账号](/document/dingstart/bind-a-dingtalk-account-to-an-alibaba-cloud-account)。
4. 选择账号后，勾选**同意《计算巢服务安全检测授权协议》，**并单击**授权计算巢。**
5. 单击**资产管理**，跳转关联计算巢**应用管理。**
6. 进入计算巢**应用管理。**

   进入应用管理后，请根据云资源所在的地域创建对应地域的**应用分组**，支持分别创建多个地域的应用分组。

   ![[dingstart-import-resources-compute-nest_p986945.png]]

   例如云资源在杭州，点击“**创建应用分组**”，自定义分组名称，地域选择“**华东1（杭州）**”，在创建过程时可以选择需要导入的云资源，默认应导入SLB、ECS、RDS、EIP等云资源实例。

   ![[dingstart-import-resources-compute-nest_p986949.png]]
7. 完成应用分组创建后，也可以支持通过“**资源管理**”的“**导入资源**”选择需要导入的云资源及实例。

   ![[dingstart-import-resources-compute-nest_p700849.png]]

## 重要

1. 若需使用RDS数据推送功能，需在计算巢应用中导入**华东1**或**华北3**区域的RDS实例，随后在应用部署的RDS页面配置推送数据源。
2. 私网SLB支持的地域与可用区如下表所示：

   |  |  | | --- | --- | | 地域 | 可用区 | | 华东1（杭州） | 可用区B、可用区E、可用区F、可用区G、可用区H、可用区I、可用区J、可用区K | | 华东2（上海） | 可用区B、可用区E、可用区F、可用区G、可用区L、可用区M、可用区N、可用区I、可用区H | | 华北2（北京） | 可用区B、可用区C、可用区D、可用区E、可用区F、可用区G、可用区H、可用区I、可用区K、可用区L | | 华北3（张家口） | 可用区A、可用区B、可用区C | | 华南1（深圳） | 可用区D、可用区E、可用区F |

## **后续步骤**

资源导入完成后，你需要[配置并更新安全域名](/document/dingstart/configure-update-secure-domain-name)，以确保前端服务可正常访问。
