更新于 2026-01-22在资源导入完成后，您需要配置钉钉安全域名以确保应用服务的正常通信与安全性。本文档将指导开发者完成安全域名的配置与更新流程，适用于通过计算巢部署、使用私网SLB并集成钉钉API。请按照以下步骤操作，并注意相关权限和网络要求。

## 适用对象

本文档面向**第三方企业应用**开发者，需具备以下角色权限：

* 拥有钉钉企业管理员权限，可进入开发者后台进行应用发布与配置。
* 能访问阿里云计算巢控制台，管理SLB实例与ECS资源。
* 可查看应用的AppKey/AppSecret等凭证信息。

## 前提条件

开始前，必须确保开发者后台关联的阿里云（计算巢部署）的云账号下已购买SLB，并且SLB配置了[HTTP协议的监听端口](https://help.aliyun.com/document_detail/85949.html?spm=a2c4g.86438.0.0.19195491uwAdyO)，目前只支持CLB。

**重要**

**禁止使用 HTTPS 常用端口（如443、8443）或将HTTP重定向至HTTPS**，否则可能导致回调失败或审核不通过。

> **重要提醒**：安全域名绑定依赖于SLB的稳定性和合规性，建议提前完成SLB健康检查配置，确保后端服务可达。

## **操作步骤**

1. 登录开发者后台，进入应用详情页，依次点击 **应用发布 > 应用自检**。
2. 选择要关联的私网SLB，然后勾选授权协议，再单击**关联**。
3. 关联完成后，等待域名审核和域名配置初始化。

   **说明**

   安全域名绑定申请提交后，需要等待钉钉安全审核，通常需要1~2个工作日完成审核。审核通过后流程会自动绑定私网SLB到安全域名，生效时间在1~2个工作日，请留意状态更新。请求被拒绝后请修改并重新提交申请。
4. 应用中更新安全域名：

   * 若推送类型是SyncHTTP，要求在回调URL的推送地址中使用钉钉安全域名。
   * 开发者服务端接口域名，要求使用钉钉安全域名。
   * 若应用属于非前后端分离架构，要求应用首页地址使用钉钉安全域名。

## **后续操作**

将应用服务端程序部署至[计算巢中的ECS](https://help.aliyun.com/document_detail/444595.html?spm=a2c4g.290076.0.0.60715167ZJIyYr)，确保其可通过私网SLB访问。

## 常见问题与注意事项

|  |  |  |
| --- | --- | --- |
| 问题 | 原因 | 解决方案 |
| 审核失败 | 使用了443/8443端口或启用了HTTP重定向 | 修改SLB监听规则，禁用HTTPS重定向 |
| 回调超时 | 后端服务响应慢或不可达 | 检查ECS网络策略、SLB健康状态及服务日志 |
| 域名未生效 | 审核尚未完成 | 等待1~2个工作日，勿重复提交 |
