更新于 2025-10-24本文介绍如何使用钉钉鸿蒙OAuth SDK接入钉钉授权登录。

## **准备工作**

在使用鸿蒙接入钉钉分享之前，你需要完成以下准备工作：

* 搭建 鸿蒙 开发环境。
* 安装钉钉客户端 8.0.6 及以上的版本。
* Demo参考 [OpenAuthDemo.zip](https://help-static-aliyun-doc.aliyuncs.com/file-manage-files/zh-CN/20250911/kvhldz/OpenAuthDemo.zip)。

**说明**

下载并使用本SDK，即表示您已阅读并同意《[钉钉开放平台隐私权政策](/document/services/open-latform-privacy-policy)》、《[钉钉开放平台开发者服务协议](/document/services/open-platform-developer-service-agreement)》。

## **步骤一：接入钉钉分享**

1. 登录[开发者后台](https://open-dev.dingtalk.com/#/shareMan)，创建应用。

   ![[dingstart-hongmeng-application-authorization_p1005967.png]]
2. 找到自己的创建的应用，单击应用进入应用详情页。

   ![[dingstart-hongmeng-application-authorization_p1005965.png]]
3. 在应用信息页面，获取应用的 **Client ID**。

   ![[dingstart-hongmeng-application-authorization_p1005963.png]]
4. 在应用详情页，单击**分享设置**，进入登录与分享页面。

   ![[dingstart-hongmeng-application-authorization_p1005966.png]]
5. 在**接入分享**栏，单击右侧**编辑**。

   ![[dingstart-hongmeng-application-authorization_p1005964.png]]
6. 单击开启 鸿蒙**分享**，然后依次填写鸿蒙应用 **Bundle Name**和 **Identifier**，最后单击保存。

   **说明**

   **Identifier** 为 鸿蒙应用唯一标识 **appIdentifier**而不是 appId，详情见官方文档：[BundleInfo](https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V14/js-apis-bundlemanager-bundleinfo-V14)

   ![[dingstart-hongmeng-application-authorization_p1005961.png]]

## **步骤二：搭建开发环境**

DevEco Studio 环境下：

1. 在oh-package.json5文件中，添加@dingtalk/openauth依赖：

   ```json
   {
     "name": "entry",
     "version": "1.0.0",
     "description": "",
     "main": "",
     "author": "",
     "license": "",
     "dependencies": {
       "@dingtalk/openauth": "1.1.0"
     }
   }
   ```
2. 在module.json5中添加querySchemes及requestPermissions权限配置。

   ```json
   {
     "module": {
       "querySchemes": [
         "dingtalk"
       ],
       "requestPermissions": [
         {
           "name" : "ohos.permission.INTERNET"
         }
       ]
     }
   }
   ```

## **步骤三：发送请求到钉钉**

执行下方代码：

```javascript
import { DDAuthApiFactory, AuthLoginParam } from '@dingtalk/openauth';

let param:AuthLoginParam = {
  // 钉钉开放平台应用标识
  appId: "Your AppId",
  // 创建应用时填写的回调域名，仅用于校验域名是否一致，不会跳转
  redirectUri: "Your redirectUri",
  // 返回时拉起的Ability页面
  callBackAbilityName: "EntryAbility",
  // 固定值code，授权通过后返回authCode
  responseType: "code",
  // 应用授权作用域 openid：授权后可获得用户openid, openid%20corpid：授权后可获得用户openid和登录过程中用户选择的组织corpId
  scope: "openid",
  // 固定值为consent，首次会进入授权确认页
  prompt: "consent",
  // 用于保持请求和回调的状态，授权请求后原样带回给第三方
  state: "state"
}
DDAuthApiFactory.createDDAuthApi(param).authLogin(component)
```

**参数说明：**

|  |  |  |
| --- | --- | --- |
| 参数 | 是否必填 | 备注 |
| appId | 是 | 钉钉开放平台应用标识，即第一步中获取的ClientId。 |
| redirectUri | 是 | 创建应用时填写的回调域名，仅用于校验域名是否一致，不会跳转。  **说明**  需要与创建应用时填写的回调域名保持一致。 |
| responseType | 是 | 当前只支持固定值code，授权通过后返回authCode。 |
| scope | 是 | 应用授权作用域。授权页面显示的授权信息以应用注册时配置的为准  当前只支持两种输入：   * openid：授权后可获得用户oepnid。 * openid%20corpid：授权后可获得用户openid和登录过程中用户选择的组织corpId。 |
| prompt | 是 | 固定值为consent，会进入授权确认页。 |
| callBackAbilityName | 是 | 授权成功后显式Want唤起指定的Ability。  **说明**  Ability需要在module.json5中配置"exported": true |
| nonce | 否 | 字段值任意填写，授权登录后原样返回。 |
| state | 否 | 用于保持请求和回调的状态，授权请求后原样带回给第三方。  **说明**  该参数可用于防止csrf攻击（跨站请求伪造攻击），建议第三方带上该参数，可设置为简单的随机数加session进行校验。 |
| targetBundleName | 否 | 指定的钉钉BundleName, 用于指定跳转钉钉专属版，不填时默认跳转钉钉标准版 |
| webForceLogin | 否 | true时钉钉未安装时不保留登录状态，每次授权都要求重新登录，默认为false |

## **步骤四：接收授权结果**

在步骤三内填写的callBackAbilityName对应的Ability内接收授权结果：

```javascript
export default class EntryAbility extends UIAbility {

  onNewWant(want: Want, launchParam: AbilityConstant.LaunchParam): void {
    if (want?.parameters?.action === "hmos.dingtalk.oauth2") {
      const authCode = want?.parameters?.authCode as string
      const state = want?.parameters?.state as string
      const error = want?.parameters?.error as string
      if (authCode) {
        promptAction.showToast({
          message: `授权成功 ${authCode}`
        });
      } else {
        promptAction.showToast({
          message: `授权失败 ${error}`
        });
      }
    }
  }
}
```

**返回值说明**：

|  |  |
| --- | --- |
| 返回值 | 说明 |
| error | 取值：   * **ERR\_OK = 0**：用户同意 * **ERR\_AUTH\_DENIED = -4**：用户角色授权 * **ERR\_USER\_CANCEL = -2**：用户取消 |
| authCode | 授权码，可换取个人用户accessToken的凭证。  **说明**  authCode只能使用一次。 |
| state | 调用方标识其请求的唯一性的标志，state字符串长度不能超过1K。 |
