更新于 2026-01-29本文档指导开发者如何在钉钉酷应用中接入消息菜单功能，实现通过消息快速触发应用操作。

## **前提条件**

* 已完成[创建酷应用](/document/dingstart/create-coolapp)流程，获得可配置的应用实例。
* 应用类型须为支持消息菜单能力的类型（如企业内部应用、第三方企业应用）。

## **操作步骤**

1. 填写消息菜单信息：

   1. 在 **基本信息 > 简介** 区域填写应用简要描述。
   2. 配置快捷操作项：

      |  |  | | --- | --- | | **配置项** | **说明** | | 操作名称 | * **填写说明：**名称长度不超过6个字。 * **填写示例：**如提交需求。 | | 桌面端访问地址 | * **填写说明：**请输入https://或dingtalk://协议地址。 * **填写示例**：如https://open.dingtalk.com/developer。 **说明**  dingtalk协议参见[群相关跳转协议](/document/dingstart/group-jump-protocol)。 | | 移动端访问地址 | * **填写说明：**请输入https://或dingtalk://协议地址。 * **填写示例**：如https://open.dingtalk.com/developer。 **说明**  dingtalk协议参见[群相关跳转协议](/document/dingstart/group-jump-protocol)。 |
   3. 完成配置后，单击【保存】按钮，系统将校验输入内容并暂存配置。
2. 保存成功后，单击【全量发布】按钮，使消息菜单对所有成员生效。

## **后续步骤**

* 消息菜单配置发布完成后，仍需完成[发布应用](/document/dingstart/publish-dingtalk-application)，确保应用对目标用户可见可用。
* 若需获取用户通过消息菜单选中的原始消息内容，可通过调用[客户端API-获取消息内容](/document/development/message-menu-api)API实现。
