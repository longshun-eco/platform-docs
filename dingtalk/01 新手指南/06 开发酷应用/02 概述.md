## **什么是酷应用**

在**企业内部应用**中，能让应用随处可见、随手可用。酷应用是一种全新的场景化应用，将应用功能与钉钉内多产品场景融合，打造沉浸式使用体验，开创全新交互方式，提升数据赋能能力，从人找应用到应用找人，打造场景化协同新体验。更多详情参见[酷应用](https://open.dingtalk.com/coolapp)。

在**第三方企业应用**中，**酷应用**让用户在钉钉上更便捷地使用应用，多步变一步，从“**人找应用**”到“**应用找人**”，从“**人找数据**”到“**数据找人**”，从“**被动工作**”到“**主动探索**”。首先通过视频和一些场景快速了解酷应用，以及酷应用如何让用户在钉钉上更便捷地使用应用。

[](https://help-static-aliyun-doc.aliyuncs.com/file-manage-files/zh-CN/20220325/llab/钉钉酷应用.mp4)

**场景一：**例如，几个员工出差，使用不同的系统提交不同的信息，步骤非常繁琐。员工首先需要拉群各自提交信息，使用订票软件订机票，切换软件订酒店，切换软件打车出行等，如果需要申报行程、申请款项等，需要切换公司系统，完成所有步骤后再通知其他员工。

使用酷应用，员工不用切换软件，在群里轻松点击就能快速预定机票、酒店、预约车辆。如果行程跟之前的同事一样，直接点击 +1，一键同行。行程数据也会跟公司内部系统自动同步，不用员工再手动操作。

**场景二：**员工小明正在群聊中处理某个项目，小明的同事A在跟其他同事沟通这个项目；同事B可能正在写一份项目相关文档，同事C在钉钉工作台，同事D能正在查看日历。小明只需要在群聊中更新任何项目数据，同事ABCD都可以实时看到，并且不用跳出所在界面，直接做出响应。

**酷应用**可以将应用与应用、应用与人、人与人之间的数据进行同步，改变员工的工作方式，为更多人带来价值。酷应用也鼓励更多员工以数字化方式和思维来开展工作，让用户在一种沉浸式的炫酷体验中，共同看见问题，共同解决问题，共同实现目标。

![[dingstart-coolapp-overview_p422761.png]]

**酷应用**是一种基于沉浸式、场景化、网络协同和数据普惠理念的全新应用形态。不同于传统TOB应用，由比较复杂的功能、单据、流程和报表组成，酷应用使用非常轻量化的方式、将复杂功能、流程和报表解构、碎片化、千人千面化，让这些功能出现在高频的协作场景中，比如单聊、群聊、日程、文档、搜索、工作台等。第三方企业应用酷应用示例如下。

## **酷应用类型**

|  |  |
| --- | --- |
| **场景** | **说明** |
| [扩展到群会话](/document/dingstart/group-chat-coolapp-overview) | 群聊酷应用可以被安装到群中，一旦安装完成，群内成员就可以在沟通过程中高效地查看和使用酷应用提供的数据和功能，以便更好地完成业务协作。 |
| [扩展到消息菜单](/document/dingstart/message-menu-coolapp-overview) | 消息菜单能力可以通过长按消息来唤起消息菜单，并通过菜单中的功能选项对消息进行操作，例如消息转发、消息转日程、消息转任务等。同时，开发者可以根据自己的业务场景进行能力定制，新增快捷操作功能到消息菜单中。这样可以使消息菜单更加便捷和灵活地满足用户的需求。 |
| [链接增强](/document/dingstart/link-enhancements-coolapp-overview) | 链接增强能力可以实现在钉钉聊天中的链接转换功能。一旦接入该功能，链接会自动被识别并转换为卡片形式显示。用户可以直观地查看卡片中的内容信息，并且可以通过卡片上的交互功能直接进行业务操作。 |
| [扩展到单聊会话](/document/dingstart/private-chat-coolapp-overview) | 单聊酷应用可以被安装到人与人的聊天中，一旦安装完成，应用可以在单聊会话实现任务协同、目标共创、商机共享、战报通告等沉浸体验。 |
| [扩展到成员资料页](/document/dingstart/member-profile-coolapp-overview) | 成员资料页酷应用可以被安装到成员个人资料页，在我们的日常工作中，我们经常需要查看某个人的成员资料页，例如查看他的个人信息、所属组织等。然而，这些信息都是钉钉平台原生提供的，外部用户希望在成员资料页上扩展更多信息，例如员工的上下班情况、销售业绩展示等。成员资料页扩展功能可以实现成员资料页的扩展能力，钉钉提供了一个扩展卡片区域的能力，开发者可以利用该卡片区域丰富的渲染和交互能力，实现更多业务需求的想象。 |
| [扩展到文档](/document/dingstart/develop-document-coolapp-overview) | 文档酷应用可以被安装到钉钉文档内，它可以帮助我们将重复又耗时的编辑工作转变成自动化任务。一旦接入了文档酷应用，它可以及时地帮助用户感知指定的操作，并快速进行文档内容的读写和校验。这样，我们就能够更高效地处理文档相关的任务，节省时间和精力。  **说明**  文档酷应用支持在**钉钉表格**中使用，提供了扩展钉钉表格右侧面板和弹窗容器的能力，开发者可以借助其丰富的渲染能力和交互能力实现更多文档相关业务。 |

## **企业内部应用**

### **最佳实践**

* 钉钉酷应用打通了OKR群应用的场景，实现轻量级便捷体验，让OKR落地协同更高效。详情参见[酷应用开启OKR线上管理的新模式](https://open.dingtalk.com/abilities/182?spm=ding_open_doc.document.0.0.56d9722fz98cBG)。
* 钉钉酷应用通过任务卡片、项目吊顶、项目日报、我的待办等功能，打造一款轻量级项目管理酷应用。详情参见[如何设计一款项目管理酷应用](https://open.dingtalk.com/abilities/184)。
* 钉钉酷应用实现全渠道客户连接、客户全生命周期管理与精细化销售过程管理。详情参见[销帮帮酷应用帮助企业销售团队实现精细化管理](https://open.dingtalk.com/abilities/203)。

## **第三方企业应用**

### **酷应用会带来哪些改变**

* **寻求组织治理和员工体验之间的平衡**

  近些年，大家对组织管理或治理模式的讨论较多，各种提法例如：金字塔或扁平化、管控或赋能，基本是以对立的视角在看过去和未来的组织设计，认为这是两种不同的管理或治理模式。

  未来的组织不再是简单的管控，而是要在清晰的业务战略下如何自上而下梳理组织的文化、机制和流程，并以此为出发点去规划数字化的体系。例如，是否需要财务共享、人力资源共享，升级客户服务中心，是否要启动OKR、人才盘点、员工积分激励计划，是否要发布灵活的固定资产管理制度，根据不同的业务属性是否要升级员工激励和晋升制度等。而且这些制度也不是大而全的，对于多样化经营的组织，还可以根据组织属性、业务属性做多样化、差异化的设计。

  未来的组织也不再是简单的扁平化或信息透明化，核心是如何让员工自然的融入到业务和组织体系中，既能让机制和流程规则得以有效实施，又能让员工感受不到明显的机制和规则约束。基于钉钉上的组织规模、高频协作场域和面向场景设计的酷应用，可以让机制、流程和规则成为组织高效、灵活运转背后那只无形的手，解决很多组织目前面临的困境。

  ![[dingstart-coolapp-overview_p422749.png]]
* **通过高频协同场域带动软件的低频，加速数字化的渗透**

  过去企业花费大力气，想要加速消息的传递、数据的流转、业务的推进，为此建立了多个烟囱林立的信息化系统，钉钉解决了这些问题。钉钉首先是一个IM平台，解决了沟通信息传递的问题。其次是一个协作平台，协作的基础是数据能自由流转且一致，低代码让钉钉成为一个应用开发平台。而酷应用，则进一步为企业应用开辟了一个高频使用的入口。

  ![[dingstart-coolapp-overview_p422748.png]]
* **从微服务走向微应用，解决扩展难题**

  在所有的逻辑计算方式中，有两种思维方式，一种叫 Top Down，一种叫 Bottom Up。Top Down 是从最高层开始，一层层分解，最终得出整个蓝图。TOB复杂应用大部分是通过Top Down的方式设计出来的，比如ERP里各模块的划分、功能流程的设计；场景化应用则需要采用Bottom up的思路。

  酷应用想要真正实现沉浸式和高活跃，需要把两者有机的结合起来，对于中后台的服务，还是需要采用Top Down的方式，做领域服务的设计沉淀。对于酷应用交互的设计，一定要采取Bottom up，需要从用户实际协作场景出发，酷应用终究会驱动中后台服务和端侧交互创新走向截然不同的风格。

### **酷应用和低代码/宜搭的关系**

钉钉之前提出的**低代码革命**是对ToB应用开发方式的一次革新，而**酷应用**则是对ToB应用使用方式的一次革新。

酷应用是对传统软件体系的解构， 也是对场景化需求的重新整合，让应用或工具触手可及，让应用及数据能力普惠到员工。 [宜搭](https://www.aliwork.com/)或钉钉搭从低代码角度，以一种低门槛的方式让更多懂业务的人可以动手搭建应用，随着酷应用理念的推广和被广泛认可，宜搭或钉钉搭等低代码平台也必将成为酷应用工厂，让酷应用的搭建更加简单。

### **如何设计酷应用**

酷应用设计源文件已开放。包含酷应用设计建议及通用样式规范，助力应用酷化体验更优。以及群置顶 、动态卡片、工作台、个人profile、移动端半屏容器和桌面端侧边栏等6个场域设计建议和设计案例模板。可前往[酷应用设计规范](https://ding.design/#/cate/1/page/818)查看下载。

#### **酷应用设计指南**

1. 明确应用最核心的价值优势。
2. 设计恰当的欢迎体验。
3. 选择合适酷应用的载体，并设计对应内容。
4. 卡片信息简洁，易于浏览。
5. 信息跟随交互实时变化。
6. 按钮位置建议。
7. 覆盖多端，多端交互符合使用习惯。
8. 适配深色模式
