## **开发需知**

在正式进入开发前，建议开发者优先阅读以下基础内容，全面了解钉钉开放平台的核心概念与权限体系：

* **基础概念**：了解钉钉开放平台的整体架构、身份模型（如企业、用户、组织）、应用上下文等关键术语。
* **应用类型介绍**：掌握企业内部应用、第三方企业应用、个人应用等不同类型的应用场景与部署方式。
* **应用能力介绍**：熟悉钉钉提供的各类开放能力，包括消息通知、身份认证、日程管理、文件协作、事件订阅等。
* **申请开发者权限**：指导如何注册开发者账号并获取相应的API调用权限。

> 提示：所有开发操作均需基于已配置完成的应用实例，确保具备正确的 `Client ID` 和 `Client Secret`，并在“权限管理”中添加所需接口权限。

## 关键术语说明

为便于理解后续内容，以下列出常用专业术语及其定义：

|  |  |  |
| --- | --- | --- |
| 术语 | 定义 | 获取路径 |
| **AppKey / Client ID** | 应用的唯一标识符，用于识别调用方身份 | 在应用详情页 > **基础信息 > 凭证与基础信息** 中查看。 |
| **AppSecret / Client Secret** | 应用密钥，用于生成 `access_token` | 同上，首次生成后需妥善保管。 |
| **authCode** | 临时授权码，用户登录后由前端返回，用于换取用户身份信息 | 通过 JSAPI 或 SDK 调用免登接口获得。 |
| **userId** | 钉钉企业内用户的唯一标识，在单个企业范围内有效 | 调用接口获取。 |
| **unionId** | 用户在多个关联企业中的全局唯一标识 | 用户首次登录时可通过 authCode 换取。 |
| **access\_token** | 接口调用凭证，大多数 API 需携带此令牌才能访问 | 使用 `Client ID` 和 `Client Secret` 调用接口获取。 |

## **工作台接入**

工作台是企业员工日常办公的核心入口。通过自定义工作台与组件集成，可将业务系统深度嵌入钉钉主界面，实现统一门户体验。

工作台目录中，主要包含自定义工作台、组件的搭建流程，并提供了合作伙伴相关的工作台模板及第三方低码组件、第三方全码组件、共享组件三种不同教程，可供开发者进行选择。
