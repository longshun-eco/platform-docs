## biz\_type=50

家校通讯录2.0，部门变更推送。subscribe\_id, corp\_id, biz\_id, biz\_type组成唯一索引。

|  |  |
| --- | --- |
| 字段 | 说明 |
| subscribe\_id | 第三方企业应用的suiteid加下划线0。 |
| corp\_id | 开通第三方企业应用的企业corpid。 |
| biz\_id | 无业务含义，只是与其他字段组成唯一索引。  拼接规则：d\_${id}\_${action} |
| biz\_type | 业务类型。家校通讯录部门变更的值为50。 |
| biz\_data | 数据为Json格式。 |

## biz\_type=51

家校通讯录2.0，人员变更推送。

subscribe\_id, corp\_id, biz\_id, biz\_type组成唯一索引。

|  |  |
| --- | --- |
| 字段 | 说明 |
| subscribe\_id | 第三方企业应用的suiteid加下划线0。 |
| corp\_id | 开通第三方企业应用的企业corpid。 |
| biz\_id | 无具体业务含义，只与其他属性构成唯一索引。  拼接规则：   * 人员关系事件：u\_r\_${id}\_${action} * 监护人身份事件：g\_${id}\_${action} * 学生身份事件：s\_${id}\_${action} * 老师身份事件：t\_${id}\_${action}   示例：g\_1233\_insert |
| biz\_type | 业务类型，家校通讯录人员变更的值为51。 |
| biz\_data | 数据为Json格式。 |

家校通讯录人员变更，主要包括两部分：人员在家校业务场景下的身份以及用户关系。

**说明**

关系目前只有监护人与学生的关系。

* **人员身份biz\_data说明**

* **人员关系biz\_data说明**

  + **edu\_user\_relation\_insert**

    此事件中各个字段的理解，可以总结为如下一句表达式：在班级${classId}中，${from}是${to}的 ${relationName}。

    示例：在班级12345中，"123457896"是"34567890"的"爸爸"。在家校场景下，from为监护人，to为学生。

    |  |  |  |  | | --- | --- | --- | --- | | 名称 | **类型** | **示例** | **说明** | | from\_userid | String | 123457896 | 用户的userid。 | | to\_userid | String | 34567890 | 用户的userid。 | | class\_id | Number | 12345 | 班级ID。 | | relation\_code | String | F | 关系code。 | | relation\_name | String | 爸爸 | 关系名。 | | syncAction | String | edu\_user\_relation\_insert | 事件类型。 | | errcode | Number | 0 | 返回码。 | | errmsg | String | ok | 返回码描述。 |
  + **edu\_user\_relation\_update**

    此事件中各个字段的理解，可以总结为如下一句表达式：在班级${classId}中，将${from}与${to}的关系更新为 ${relationName}。

    示例：在班级12345中，将"123457896"的"34567890"关系更新为"妈妈"。在家校场景下，from为监护人，to为学生。

    |  |  |  |  | | --- | --- | --- | --- | | 名称 | **类型** | **示例** | **说明** | | from\_userid | String | 123457896 | 用户的userid。 | | to\_userid | String | 34567890 | 用户的userid。 | | class\_id | Number | 12345 | 班级ID。 | | relation\_code | String | M | 关系code。 | | relation\_name | String | 妈妈 | 关系名。 | | syncAction | String | edu\_user\_relation\_update | 事件类型。 | | errcode | Number | 0 | 返回码。 | | errmsg | String | ok | 返回码描述。 |
  + **edu\_user\_relation\_delete**

    此事件中各个字段的理解，可以总结为如下一句表达式：在班级${classId}中，将${from}是${to}的 ${relationName}的关系删除。

    示例：在班级12345中，删除"123457896"是"34567890"的"爸爸"这样一层关系。在家校场景下，from为监护人，to为学生。

    |  |  |  |  | | --- | --- | --- | --- | | 名称 | **类型** | **示例** | **说明** | | from\_userid | String | 123457896 | 用户的userid。 | | to\_userid | String | 34567890 | 用户的userid。 | | class\_id | Number | 12345 | 班级ID。 | | relation\_code | String | M | 关系code。 | | relation\_name | String | 妈妈 | 关系名。 | | syncAction | String | edu\_user\_relation\_delete | 事件类型。 | | errcode | Number | 0 | 返回码。 | | errmsg | String | ok | 返回码描述。 |
