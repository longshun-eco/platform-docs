更新于 2025-10-16本文可以帮助你了解第三方企业应用事件与回调的流程，通过本文你将了解如何配置第三方企业应用的事件以及如何接收第三方企业应用的事件回调内容。

## 教程介绍

本教程以事件订阅中企业内部用户变更事件为例展示第三方企业应用事件与回调的流程，通过本教程您将学习到：

* 配置第三方企业应用事件订阅。
* 接收第三方企业应用的事件回调内容。

## 接入流程

步骤一：创建第三方企业应用。

步骤二：设置开发管理。

步骤三：配置事件与回调。

步骤四：设置体验组织授权开通（可选）。

步骤五：测试事件订阅与回调（可选）。

## 准备工作

在开始本文前，确保你已经完成了以下准备工作：

* 入驻成为产品方案商，详情请参考文档[入驻成为产品方案商](/document/isvapp/become-an-application-service-provider)。
* 需要成为钉钉开发者，详情请参考[成为钉钉开发者](/document/isvapp/become-a-dingtalk-developer)。

## 步骤一：创建第三方企业应用

在本部分，你需要在开发者后台创建一个第三方企业应用-H5微应用。

**说明**

如果已有第三方企业应用，可跳过此步骤。

1. 登录[钉钉开发者后台](https://open-dev.dingtalk.com/#/index)。
2. 在**开发者后台**页面，选择**第三方企业应用**，然后单击**创建应用**。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p439989.png]]
3. 在弹出的创建应用页面中填写基本信息，单击确定创建。

   * 应用类型：选择H5微应用。

     ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p379419.png]]
4. 应用创建完成后，在**基础信息**-**应用信息**页面，可以查看应用的**SuiteKey**和**SuiteSecret**。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p439986.png]]

## 步骤二：设置开发管理

1. 单击**基础信息 > 开发管理**进入开发管理页面，单击**修改**，并根据以下内容配置开发信息。

   * **服务器出口IP：**输入调用钉钉服务端API时使用的IP即企业服务器的公网IP，多个IP请以英文逗号","隔开，支持带一个\*号通配符的IP格式。
   * **应用首页地址：**请输入http或https开头的网址链接，如`https://open.dingtalk.com/document/`。
2. 添加完成后，单击**保存**。

![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p439992.png]]

## 步骤三：配置事件与回调

单击**应用功能 > 事件与回调**进入事件回调页面。

1. 选择对应的推送方式，本文选择SyncHttp推送。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p439997.png]]
2. 设置对应信息。

   * 生成加密Aes\_key，回调消息内容的加解密参数，是AES密钥的Base64编码。
   * 生成签名Token，钉钉每次向你的地址推送事件数据时都会携带`token`，用于生成签名、校验回调请求的合法性。必须为英文或数字，长度为3~32个字符。
   * 设置回调请求地址，用于接收事件订阅请求的URL。当应用订阅的事件触发时，钉钉会向该网址发送相应的 HTTP POST 请求。

     ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440017.png]]
3. 校验回调有效性。

   **说明**

   接收事件回调的 URL，必须是公网可以访问的url地址，需保证URL地址所在服务可以正常访问。

   测试阶段，可使用frp内网穿透测试工具，详情请参考[frp内网穿透工具](/document/resourcedownload/alibaba-cloud-frp-intranet-penetration-tool)。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440008.png]]

   **校验有效性流程：**

   1. 引入pom依赖文件。

      ```
              org.springframework.boot
              spring-boot-starter-parent
              2.1.4.RELEASE

              UTF-8
              8
              8

                  org.springframework.boot
                  spring-boot-starter

                  org.springframework.boot
                  spring-boot-devtools
                  true
      ```
   2. 引入消息加解密类，详情请参考[dingtalk-callback-Crypto](https://github.com/open-dingtalk/dingtalk-callback-Crypto)。
   3. 接收并响应回调事件。

      ```java
      @RestController
      @RequestMapping("/")
      public class CallbackController {

          private final Logger log = LoggerFactory.getLogger(getClass());

      //*
      //     * 创建应用，验证回调URL创建有效事件（第一次保存回调URL之前）

          private static final String EVENT_CHECK_CREATE_SUITE_URL = "check_create_suite_url";

      //*
      //     * 创建应用，验证回调URL变更有效事件（第一次保存回调URL之后）

          private static final String EVENT_CHECK_UPADTE_SUITE_URL = "check_update_suite_url";

      //*
      //     * suite_ticket推送事件
      ```
4. 点击**保存**，查看回调URL所在服务，保存推送的suiteTicket参数，调用服务端接口时需要使用。

   **说明**

   以下数据为解密biz\_type=2的suiteTicket最新状态，5个小时重新推送一次，详情请参看[数据格式](/document/isvapp/data-formats)。

## 步骤四：设置体验组织授权开通（可选）

单击**部署与发布 > 版本管理与发布**进入页面。

1. 授权体验组织，若无体验组织需创建体验组织。

   **说明**

   点击授权时务必保证回调URL所在的服务开启，否则无法授权成功。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p490962.png]]
2. 授权成功后，查看已授权体验组织。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p493574.png]]

## 步骤五：测试事件订阅与回调（可选）

**说明**

测试事件订阅与回调时务必保证回调URL所在的服务开启。

1. 申请相应的事件订阅的接口权限。本文以**通讯录> 企业内部用户变更**事件为例。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440034.png]]
2. 开启事件订阅。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440070.png]]
3. 事件订阅成功后，展示相应事件订阅。

   ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440071.png]]
4. 测试事件订阅。

   1. 登录体验组织[钉钉管理后台](https://oa.dingtalk.com/#/welcome)。
   2. 修改企业内员工信息。

      ![[development-third-party-enterprise-application-address-book-change-event-subscription-process_p440078.png]]
