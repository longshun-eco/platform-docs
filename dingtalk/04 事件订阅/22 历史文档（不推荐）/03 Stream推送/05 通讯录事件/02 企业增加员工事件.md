## 事件信息

| 名称 | 值 |
| --- | --- |
| 中文名称 | 企业增加员工事件 |
| 英文名称 | user\_add\_org |

## 功能描述

企业内部用户变更事件，eventType为user\_add\_org，表示企业发生员工增加时的推送的数据，字段值来自于[根据userId获取用户详情](https://open.dingtalk.com/document/isvapp/address-book-events)接口 。

## 支持应用类型

| 应用类型 | 是否支持 |
| --- | --- |
| 企业内部应用 | 不支持 |
| 第三方企业应用 | 支持 |

## **事件体描述**

### header部分

| 名称 | 类型 | 示例值 | 描述 |
| --- | --- | --- | --- |
| eventUnifiedAppId | String | bbb381b6-f01xxxxx58daac | 统一应用身份Id。 |
| eventCorpId | String | ding9f50b15bxxxx16741 | 事件所属的corpId。 |
| eventType | String | user\_add\_org | 事件类型。 |
| eventId | String | c7c7120f2c07419\*\*ebdba0318c8 | 事件的唯一Id。 |
| eventBornTime | Long | 1683533823336 | 事件生成时间。 |

### data部分(事件业务信息)

| 名称 | 类型 | 示例值 | 描述 |
| --- | --- | --- | --- |
| unionEmpExt | Object |  | 关联组织信息。  用户所在企业存在关联关系的企业，返回该字段。 |
| unionid | String | m8axYHBIiSxxxx | 员工在当前开发者企业账号范围内的唯一标识。 |
| exclusiveAccount | Boolean | false | 是否专属帐号。 |
| orderInDepts | String | {1234:12345} | 部门列表。 |
| dingId | String | $:LWCP\_v1:$LT | 钉钉ID。 |
| active | Boolean | true | 是否已经激活：   * true：已经激活 * false ：未激活 |
| errcode | Integer | 0 | 返回码。 |
| errmsg | String | ok | 返回码信息。 |
| avatar | String | http://xxxxx | 头像URL。 |
| isAdmin | Boolean | true | 是否为企业的管理员：   * true：是 * false：不是 |
| userid | String | user123 | 员工在当前企业内的唯一标识，也称staffId。 |
| isHide | Boolean | true | 是否号码隐藏：   * true：隐藏   隐藏手机号后，手机号在个人资料页隐藏，但仍可对其发DING、发起钉钉免费商务电话。   * false：不隐藏 |
| jobnumber | String | 12345 | 员工工号，对应显示到OA后台和客户端个人资料的工号栏目。 |
| isLeaderInDepts | String | {1:false} | 在对应的部门中是否为主管。  key是部门的ID，value是人员在这个部门中是否为主管，true表示是，false表示不是。 |
| isBoss | Boolean | false | 是否为企业的老板：   * true：表示是 * false：表示不是 |
| isSenior | Boolean | true | 是否开启高管模式:   * true：开启   开启后，手机号码对所有员工隐藏。普通员工无法对其发DING、发起钉钉免费商务电话。高管之间不受影响。   * false：不开启 |
| name | String | 钉钉 | 成员名称。 |
| position | String | 钉钉技术支持 | 职位名称。 |
| department | Array<Integer> | [1,2] | 数组类型，数组里面值为整型，成员所属部门ID列表。 |
| realAuthed | Boolean | true | 是否实名认证：   * true：是 * false：否 |
