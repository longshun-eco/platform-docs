## 事件信息

| 名称 | 值 |
| --- | --- |
| 中文名称 | 家校通讯录部门变更(部门节点更新) |
| 英文名称 | edu\_dept\_update |

## 功能描述

家校通讯录2.0部门信息变更，家校通讯录架构中各个部门发生变更时的信息,当eventType为edu\_dept\_update时数据为部门节点更新事件。

## 支持应用类型

| 应用类型 | 是否支持 |
| --- | --- |
| 企业内部应用 | 支持 |
| 第三方企业应用 | 支持 |

## **事件体描述**

### header部分

| 名称 | 类型 | 示例值 | 描述 |
| --- | --- | --- | --- |
| eventUnifiedAppId | String | bbb381b6-f01xxxxx58daac | 统一应用身份Id。 |
| eventCorpId | String | ding9f50b15bxxxx16741 | 事件所属的corpId。 |
| eventType | String | edu\_dept\_update | 事件类型。 |
| eventId | String | c7c7120f2c07419\*\*ebdba0318c8 | 事件的唯一Id。 |
| eventBornTime | Long | 1683533823336 | 事件生成时间。 |

### data部分(事件业务信息)

| 名称 | 类型 | 示例值 | 描述 |
| --- | --- | --- | --- |
| nick | String | 苹果班 | 部门别名。 |
| errcode | Integer | 0 | 返回码。 |
| chain | String | [123,456] | 从顶层部门到当前节点部门的部门链，其内容不包含当前节点。 如classic类型下的班级：[校区id,学段id,年级id]；如classic类型下的校区： []。 |
| feature | String | {"grade\_level":0,"start\_year":"2023"} | 部门feature，JSON格式。 各种节点部门存在不同的属性。 |
| name | String | 一年级2班（苹果班） | 部门名称。 |
| deptId | Long | 123456 | 部门ID。 |
| errmsg | String | ok | 返回码说明。 |
| contactType | String | classic | 家校通讯录类型：   * classic: 传统经典4层结构。校区/学段/年级/班级。 * custom：自定义结构，但是叶子节点仍旧是班级。 |
| deptType | String | grade | 部门类型：   * campus：校区/学院 * period：学段 * grade：年级 * class：班级 * dept：没有业务含义，只是一个部门节点。 |
