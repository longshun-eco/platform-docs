## 事件信息

| 名称 | 值 |
| --- | --- |
| 中文名称 | 审批任务开始，结束，转交 |
| 英文名称 | bpms\_task\_change |

## 功能描述

当审批事件发生审批任务开始、结束、转交时，推送给订阅者的内容。

## 支持应用类型

| 应用类型 | Stream模式推送 | HTTP推送 | SyncHTTP/RDS推送 |
| --- | --- | --- | --- |
| 企业内部应用 | 支持 | 支持 | 不支持 |

HTTP推送

| --- | --- | --- | --- |
| EventType | String | bpms\_task\_change | 事件英文名称。 |
| EventTime | Long | 1663143335567 | 事件发生的时间。 |
| CorpId | String | ding9f50b15bxxxx16741 | 企业corpId。 |
| BizId | String | 1663\*\*35567 | 无业务意义，幂等。 |
| eventId | String | c7c7120f2c07419\*\*ebdba0318c8 | 事件的唯一Id。 |
| processInstanceId | String | ce133dd0-5b22-9516-xxxxxxxxxxxx | 审批实例id。 |
| finishTime | Long | 1670983893000 | 结束任务的时间。时间戳，单位毫秒。  审批开始无该数据。 |
| createTime | Long | 1670983873000 | 创建任务的时间。时间戳，单位毫秒。 |
| processCode | String | PROC-27BBC5E6-DFFA-4EC3-A1F1-xxx | 审批模板的唯一码。 |
| bizCategoryId | String | attendance.goout | 业务类目。 |
| businessId | String | 20xxxx38 | 流程实例业务标识 |
| remark | String | 同意 | 操作时写的评论内容。 |
| type | String | finish | 任务状态变更类型：   * start：审批任务开始 * finish：审批任务正常结束（完成或转交） * cancel：说明当前节点有多个审批人并且是或签，其中一个人执行了审批，其他审批人会推送cancel类型事件 * comment：审批任务评论。 |
| title | String | 考勤-测试 | 实例标题。 |
| taskId | Long | 811165 | 任务id。 |
| staffId | String | 08058646137 | 用户userId：   * 当前任务的审批人userId * 操作转交动作的用户userId |
| result | String | agree | 审批结果：   * agree：同意 * refuse：拒绝 * redirect：表示审批任务转交 * audit：表示当前节点为办理人节点，audit为办理结果 |
