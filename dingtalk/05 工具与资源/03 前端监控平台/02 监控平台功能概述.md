## **简介**

### **背景**

随着前端应用复杂度不断提升，页面性能下降、JavaScript异常频发、接口调用失败等问题直接影响用户体验和业务转化率。传统的日志排查方式滞后且难以覆盖真实用户场景，无法满足高效定位问题的需求。

钉钉开放平台提供的前端监控服务，基于探针JSSDK实现对Web页面、H5应用、小程序等多端环境的实时监控，能够全面采集页面加载性能、JS运行错误、资源加载异常、API请求成功率等关键指标，助力团队构建稳定可靠的前端体验。

![[download-monitoring-platform-1_p328068.png]]

### **目的**

本监控平台旨在：

* 实时发现并定位前端异常（如JS报错、白屏、卡顿），缩短故障响应时间；
* 量化页面性能表现，辅助优化首屏加载速度与用户体验；
* 提供多维度数据分析能力，支持按操作系统、网络类型、客户端版本等维度下钻分析；
* 帮助开发者建立完善的前端质量监控体系，推动持续改进。

### **适用范围**

本平台适用于以下应用场景：

* 钉钉内部应用或企业自建应用中的H5页面、微应用、工作台入口页；
* 基于钉钉容器运行的小程序或融合通信类应用；
* 使用钉钉JSSDK进行功能开发的前端项目。

> **注意**：当前监控能力主要面向运行在钉钉客户端内的Webview环境，建议在HTTPS协议下部署页面以确保探针正常加载与数据上报。

## 用户指引

### 适用对象

* 前端开发工程师：负责集成探针SDK、配置采样规则、分析异常堆栈；
* 应用管理员：具备权限创建应用、获取AppKey、管理监控配置；
* 质量保障人员：通过大盘查看趋势数据，识别高频异常与性能瓶颈。

### 权限要求

* 需拥有**企业管理员**或**应用开发者**权限，方可完成应用创建与探针配置。
* 接入监控前需在钉钉开放平台注册应用，并开通相关API权限。

## 常见名词及解释

|  |  |
| --- | --- |
| 名词 | 说明 |
| **应用** | 平台下不同的应用区分。 |
| **页面性能** | 即页面完全加载时间，平台中统计的页面性能为 耗时均值。 |
| **2s快开比** | 页面完全加载时间≤2s的页面占比。  **说明**  完全加载时间≤2s的页面性能PV / 页面性能总PV \* 100% |
| **JS异常** | 即JS ERROR。 |
| **异常PV** | 在一个页面访问过程中，如果发生过JS异常，则此PV为一次异常PV。  例如：用户在一次访问过程中出现了三种不同的JS异常，则此次访问计为三次异常PV。 |
| **PV** | 访问量（PV）是Page Visit的简写，表示页面被访问了多少次。 |
| **UV** | 用户量（UV）是User Visit的简写，表示有多少独立用户进行了访问。 |
| **采样率** | 平台规定的监控指标的采样率。  默认所有指标日志均是100%上报，页面访问调用明细日志可设置为10%采样率。 |
| **影响用户** | JS异常等情况下，对影响用户排重计数。 |
| **网络类型** | 用户访问页面/发起网络请求的网络类型。 |
| **运营商** | 用户访问页面/发起网络请求的运营商。 |
| **操作系统** | 用户访问页面使用的设备操作系统。 |
| **客户端** | 用户访问页面使用的客户端应用。 |
| **前端版本** | 用户访问页面的发布版本。 |
| **探针版本** | 页面植入的前端监控JSSDK的版本，JSSDK统称「探针」。 |

## API

|  |  |
| --- | --- |
| 名词 | 说明 |
| **请求次数** | 在一个页面访问过程中，统计页面发起AJAX请求的次数。  例如：用户在一次页面访问过程中发起10个AJAX，则请求次数计为10次。 |
| **请求成功** | 请求返回状态码 为 2xx或304 。 |
| **请求失败** | 请求返回状态码 不为 2xx或304。 |
| **成功率** | 请求成功次数 / 总请求次数。 |
| **请求耗时** | 即AJAX请求完成的时间，平台中统计的请求耗时为 耗时均值。 |
| **失败影响用户** | 出现请求失败的用户数量。 |
| **失败URL** | 出现请求失败的AJAX URL。 |
| **采样率** | 默认100%上报，可设置为10%采样率。 |
