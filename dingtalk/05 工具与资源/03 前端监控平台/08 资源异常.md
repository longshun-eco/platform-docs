更新于 2026-01-08资源加载异常分析功能主要用于监控和诊断页面加载过程中出现的各类资源异常，涵盖 JS、CSS、图片等关键静态资源。通过多维度的数据统计与可视化图表，帮助开发者快速识别异常资源、定位问题根源，并结合网络环境、运营商、地域等因素进行深度分析。

## **使用场景说明**

该功能适用于以下典型业务场景：

* 当网站或应用出现大面积页面加载缓慢或白屏时，可通过资源异常分析确认是否由特定 JS 或 CSS 文件加载失败导致。
* 在用户反馈“按钮无响应”或“样式错乱”等问题时，可快速排查是否因核心脚本或样式表资源加载异常引起。
* 运维人员在日常巡检中发现性能指标波动，可利用此工具下钻至具体资源域名和请求链路，判断是 CDN 故障、第三方服务不可用还是客户端网络问题。

通过设置筛选条件并联动图表与日志数据，团队能够高效完成从“现象发现”到“根因定位”的完整分析流程。

## 操作示例：资源异常分析

以下是一个典型的资源异常分析流程示例，演示如何从发现问题到定位根因：

1. 观察【今日指标】面板，发现“失败资源数”较平时增长 300%，且“影响用户占比”达 12%。
2. 进入【图表趋势对比】查看成功率曲线，发现上午 10:15 出现明显断崖式下跌。
3. 点击该时间点，系统自动将其设为筛选条件，下方列表刷新为该时刻的异常资源。
4. 在 TOP 列表中发现某 CDN 域名 `static.example-cdn.com` 占比高达 87%，遂将其设为筛选条件。
5. 切换至【资源域名分析】，查看该域名下所有异常资源，发现多个 JS 文件加载失败，错误类型为 `net::ERR_CONNECTION_TIMED_OUT`。
6. 点击其中一条记录的“查看日志”，查看详情发现大量来自某省份移动网络的请求超时，其他运营商正常。
7. 结合网络拓扑信息，初步判断为该 CDN 节点在特定地区与运营商之间的链路异常。
8. 输出结论：本次异常由 `static.example-cdn.com` 在某省移动网络链路抖动导致，建议临时切换备用域名或通知 CDN 服务商排查。

通过以上步骤，可实现从宏观指标异常到微观日志细节的闭环分析，大幅提升排障效率。
