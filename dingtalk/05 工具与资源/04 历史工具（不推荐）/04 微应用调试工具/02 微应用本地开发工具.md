## 简介

**H5微应用本地开发工具**是钉钉开放平台新推出的H5微应用本地开发工具，开发者通过这个工具可以做到在Web浏览器中调试H5微应用和JSAPI，像开发一个普通H5应用一样开发钉钉H5微应用。

[](https://cloud.video.taobao.com/play/u/3691671841/p/1/e/6/t/1/345816423944.mp4)

## 功能简介

H5微应用开发工具目前已支持以下功能：

* 支持本地调试JSAPI。在Web浏览器开发、调试钉钉H5微应用，及调试常用的40+个JSAPI。
* JSAPI免鉴权。使用H5微应用开发工具开发H5微应用时，本地调用JSAPI不需要鉴权。
* 支持调试标签、样式、打印日志、查看网络请求等。
* 支持断点调试。
* 支持热更新。

## 使用流程

参考以下步骤，使用H5微应用开发工具进行开发。

**重要**

后续该工具将不再维护，开发H5微应用可以使用[Visual Studio Code](https://code.visualstudio.com/)，开发小程序可以使用[小程序开发工具](/document/download/miniapp-tool)，若需调试可使用[微应用四端调试工具—网页版](/document/download/micro-application-four-terminal-debugging-tool-web-edition)或[vConsole](https://www.kancloud.cn/york_web/web_node_plugin_document/2232008)代替。

1. 执行以下命令，安装**DingTalk-Design-CLI**。

   ```bash
   npm i dingtalk-design-cli@0.20.4 -g
   ```
2. 执行以下命令，查看是否已经成功安装**DingTalk-Design-CLI**。

   ```
   ding -v
   ```
3. 执行以下命令，初始化代码模板到本地。

   ```
   ding  init -o h5Test -a h5 -t h5_jsapi_component_demo_vue -l javascript
   ```

   完成初始化后，会在h5Test目录创建好代码模板，目录结构如下：

   ![[download-local-development-tools-for-microapplications_p384852.png]]
4. 执行以下命令，启动开发、调试功能。

   在刚刚初始化好的代码目录下执行以下命令，启动本地开发、调试。

   ```
   cd h5Test
   ding dev web
   ```

   命令执行成功后，会默认在本地10003端口打开「H5微应用本地开发工具」，预期效果如下。

   到这里，你就可以在Web浏览器像开发一个普通H5应用一样开发钉钉H5微应用了。

   ![[download-local-development-tools-for-microapplications_p384876.png]]

## 常见问题

### Q：本地已经存在了H5微应用，也想使用H5微应用本地开发工具，该如何使用？

**A：**如果开发者不想使用DingTalk Design CLI提供的模板进行开发，但又希望可以使用上H5微应用本地模拟器功能，我们也提供了一个额外的配置参数来解决，**请注意这是一个内测功能，在使用过程遇到任何问题请联系我们协助解决。**

只需要在执行`ding dev web`时配置参数**targetH5Url**成你本地已经存在的H5微应用的本地开发地址即可。

```
ding dev web --targetH5Url <你的调试链接>
```

例如：如果你的调试地址为`http://127.0.0.1:3000`，那么直接在`ding dev web`命令后添加你的调试地址即可。

```
ding dev web --targetH5Url http://127.0.0.1:3000
```

请注意，你本地调试的页面链接，需要在Response Headers中配置Access-Control-Allow-Origin: \*。

假如你用的是Webpack-dev-server来挂载你本地调试的页面链接，那么可以参考[webpack中文文档](https://webpack.docschina.org/configuration/dev-server/#devserverheaders)进行配置自定义的Response Header。

假如你用的是vue-cli初始化的项目，需要在项目目录下添加vue.config.js，并补充如下配置：

```
module.exports = {
    configureWebpack: {
        devServer: {
          headers: {
            'Access-Control-Allow-Origin': '*'
          }
        }
    },
}
```

### Q：为什么在模拟器中，history、location的表现异常？

A：本地开发模拟器暂时不支持history路由模式，请使用hash路由模式。

### Q：调用JSAPI runtime.permission.requestAuthCode 时报错“对应企业没有xxx域名微应用”？

![[download-local-development-tools-for-microapplications_p390890.png]]

A：将报错中显示的域名填入对应企业的微应用的应用首页地址中。
